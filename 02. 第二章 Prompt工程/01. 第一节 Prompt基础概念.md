# 第一节 Prompt 基础概念

## 从"说话"到"说对话"

你有没有过这种经历：你跟 AI 说了一句话，它给了你一个看起来很对、但仔细看根本不是你想要的回答？或者 AI 突然开始答非所问？或者同样的问题换了个说法，AI 的回答质量就大幅下降？

这些问题几乎都与一个核心概念有关：**Prompt 是什么，AI 怎么处理 Prompt**。

理解 AI 处理 Prompt 的机制，不是为了变成 AI 专家，而是为了写 Prompt 时有直觉、有判断力。当你知道模型"看到"的是什么，就知道该怎么"说"它才能理解。

---

## Prompt 的本质

### 文字输入 = 条件概率

AI 语言模型的工作方式，粗暴地理解就是：**给定一串已知的文字，预测下一个最可能出现的字**。

当你输入"请帮我写一封求职信，应聘"时，AI 做的不是"理解你的需求然后生成内容"，而是根据它训练时学到的语言规律，计算接下来每个字出现的概率，选择概率最高的那个。

这个机制有两个关键推论：

**第一，Prompt 的措辞会直接影响输出。** 因为措辞不同，模型计算出的概率分布就不同。所以"帮我写一封求职信"和"请用正式商务文体，为我撰写一封针对产品经理岗位的求职信，应聘互联网公司"给模型激活的概率空间完全不同。

**第二，模型的输出受训练数据影响。** 模型见过的语言模式越多，在特定 Prompt 下的预测就越准确。这解释了为什么用英文写 Prompt 有时效果更好（因为模型在英文语料上训练更充分）。

### Token 的概念

AI 不是处理"字"，而是处理 **Token**。Token 是文本被切分的最小单元。在中文里，一个汉字通常是一个 Token；在英文里，一个单词可能被切成一个或多个 Token（比如 "learning" 可能切成 "learn" + "ing"）。

Token 数量直接影响：
- **费用**：API 按 Token 计费
- **速度**：Token 越多，处理越慢
- **上下文限制**：模型有 Token 上限，超出就会被截断

理解 Token 有助于设计 Prompt 时有成本意识和效率意识。

## AI 理解 Prompt 的内部过程

### Attention 机制：理解词与词的关系

现代语言模型的核心是 **Attention（注意力）机制**。它的作用是：理解输入文本中每个词与其他词的关系——哪些词修饰哪些词，哪些词是主语哪些词是宾语，哪些词和哪些词在讨论同一个概念。

当你写一个 Prompt，"我有一份数据报告需要你分析，帮我找出关键结论和建议"，Attention 机制会帮你建立：报告 → 分析、关键结论 → 找出、建议 → 关键结论之间的关系。

这个机制揭示了为什么**信息组织顺序和结构会影响 AI 的理解质量**。把关键信息放在 Prompt 开头还是结尾，在某些情况下效果不同，因为 Attention 对不同位置的词的权重分配不同。

### 上下文窗口：AI 一次能看多少

每个模型都有一个 **Context Window（上下文窗口）**，这是 AI 一次能处理的最大 Token 数。超出这个范围的文本会被截断。

以 Claude 3.5 为例：
- Claude 3.5 Sonnet：200K Token 上下文窗口
- Claude 3 Opus：200K Token 上下文窗口
- Claude 3 Haiku：200K Token 上下文窗口

这意味着理论上你可以输入一部长篇小说的内容让 AI 分析。但实际使用时需要注意：
- 信息密度：模型对开头和结尾的内容注意力最强
- 位置效应：重要信息放在 Prompt 开头或结尾效果通常更好
- 截断风险：超长 Prompt 末尾的内容可能被模型"看不到"

## 好 Prompt 的底层逻辑

### 三层信息结构

一个完整有效的 Prompt，通常包含三层信息：

**第一层：任务定义（Task）**
"做什么"——你要 AI 完成什么任务。这是 Prompt 的核心，必须清晰明确。

**第二层：上下文（Context）**
"在什么情况下做"——提供必要的背景信息。AI 没有你的生活经验，需要你提供必要的背景。

**第三层：约束条件（Constraint）**
"做到什么程度/以什么方式做"——输出格式、语言风格、长度限制、专业范围等。

```
[任务] 请为我们的产品撰写一段产品介绍文案
[上下文] 产品是一款 AI 驱动的数据分析工具，面向数据分析师用户，核心卖点是"无需编程即可做复杂分析"
[约束] 300字以内，专业但易懂，包含数据指标示例，适合投放在公司官网
```

这个结构看似简单，但大多数 Prompt 只包含第一层，缺少第二和第三层，导致 AI 只能给出一个"通用版本"，而非"你要的那个版本"。

### 模糊与清晰的代价

| 问题类型 | 模糊 Prompt 的结果 | 清晰 Prompt 的结果 |
|---------|-----------------|-----------------|
| 任务不清 | AI 猜测意图，方向可能偏 | AI 执行明确的任务 |
| 上下文不足 | AI 用通用知识填补空白 | AI 用你的具体信息回答 |
| 约束不明 | AI 用自己的默认标准 | AI 按你的标准输出 |
| 示例缺失 | AI 可能误解格式要求 | AI 有明确的参照 |

## 常见误解

### 误解 1：Prompt 越长越好

不完全对。Prompt 需要**足够完整**，但不是**越长越好**。

过长的 Prompt 有几个问题：
- 增加 Token 成本
- 核心信息被稀释，模型注意力分散
- 可能触发模型的行为疲劳（在超长上下文中表现下降）

正确的思路是：**关键信息完整 + 无关信息省略**。

### 误解 2：一次 Prompt 就能完美

Prompt 是一个**迭代优化**的过程。写出一个"能用"的 Prompt 通常几分钟，但优化到一个"好用"的 Prompt 需要测试、观察、调整。

这很正常。Prompt 工程师的工作方式就是：写 → 测 → 看效果 → 改 → 再测 → 再改。

### 误解 3：AI"理解"你说的话

AI 不是真的理解，是**模式匹配**。你的 Prompt 激活了它在训练时学到的某些模式，它根据这些模式生成输出。

这意味着：AI 对"措辞"和"结构"很敏感。同样意思的两句话，不同的说法可能导致完全不同的输出。理解这一点，你就不会对 AI"听不懂人话"感到惊讶——因为它真的不是用"理解"的方式处理语言。

## 本节小结

理解 Prompt 基础概念的三个核心要点：

**1. Prompt = 条件概率预测**
AI 不是"理解"你的话再生成，而是根据 Prompt 激活的概率分布生成下一个 Token。措辞直接影响输出。

**2. 完整 Prompt 包含三层信息**
任务定义（做什么）+ 上下文（什么情况）+ 约束条件（做到什么程度）。三层完整，输出才可能精准。

**3. Prompt 是迭代优化的过程**
没有天生完美的 Prompt，只有通过测试和改进才能达到最优效果。
