# 第二节 核心工程技术

## Prompt 工程的武器库

这一节介绍 Prompt 工程中最核心的几项技术。这些技术经过大量实践验证，是目前最有效的 Prompt 优化手段。我们会逐一解释每个技术的原理、用法、以及在 Claude 上的具体实现。

---

## Zero-shot：不做任何示范

### 什么是 Zero-shot

**Zero-shot** 意思是"零样本"——不给 AI 任何示例，直接描述你的任务。

```python
Prompt: "把这段中文翻译成英文：我今天很开心"
```

这就是一个典型的 Zero-shot Prompt。AI 没有任何参考示例，完全依靠它训练时学到的语言能力来完成翻译。

### 适用场景

Zero-shot 适合：
- 任务足够通用，模型有充足的知识储备（翻译、摘要、常见写作）
- 任务描述足够清晰，不需要示例辅助理解
- 追求简洁，追求低成本

### 局限性

Zero-shot 的问题在于：当任务有特殊要求、特殊格式、或模型对这类任务的熟悉度不够时，效果会明显下降。

比如："帮我写一段话"——AI 不知道写什么主题、什么风格、多长。Zero-shot 在这类开放任务上容易给出一个"平均水平的通用回答"。

### Claude 上的实践

```python
# Zero-shot 示例（Claude）
prompt = """
请为以下产品写一句 Slogan：
产品：一款帮助你专注工作的 App，核心功能是屏蔽手机通知和社交媒体
"""

response = model.generate(prompt)
```

## Few-shot：给你看几个例子

### 什么是 Few-shot

**Few-shot** 意思是"少样本"——在 Prompt 中提供 1-5 个示例，让 AI 学会你想要的模式，然后生成类似的内容。

```python
Prompt = """
请判断评论是正面的还是负面的：

评论："这个产品太棒了，完全超出预期！"
情感：正面

评论："一般般，没有宣传的那么好用"
情感：负面

评论："还行，能满足基本需求"
情感：
"""
```

这里提供了两个示例，AI 会理解"这是什么任务"+"正确的回答是什么"，然后在第三个示例中应用这个模式。

### 为什么不直接教它做？

Few-shot 比直接说"请判断情感"更有效，原因在于：

**1. 示例比描述更具体**
"判断正面还是负面"可能有歧义（"一般般"算正面还是负面？），但示例直接给出了边界案例。

**2. 示例建立了输出格式的标准**
AI 学会了你期望的输出格式：只要"评论："和"情感："这两个标签的格式。

**3. 示例传递了风格和标准**
不仅是任务本身，还包括判断标准和表达方式。

### 最佳实践：示例选择

Few-shot 的效果高度依赖示例质量。好的示例应该：

**覆盖典型场景**：包括正面、负面、中性等不同情况
**具有边界案例**：包含容易混淆的例子
**格式一致**：所有示例格式统一，AI 才能学会模式
**数量适度**：通常 3-5 个足够，过多反而增加干扰

```python
# Few-shot 示例（Claude）
prompt = """
请将中文成语翻译成英文和解释：

塞翁失马，焉知非福
英文：a blessing in disguise
解释：坏事可能变成好事

亡羊补牢，为时未晚
英文：better late than never
解释：出了问题及时补救还来得及

杯水车薪
英文：
解释：
"""
```

### 对比 Zero-shot vs Few-shot

| 维度 | Zero-shot | Few-shot |
|------|-----------|----------|
| 示例数量 | 0 | 1-5 |
| 适用任务复杂度 | 简单、通用 | 复杂、有特殊要求 |
| 输出格式控制 | 弱 | 强 |
| 成本 | 低 | 略高 |
| 调教效果 | 一般 | 显著提升 |

## Chain-of-Thought（CoT）：让 AI 一步步想

### 什么是 Chain-of-Thought

**Chain-of-Thought（思维链）** 是一种让 AI 在给出最终答案之前，先展示推理过程的技术。

核心发现：当 AI 被要求"展示你的思考过程"时，它不仅能给出更准确的答案，推理过程本身也更可靠。

```python
# 普通 Prompt
Prompt: "小明有5个苹果，小红给了他3个，小明又吃了2个，小明现在有几个苹果？"
Answer: 6个

# CoT Prompt
Prompt: """
请一步步计算：
1. 小明一开始有5个苹果
2. 小红给了他3个，所以现在有 5 + 3 = 8 个
3. 小明吃了2个，所以现在有 8 - 2 = 6 个
答：小明现在有 6 个苹果
"""
```

在复杂推理任务上，CoT 的效果尤为明显。研究表明，CoT 在数学问题、逻辑推理、多步计算等任务上能显著提升准确率。

### Zero-shot CoT vs Few-shot CoT

**Zero-shot CoT**：直接要求 AI 展示思考过程，不需要示例

```python
Prompt: """
小明有5个苹果，小红给了他3个，小明又吃了2个，小明现在有几个苹果？
请一步步思考，然后把最终答案放在【答案】里。
"""
```

**Few-shot CoT**：提供带推理过程的示例

```python
Prompt: """
请一步步计算：

题目：小明有10块钱，买了3块钱的橡皮，剩下多少钱？
解答：10 - 3 = 7，所以小明还剩 7 块钱。

题目：小明有5个苹果，小红给了他3个，小明又吃了2个，小明现在有几个苹果？
解答：
"""
```

### Self-consistency（自洽性）

在 CoT 基础上，一个更高级的技巧是 **Self-consistency（自洽性）**——让 AI 用同一种方法多次推理，选择出现次数最多的答案。

```python
prompt = """
请用三种不同的方法计算这道题，并给出答案：
小明有5个苹果，小红给了他3个，小明又吃了2个，小明现在有几个苹果？

方法1：...
方法2：...
方法3：...

最终答案（选择出现次数最多的）：...
"""
```

Self-consistency 能显著降低模型在推理任务上的错误率，尤其在复杂数学问题上效果明显。

### Tree-of-Thought（ToT）

**Tree-of-Thought（思维树）** 是 CoT 的进阶版。在 ToT 中，AI 不只是沿着一条推理路径走，而是同时探索多条可能的推理路径，评估每个分支的可行性，然后选择最优路径。

```
                        问题
                       /  |  \
                    分支1 分支2 分支3
                    / \     |     / \
               延伸A  延伸B  延伸C  延伸D  延伸E
                  |    |      |      |
               可行✓  不可行✗  可行✓  不可行✗
                  |                        |
               选择继续                放弃并回溯
```

ToT 适合需要探索和规划的任务，比如：
- 复杂问题的多步规划
- 需要回溯的搜索问题
- 创意写作的方案探索

## Skeleton-of-Thought（思维骨架）

**Skeleton-of-Thought（思维骨架）** 是一种先让 AI 生成输出大纲，再逐项展开的技术。适用于生成长文本或复杂内容。

```python
# 第一步：生成骨架
prompt = """
为一篇"如何提升编程能力"的文章生成大纲，包含：
1. 引言
2. 至少5个核心方法（每个方法有简短的1-2句描述）
3. 总结

请只输出大纲，不要展开。
"""

# 第二步：逐项展开
prompt = """
根据以下大纲，展开写一篇完整的文章：

大纲：
## 一、引言
## 二、方法
   1. 每天写代码：坚持编码习惯
   2. 阅读源码：学习优秀项目
   3. 做项目：实战中学习
   4. 参与开源：协作中成长
   5. 写博客：输出巩固输入
## 三、总结

请展开写，每部分 200-400 字，保持专业但易读的风格。
"""
```

这个技术的好处：
- 大幅提升生成内容的结构性和完整性
- 避免长文写作时"跑题"的问题
- 生成速度更快（骨架简单，展开可控）

## Prompt Chaining（提示链）

**Prompt Chaining（提示链）** 是把一个复杂任务分解成多个步骤，每个步骤用单独的 Prompt，步骤之间传递输出。

```python
# 步骤1：提取信息
prompt1 = """
从以下文本中提取所有关键数据指标：
[文本内容...]
"""

# 步骤2：分析指标
prompt2 = f"""
基于以下指标，进行同比和环比分析：
{步骤1的输出}
"""

# 步骤3：生成报告
prompt3 = f"""
基于以下分析结果，生成一份数据分析报告：
{步骤2的输出}

报告要求：
- 结构清晰，包含摘要、发现、建议
- 关键数据用表格呈现
- 建议分短期和长期
"""
```

Prompt Chaining 的优势：
- 每个步骤的 Prompt 更简单，效果更稳定
- 便于单独调试和优化每个环节
- 可以中途插入人工审核节点
- 降低模型在长任务中的"遗忘"问题

## 技巧对比与选用指南

| 技巧 | 适用场景 | 复杂度 | 效果提升 |
|------|---------|--------|---------|
| Zero-shot | 简单通用任务 | 低 | 基线 |
| Few-shot | 有特殊格式/风格要求 | 中 | 显著提升格式控制 |
| Zero-shot CoT | 推理/计算任务 | 低 | 中等提升推理准确率 |
| Few-shot CoT | 复杂推理任务 | 高 | 大幅提升推理准确率 |
| Self-consistency | 高准确率要求的推理 | 高 | 显著降低错误率 |
| ToT | 需要探索和多方案的任务 | 很高 | 复杂任务效果更好 |
| Skeleton-of-Thought | 生成长文本/复杂内容 | 中 | 显著提升结构性 |
| Prompt Chaining | 多步骤复杂任务 | 中 | 显著提升稳定性 |

**选用建议**：
- 简单任务直接 Zero-shot
- 有格式要求 → Few-shot
- 有推理要求 → CoT
- 复杂长任务 → Chaining 或 Skeleton
- 需要高质量推理 → Self-consistency 或 ToT

## 本节小结

本节介绍了 Prompt 工程的核心技术：

- **Zero-shot**：无示例直接做，适合通用简单任务
- **Few-shot**：通过示例教会 AI 模式，显著提升格式和风格控制
- **CoT**：让 AI 展示推理过程，提升复杂任务准确率
- **Self-consistency**：多路径推理取最优，降低错误率
- **ToT**：树状探索多方案，适合需要规划的任务
- **Skeleton-of-Thought**：先大纲后展开，提升长内容质量
- **Prompt Chaining**：分解多步骤，降低复杂度提升稳定性

这些技术不是互相排斥，而是可以组合使用。实际应用中，Few-shot + CoT 是非常强大的组合。
