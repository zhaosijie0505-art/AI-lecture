# 第四节 任务特定 Prompt

## 不同任务的最优写法

代码任务、分析任务、写作任务、推理任务——它们对 Prompt 的需求不同。这一节我们针对四种最常见的任务类型，讲解最优 Prompt 策略。

---

## 代码任务

### 核心原则

代码任务的 Prompt 有几个独特要求：
- **精确性**：代码语法、函数名、参数必须准确
- **可执行性**：生成的代码不仅要看起来对，还要能跑
- **上下文相关**：AI 需要知道代码环境、依赖、语言版本

### 有效的代码 Prompt 结构

```python
prompt = """
【任务】写一个 Python 函数

【功能需求】
- 函数名：calculate_daily_return
- 输入：一只股票每日收盘价列表 [float]
- 输出：该股票每日收益率列表 [float]
- 收益率计算：(今日收盘价 - 昨日收盘价) / 昨日收盘价

【环境约束】
- Python 3.9+
- 不能使用 TA-Lib 等第三方库
- 需要处理边界情况（数据不足、负数等）

【代码规范】
- 有类型注解
- 有 docstring
- 包含基本的错误处理
"""
```

### 代码 Prompt 的关键要素

**1. 明确说明语言和环境**
"写一段代码" vs "用 Python 3.9 写一段代码"，后者 AI 更容易给出正确版本。

**2. 给出输入输出的具体类型**
这帮助 AI 理解数据结构和边界。

**3. 说明依赖限制**
"不能用第三方库"vs"可以用 pandas"会导致完全不同的代码。

**4. 要求代码质量**
类型注解、文档注释、错误处理——明确要求才能得到符合生产标准的代码。

### 代码调试 Prompt

```python
prompt = """
请帮我调试这段 Python 代码。

【代码】
def calculate_average(prices):
    total = sum(prices)
    return total / len(prices)

prices = [100, 200, None, 300]
result = calculate_average(prices)
print(result)

【问题描述】
运行时报错：TypeError: unsupported operand type(s) for +: 'int' and 'NoneType'

【已尝试的解决】
无

请：
1. 定位问题
2. 解释原因
3. 给出修复后的完整代码
"""
```

## 分析任务

### 核心原则

分析任务的核心是**结构化输出 + 数据支撑**。分析报告不能只有结论，必须有数据来源、推理过程和判断依据。

### 有效的分析 Prompt 结构

```python
prompt = """
【任务】竞品分析

【分析对象】
竞品 A：[信息]
竞品 B：[信息]
我们产品：[信息]

【分析维度】
1. 功能对比
2. 定价策略
3. 用户体验
4. 市场定位

【约束】
- 每个维度都要有具体数据支撑，不能只说"功能更丰富"
- 要给出可量化的对比结论
- 字数 1500 字以内
- 结构：摘要 → 各维度分析 → 结论 → 建议

【背景】
我们是市场新入者，定位中高端
```

### 分析 Prompt 的关键要素

**1. 提供可分析的数据**
分析需要原材料。如果你手头有数据、报告、产品信息，全部提供。如果没有，AI 只能基于公开知识分析。

**2. 明确分析维度**
"分析一下这个产品"太开放。明确说"从功能/价格/用户体验三个维度分析"，AI 会更系统。

**3. 要求数据支撑**
明确要求"每个结论必须有数据支撑"，避免 AI 写出空洞的废话分析。

**4. 指定输出结构**
分析报告最重要的不是文笔，而是结构。明确告诉 AI 报告结构：摘要/发现/结论/建议。

### 数据解读 Prompt

```python
prompt = """
【数据】
用户行为数据：
- 日活用户：10万
- 付费转化率：3%
- 付费用户月均消费：50元
- 用户留存率（30天）：25%

【分析任务】
请分析这个产品的商业模式健康度。

【要求】
1. 计算关键指标（LTV、CAC、ROI）
2. 识别潜在问题（假设获取一个付费用户成本是 30 元）
3. 给出优化建议

【输出格式】
## 指标计算
（列出计算过程和结果）

## 问题诊断
（3-5 个具体问题）

## 优化建议
（分优先级，可执行的建议）
"""
```

## 写作任务

### 核心原则

写作任务的核心是**风格控制 + 结构清晰 + 目标明确**。AI 写作最大的问题是"千人一面"，所以 Prompt 必须提供足够的风格引导。

### 有效的写作 Prompt 结构

```python
prompt = """
【任务】产品文案

【产品】[产品描述]

【目标读者】
- 群体特征：25-35 岁城市白领
- 痛点：工作忙碌、缺乏运动时间
- 需求：便捷的健身解决方案

【文案要求】
- 长度：300-500 字
- 调性：积极、温暖、不鸡血
- 风格：类比"Keep"的文案风格——真实、不说教
- 必须包含：产品核心卖点、使用场景、用户收益

【禁止】
- "革命性"、"颠覆"等夸张词汇
- 健身焦虑营销（不要暗示"不运动就完蛋"）
- 长难句（一句不超过 20 个字）
"""
```

### 不同写作类型的 Prompt 要点

**1. 技术文档**
```python
prompt = """
写一段 MongoDB 的使用文档。

要求：
- 读者是后端开发工程师
- 包含：功能说明、参数列表、使用示例、注意事项
- 风格：准确、简洁、不废话
- 示例代码必须可运行
"""
```

**2. 营销文案**
```python
prompt = """
写一条朋友圈推广文案。

产品：便携咖啡杯（保温12小时、不漏水、好清洗）

要求：
- 200 字以内
- 场景化（描述具体使用场景）
- 口语化，像朋友推荐
- 不能太像广告
- 包含：产品名、核心卖点、一句话推荐理由
"""
```

**3. 邮件**
```python
prompt = """
写一封道歉邮件。

场景：客户购买的产品延迟发货，延迟原因是仓库搬迁。

要求：
- 称呼：姓张
- 邮件结构：道歉 → 原因说明 → 补偿方案 → 承诺
- 语气：真诚、不找借口
- 补偿：免费升级快递 + 50元优惠券
- 结尾邀请客户联系
"""
```

### 写作 Prompt 的关键要素

**1. 目标读者要具体**
"写给开发者的文档" vs "写给有3年以上经验、熟悉REST API、但没接触过MongoDB的后端工程师"。后者更精准。

**2. 提供风格示例**
如果可能，给出 1-2 个"你希望的效果"的参考例子。

**3. 明确禁止项**
"不要什么"和"要什么"同样重要。

**4. 字数限制要明确**
AI 写作容易"刹不住车"，明确字数上限可以控制输出长度。

## 推理任务

### 核心原则

推理任务需要 AI "展示思考过程"而不是只给答案。这类任务用 CoT（Chain-of-Thought）效果最好。

### 有效的推理 Prompt 结构

```python
prompt = """
【任务】逻辑推理

【问题】
小明、小红、小李三人。已知：
1. 小明比小红大
2. 小李比小明大
3. 小红比小李小

请问谁的年龄最大？谁排第二？

【要求】
请一步步推理，每一步写出：
- 已知条件
- 推导过程
- 中间结论

最后给出最终答案。
"""
```

### 数学推理 Prompt

```python
prompt = """
请解答以下数学问题，并展示完整推理过程。

问题：一辆汽车从 A 地开往 B 地，时速 60 公里。
返回时因堵车，时速只有 40 公里。
已知往返总路程是 240 公里，总用时是 7 小时。
问 A 地到 B 地的距离是多少？

【要求】
1. 列出已知条件
2. 设未知数，建立方程
3. 逐步求解
4. 验证答案
"""
```

### 推理 Prompt 的关键要素

**1. 要求展示过程**
"一步步思考"或"请列出推理步骤"。

**2. 给定推理框架**
如果推理有特定框架（如数学证明、财务分析），明确告诉 AI 用什么框架。

**3. 允许 AI 质疑问题**
对于可能有问题或歧义的问题，要求 AI 先指出，再给解答。

**4. 要求验证**
让 AI 验证自己的结论是否正确。

## 任务类型选择对照表

| 任务类型 | 核心 Prompt 要素 | 推荐技术 |
|---------|----------------|---------|
| 代码编写 | 语言/环境/输入输出类型 | Few-shot + 详细约束 |
| 代码调试 | 错误信息 + 已尝试方案 | 问题-分析-解决结构 |
| 数据分析 | 分析维度 + 数据来源 | 表格化输入 + 结构化输出 |
| 竞品分析 | 对比维度 + 背景 | 分维度对比结构 |
| 写作（文案） | 读者/调性/场景 | 风格示例 + 禁止项 |
| 写作（技术） | 读者/格式/准确 | 精确描述 + 示例 |
| 数学推理 | 已知条件 + 求解目标 | CoT + 步骤展示 |
| 逻辑推理 | 条件列表 + 推理目标 | CoT + 验证步骤 |

---

## 本节小结

不同任务类型的 Prompt 最优写法：

**代码任务**：明确语言环境 + 输入输出类型 + 质量要求（类型注解/错误处理/文档）

**分析任务**：结构化输出 + 数据支撑 + 明确分析维度

**写作任务**：具体读者 + 风格引导 + 明确禁止项 + 字数限制

**推理任务**：要求展示过程 + 给定推理框架 + 验证步骤

核心原则：**越具体的任务，Prompt 可以越结构化；越开放的任务，风格引导越重要**。
