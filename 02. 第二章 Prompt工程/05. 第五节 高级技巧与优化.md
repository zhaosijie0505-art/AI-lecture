# 第五节 高级技巧与优化

## 让 Prompt 从"能用"到"好用"

这一节我们学习 Prompt 优化的高级技巧和系统化的迭代方法。写出一个"能用"的 Prompt 容易，但要让它稳定、高效、可复用，需要系统的方法。

---

## Prompt 迭代优化流程

Prompt 优化是一个循环：

```
写初版 Prompt → 测试 → 评估效果 → 定位问题 → 修改 Prompt → 再测试 → ...
```

### 第一步：定义成功的标准

优化前，先明确什么叫"做好了"。

```python
# 定义评估标准
success_criteria = {
    "准确性": "输出的答案在 95% 的情况下是正确的",
    "格式正确率": "100% 符合指定的输出格式",
    "一致性": "同样的输入重复 3 次，输出结构完全一致",
    "边界处理": "对于边界情况（空输入、极端值）能正确处理"
}
```

没有明确的成功标准，优化就没有方向。

### 第二步：构建测试集

不要只用一个例子测试。构建一个测试集，覆盖：

**正常情况**：标准的输入和预期输出
**边界情况**：空值、极端值、格式错误
**困难情况**：容易混淆或出错的例子

```python
test_cases = [
    # 正常情况
    {
        "input": "普通股票A的日交易数据",
        "expected": "完整的分析报告，包含价格、成交量、趋势分析"
    },
    # 边界情况
    {
        "input": "空数据列表 []",
        "expected": "报错提示：数据不足，无法分析"
    },
    # 困难情况
    {
        "input": "一个股票价格完全没有波动的一天",
        "expected": "如实报告无波动，并给出客观分析"
    }
]
```

### 第三步：系统性修改

不要乱枪打鸟，每次只改一个变量。

| 可能的问题 | 对应的修改方向 |
|-----------|--------------|
| 格式不稳定 | 强化输出格式约束，增加示例 |
| 内容不准确 | 提供更多背景信息，或增加验证步骤 |
| 边界处理差 | 明确边界情况下的处理规则 |
| 风格不一致 | 提供更多风格示例，强化角色设定 |
| 遗漏信息 | 增加检查清单（checklist） |

## 高级技巧

### 1. 思维锚定（Anchor）

在 Prompt 开头提供一个"锚定"陈述，帮助 AI 进入正确的思维框架。

```python
prompt = """
【思维锚定】
你是一位有15年经验的数据分析师，曾在麦肯锡和Google工作，擅长用数据讲故事。

【任务】
请分析这份销售数据...

【锚定效果】
这个开场陈述帮助 AI 调取"专业分析师"的思维模式，而不是"通用助手"的模式。
"""
```

### 2. 分解-执行-整合（Decompose-Execute-Combine）

对于复杂任务，不要让 AI 一次性完成，而是显式地要求它分解后再整合。

```python
prompt = """
【任务】为一篇长文生成摘要

【分解-执行-整合流程】

第一步：分解
请将原文分成 3-5 个逻辑段落，并为每个段落写一句话概括。

第二步：执行
对每个段落，提取：
- 主要论点（不超过3个）
- 关键数据或证据
- 与主题的相关性评分（1-5分）

第三步：整合
基于以上分析，写一段 200 字以内的摘要，涵盖原文核心观点。

请按这三步执行，每步完成后确认，再进入下一步。
"""
```

### 3. 自我反馈（Self-reflection）

要求 AI 在给出最终答案前，先进行自我审查和修正。

```python
prompt = """
请完成以下任务，然后进行自我审查。

【任务】
写一段产品介绍文案。

【自我审查检查清单】
完成初稿后，请对照以下清单检查：
1. 是否在 300 字以内？□
2. 是否突出了核心卖点（XXXX）？□
3. 是否避免了以下禁用词（XXX）？□
4. 语言风格是否符合目标读者（都市白领）？□
5. 是否有明显的语法或表达错误？□

请根据检查结果修改初稿，然后输出最终版本。
"""
```

### 4. 外部检查点（External Validation）

在长任务中插入人工审核点，确保方向正确。

```python
# 第一轮
prompt1 = """
请为我们的新产品撰写市场推广方案。

请先输出方案的框架结构：
1. 市场分析
2. 目标用户
3. 推广策略
4. 预算估算
5. 效果评估

请只输出框架，确认后再展开详细内容。
"""

# 等待用户确认框架后，再继续
prompt2 = """
基于确认的框架，请展开详细内容。
"""
```

### 5. 后置约束（Post-constraint）

有时在输出后追加约束，比在输入中包含约束更有效。

```python
prompt = """
请为这款产品写一段介绍文案。

[产品信息...]

---

【后置约束检查】
在输出你的文案后，请检查并确保：
- 字数在 200-300 字之间
- 没有使用"一流"、"顶级"等最高级词汇
- 包含了具体的用户收益而非只是功能描述

如果有任何一项不符合，请重新生成。
"""
```

### 6. 分层指令（Layered Instruction）

对于复杂场景，用多层嵌套的指令让 AI 分层理解任务。

```python
prompt = """
【第一层：角色】
你是一位资深产品经理，专注于用户体验和转化率优化。

【第二层：任务目标】
为我们的 SaaS 产品设计新用户引导流程，目标是提升激活率。

【第三层：约束】
- 用户完成引导的总步骤不超过 5 步
- 每一步的完成率目标 > 80%
- 必须包含：账户设置 → 核心功能体验 → 第一个"成功时刻"

【第四层：输出要求】
请提供：
1. 引导流程设计（步骤 + 每步的文案）
2. 每步的用户行为设计
3. 预期的转化漏斗

【第五层：验收标准】
这个设计是否达到：步骤 < 5步 AND 完成率 > 80%？
如果不满足，请调整。
"""
```

## Prompt 版本管理

当 Prompt 迭代多次后，需要管理不同版本。

### 版本命名规范

```
v1.0 - 初始版本
v1.1 - 优化了边界情况处理
v2.0 - 完全重构，适应新的产品信息
v2.1 - 调整了输出格式，改为表格输出
```

建议维护一个 Prompt 注册表：


## Claude 的高级特性

### 系统 Prompt（System Prompt）

系统 Prompt 在所有对话中持续生效，是设定 AI 长期行为的好地方。

```python
# 在 API 中使用系统 Prompt
response = model.generate(
    system="你是公司的AI助手，熟悉公司的产品、流程和规范。"
          "所有回答必须基于公司提供的信息，不要编造。"
          "当不确定时，明确说'我不确定，需要进一步确认'。",
    user="[用户的具体问题]"
)
```

系统 Prompt 适合放：
- 角色定义
- 长期行为规则
- 公司/产品的通用信息
- 绝对不能做的事

### 有效使用 XML 标签

Claude 对 XML 格式有优化。在复杂 Prompt 中使用 XML 标签区分不同类型的内容：

```python
prompt = """
<role>
你是一位资深金融分析师。
</role>

<task>
分析以下股票的 투자价值。
</task>

<data>
股票代码：AAPL
财务数据：...
</data>

<output_format>
请按以下结构输出：
## 投资概要
## 财务分析
## 风险评估
## 投资建议
</output_format>

<constraints>
- 只基于提供的数据分析，不要引用未提供的信息
- 风险评估必须包含具体风险点
</constraints>
"""
```

### Temperature 和 Top-p 控制

除了 Prompt 本身，还可以通过 API 参数控制输出的随机性：

| 参数 | 效果 | 适用场景 |
|------|------|---------|
| Temperature 低（0-0.3） | 输出更确定性，一致性高 | 代码生成、数据分析、精确任务 |
| Temperature 高（0.7-1.0） | 输出更多样，有创意 | 写作、头脑风暴、创意任务 |

```python
# 高确定性场景
response = model.generate(prompt, temperature=0.1)

# 高创意场景
response = model.generate(prompt, temperature=0.8)
```

## 常见问题与解决方案

| 问题症状 | 根本原因 | 解决方案 |
|---------|---------|---------|
| 输出格式不稳定 | 格式约束不够具体 | 增加格式示例，明确每个部分的内容要求 |
| 内容太泛 | 上下文不足 | 增加背景信息，提供具体数据 |
| 边界情况出错 | 没有覆盖边界 | 明确边界处理规则，增加边界示例 |
| 同一输入输出不一致 | 模型随机性太高 | 降低 temperature，或增加更多确定性约束 |
| 关键信息被忽略 | 信息在中间被稀释 | 把关键信息放在开头或结尾 |
| AI 过度发挥 | 任务范围不清晰 | 明确任务边界，禁止超出范围的创作 |

## 本节小结

Prompt 优化的核心方法：

**1. 系统化迭代**
定义成功标准 → 构建测试集 → 每次只改一个变量 → 记录每次变化的效果

**2. 高级技巧组合**
思维锚定 / 分解-执行-整合 / 自我反馈 / 外部检查点 / 后置约束 / 分层指令

**3. 版本管理**
命名规范 + Prompt 注册表，记录每次迭代的效果

**4. Claude 特性善用**
系统 Prompt + XML 标签 + Temperature 控制

**5. 问题定位**
格式不对 → 强化约束；内容不准 → 增加上下文；边界出错 → 明确边界规则
