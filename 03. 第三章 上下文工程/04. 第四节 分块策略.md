# 第四节 分块策略

## 长文档怎么拆

当我们需要处理超出上下文窗口限制的长文档时，分块（Chunking）就成为关键技术。分块不仅仅是"把文档切成小块"——怎么切、切多大、切成什么形状，会直接影响 AI 对内容的理解质量。

---

## 分块的基本概念

### 什么是分块

**分块（Chunking）** 是将长文档拆分成多个较短的、语义相对完整的片段的过程。每个片段作为一个独立的信息单元，可以单独检索或注入上下文。

```
长文档
  ↓
[块1] [块2] [块3] [块4] [块5] ...
  ↓
分别处理 → 单独检索 → 按需注入
```

### 为什么分块很重要

分块的重要性体现在三个方面：

**1. 上下文窗口限制**
超过窗口的内容无法被 AI 看到，分块让超长文档可以被处理。

**2. 检索效率**
在 RAG 系统中，分块决定了检索的精度——块太小容易丢失上下文，块太大容易引入噪音。

**3. 语义完整性**
如果把一句完整的话切成两半放在不同块里，AI 可能无法理解这句话的意思。好的分块确保每个块语义完整。

---

## 分块策略

### 1. 固定大小分块（Fixed Size Chunking）

最简单的分块方式——按 Token 数或字符数硬切。

```python
def fixed_size_chunk(text, chunk_size=1000, overlap=200):
    """
    固定大小分块
    chunk_size: 每个块的 Token 数（近似）
    overlap: 块之间的重叠 Token 数
    """
    chunks = []
    start = 0
    text_tokens = text.split()  # 简单按空格分词，实际用tokenizer

    while start < len(text_tokens):
        end = start + chunk_size
        chunk = " ".join(text_tokens[start:end])
        chunks.append(chunk)
        start = end - overlap  # 移动窗口，留overlap重叠

    return chunks

# 示例
text = "很长的文档内容..."
chunks = fixed_size_chunk(text, chunk_size=1000, overlap=200)
```

**优点**：简单、实现容易、可预测
**缺点**：可能在句子中间切断，破坏语义完整性

### 2. 段落分块（Paragraph Chunking）

按自然段落分割，每个块是一个或多个完整段落。

```python
def paragraph_chunk(text, min_paragraphs=1, max_paragraphs=5):
    """
    按段落分块
    """
    paragraphs = text.split("\n\n")  # 假设段落之间有两个换行
    chunks = []
    current_chunk = []

    for para in paragraphs:
        current_chunk.append(para)
        if len(current_chunk) >= min_paragraphs:
            # 达到最小段落数，可以作为一个块输出
            chunks.append("\n\n".join(current_chunk))
            current_chunk = []

    # 处理剩余段落
    if current_chunk:
        chunks.append("\n\n".join(current_chunk))

    return chunks
```

**优点**：保持段落的语义完整性
**缺点**：块大小不均匀（有的段落长，有的段落短）

### 3. 语义分块（Semantic Chunking）

基于语义相似性分块——把内容相似的句子和段落放在一起。

```python
def semantic_chunk(text, similarity_threshold=0.7):
    """
    语义分块：基于句子相似度
    """
    sentences = split_into_sentences(text)
    chunks = []
    current_chunk = [sentences[0]]

    for sentence in sentences[1:]:
        similarity = calculate_similarity(sentence, current_chunk[-1])
        if similarity > similarity_threshold:
            # 语义相似，加入当前块
            current_chunk.append(sentence)
        else:
            # 语义不相似，开启新块
            chunks.append(" ".join(current_chunk))
            current_chunk = [sentence]

    # 最后一个块
    if current_chunk:
        chunks.append(" ".join(current_chunk))

    return chunks
```

**优点**：每个块语义高度一致
**缺点**：实现复杂，需要 embedding 模型计算相似度

### 4. 递归分块（Recursive Chunking）

先用大粒度分割（如段落），如果块太大，再递归地用更小粒度分割（如句子）。

```python
def recursive_chunk(text, separators=["\n\n", "\n", ". ", " "], chunk_size=1000):
    """
    递归分块：从大粒度开始，逐步细化
    """
    def split_by_separator(text, separator):
        parts = text.split(separator)
        return [p + separator for p in parts[:-1]] + [parts[-1]] if parts else [text]

    chunks = []
    current_parts = [text]

    for separator in separators:
        new_parts = []
        for part in current_parts:
            if len(part) <= chunk_size:
                new_parts.append(part)
            else:
                # 当前块太大，用当前分隔符分割
                sub_parts = split_by_separator(part, separator)
                new_parts.extend(sub_parts)
        current_parts = new_parts

    # 合并太小的块
    chunks = merge_small_chunks(current_parts, min_size=100)

    return chunks
```

**优点**：在语义完整性和大小控制之间取得平衡
**缺点**：实现较复杂

### 5. 基于文档结构的分块（Structure-aware Chunking）

利用文档的固有结构（标题、章节、列表）来分块。

```python
def structure_chunk(document):
    """
    基于文档结构分块
    """
    chunks = []

    for section in document.sections:
        # 标题作为块的起始标记
        heading = f"## {section.title}\n"

        # 收集该章节下的所有内容
        content_parts = [heading]
        for element in section.elements:
            if element.type == "paragraph":
                content_parts.append(element.text + "\n")
            elif element.type == "list":
                # 列表作为一个整体块
                content_parts.append(str(element) + "\n")
            elif element.type == "table":
                # 表格单独作为一个块
                chunks.append(heading + str(element))
                continue

        chunk_content = "".join(content_parts)
        chunks.append(chunk_content)

    return chunks
```

**优点**：尊重文档的逻辑结构，每个块都有明确的主题
**缺点**：依赖文档有良好的结构（标题层级等）

## 块大小选择

块大小是分块策略中最影响效果的超参数。

### 块大小的权衡

```
块太小：
✓ 检索精度高（只检索到最相关的内容）
✗ 缺乏上下文（单个块可能无法让 AI 理解完整内容）
✗ 块数多，管理复杂

块太大：
✓ 上下文丰富（每个块包含足够的背景信息）
✓ 块数少，管理简单
✗ 检索精度低（引入过多无关信息）
✗ 可能超出上下文窗口
```

### 最佳实践

| 应用场景 | 推荐块大小 | 理由 |
|---------|-----------|------|
| 问答系统 | 200-500 tokens | 需要精确信息，背景不重要 |
| 摘要生成 | 500-1000 tokens | 需要足够背景，但不能太长 |
| 分析报告 | 1000-2000 tokens | 需要完整上下文 |
| 代码理解 | 函数/类级别 | 保持代码结构完整性 |

### 动态块大小

不是所有内容都适合同样的块大小。动态调整块大小：

```python
def smart_chunk(text):
    """
    根据内容类型动态选择分块策略
    """
    if is_code(text):
        # 代码：按函数/类分块
        return chunk_by_function(text)
    elif has_clear_structure(text):
        # 有明确结构：按标题分块
        return chunk_by_heading(text)
    elif is_narrative(text):
        # 叙事文本：按段落分块
        return chunk_by_paragraph(text)
    else:
        # 默认：递归分块
        return recursive_chunk(text)
```

## 块元数据

每个块应该附带元数据，帮助后续的检索和使用。

### 基本元数据

```python
class ChunkMetadata:
    chunk_id: str           # 唯一标识
    source_document: str    # 来源文档
    page_number: int        # 页码（PDF场景）
    section_title: str      # 所属章节标题
    chunk_index: int       # 在文档中的顺序
    created_at: datetime   # 创建时间

class Chunk:
    content: str           # 块的内容文本
    metadata: ChunkMetadata
    embedding: List[float] # 向量表示（用于检索）
```

### 增强元数据

```python
class EnhancedChunkMetadata:
    # 基本信息
    chunk_id: str
    source_document: str
    page_number: int
    section_title: str
    chunk_index: int

    # 内容标签
    content_type: str      # "概念解释"、"步骤指南"、"代码"、"数据"...
    topics: List[str]       # ["用户认证", "JWT", "安全"]
    entities: List[str]    # 提到的实体 ["产品A", "公司B"]

    # 质量指标
    completeness: float     # 语义完整度（0-1）
    clarity: float         # 清晰度（0-1）
```

## 块之间的连贯性

分块后单独处理每个块，可能会丢失块与块之间的上下文关联。

### 重叠窗口保持连贯

相邻块之间保留一定的重叠内容：

```
块1：[段落1][段落2][段落3]
重叠 → [段落3]
块2：      [段落3][段落4][段落5]
重叠 → [段落5]
块3：            [段落5][段落6][段落7]
```

### 显式引用

在块中加入对前后块的引用：

```python
chunk2_content = """
[接上块：上一块讨论了概念A的定义]

【本块内容：概念A的应用】

[下节预告：下一块将讨论概念A的局限性]
"""
```

### 层级索引

为块建立层级索引，表明文档的组织结构：

```python
document_tree = {
    "title": "产品手册",
    "chapters": [
        {
            "title": "第一章：入门",
            "sections": [
                {"title": "1.1 安装", "chunks": ["chunk_1", "chunk_2"]},
                {"title": "1.2 快速开始", "chunks": ["chunk_3", "chunk_4"]},
            ]
        },
        {
            "title": "第二章：进阶",
            "sections": [...]
        }
    ]
}
```

检索时，不仅返回相关的块，还返回它在文档树中的位置，帮助理解上下文。

## 本节小结

分块策略的核心要点：

| 策略 | 适用场景 | 优缺点 |
|------|---------|--------|
| 固定大小 | 快速原型、无结构文本 | 简单但可能切断语义 |
| 段落分块 | 结构清晰的文档 | 保持段落完整，但大小不均 |
| 语义分块 | 需要高精度检索 | 语义一致，但实现复杂 |
| 递归分块 | 大部分场景 | 平衡语义和大小控制 |
| 结构感知 | 有明确标题层级的文档 | 尊重逻辑结构 |

**块大小选择**：小（200-500）适合问答，大（1000-2000）适合分析，中间（500-1000）适合通用。

**块元数据**：每个块应该附带来源、位置、内容类型等元数据，便于检索和管理。

**连贯性**：通过重叠、显式引用、层级索引保持块之间的上下文关联。
