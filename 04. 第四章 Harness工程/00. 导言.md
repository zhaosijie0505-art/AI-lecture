## 为什么 AI 系统需要专门测试

### 传统软件测试 vs AI 测试

传统软件的测试是**确定性的**：

```python
# 传统软件测试：输入 A，期望输出 B
def test_add():
    assert add(2, 3) == 5  # 确定性的：2+3永远等于5
```

AI 系统的输出是**概率性的**：

```python
# AI 测试：同一个输入，可能有多种正确回答
def test_ai_summarize():
    response = ai_summarize(article)
    # response 可能是：
    # "本文讨论了气候变化的影响"（正确）
    # "文章分析了气候变化的成因和对策"（也正确）
    # "气候变化是全球关注的话题"（勉强正确）
    # 没有唯一的"正确答案"
```

这个本质差异导致：
- **传统断言不适用**：不能用 `==` 判断 AI 输出
- **需要语义评估**：判断输出是否"正确"而非是否"相等"
- **需要参考标准**：需要有 ground truth 或评估标准

### AI 测试的特殊挑战

**1. 输出的多样性**
同一个问题，AI 可能给出多个同样正确的答案。测试不能简单地判断 `expected == actual`。

**2. 评估的主观性**
"这段文字写得好不好？"——不同评估者可能有不同意见。

**3. 测试集的代表性问题**
如果测试数据分布与实际使用场景不一致，测试结果不能反映真实表现。

**4. 分布偏移（Distribution Shift）**
用户实际提问的方式可能与测试集差异很大，导致上线后效果下降。

**5. 幻觉问题**
AI 可能生成听起来正确但实际上是错误的内容。传统测试无法检测"看起来对但实际错"的情况。

## Harness 是什么

**Harness（测试工具箱）** 指的是用于评估 AI 系统的一套工具和方法。一个完整的 Harness 包括：

```
Harness（AI 测试工具箱）
├── 指标（Metrics）         → 量化评估的标准
├── 数据集（Datasets）      → 测试用的输入和参考输出
├── 评估器（Evaluators）    → 计算指标的代码
├── 报告（Reporting）      → 结果可视化
└── 基准（Benchmarks）      → 标准测试套件
```

好的 Harness 应该具备：
- **可重复性**：同样的代码和数据，每次运行结果一致
- **可解释性**：知道哪个指标差、差多少
- **可扩展性**：能添加新指标、新数据集
- **自动化**：能集成到 CI/CD 流程
