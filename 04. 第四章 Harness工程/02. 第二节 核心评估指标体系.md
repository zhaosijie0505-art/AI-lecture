# 第二节 核心评估指标体系

## 评估指标是评估体系的核心

没有指标，就没有评估。但 AI 领域的指标体系非常庞杂——有基于参考的、有无参考的、有语义的、有表面的。这一节我们系统梳理核心指标，理解它们的含义、计算方式和适用场景。

---

## 一、准确性指标

准确性衡量 AI 输出"对不对"。这是最核心的指标，但也是最复杂的——因为 AI 的正确性往往是程度问题，而非非黑即白。

### 1. Exact Match（EM）

最严格的指标：输出与标准答案完全一致才计 1 分，否则 0 分。

```python
def exact_match(prediction, reference):
    return 1.0 if prediction.strip() == reference.strip() else 0.0

# 示例
exact_match("巴黎", "巴黎")  # = 1.0
exact_match("法国的首都", "巴黎")  # = 0.0（内容对但不完全匹配）
```

**适用场景**：有唯一标准答案的任务，如选择题、填空题、事实性问题。
**局限性**：过于严格，"巴黎是法国的首都" 与 "法国的首都是巴黎" 在语义上等价，但 EM 判定为 0。

### 2. F1 Score（Token 级别）

将输出和参考都切成 Token，计算精确率和召回率的调和平均。

```python
def f1_score(prediction, reference):
    pred_tokens = set(prediction.split())
    ref_tokens = set(reference.split())

    overlap = pred_tokens & ref_tokens
    precision = len(overlap) / len(pred_tokens) if pred_tokens else 0
    recall = len(overlap) / len(ref_tokens) if ref_tokens else 0

    if precision + recall == 0:
        return 0.0
    return 2 * precision * recall / (precision + recall)

# 示例
f1_score("巴黎 法國 首都", "法國 首都")  # 考虑部分匹配
```

**适用场景**：开放性问答、实体识别、信息抽取。
**局限性**：不考虑词序和语义，"北京是首都" 和 "首都是北京" F1 可能相同。

### 3. ROUGE Score

衡量生成文本与参考文本的 n-gram 重叠度。常用变体：

| 指标 | 衡量内容 |
|------|---------|
| ROUGE-1 | unigram 重叠（单词级） |
| ROUGE-2 | bigram 重叠（词组级） |
| ROUGE-L | 最长公共子序列 |

```python
from rouge import Rouge

rouge = Rouge()
scores = rouge.get_scores(
    "巴黎是法国的首都",
    "法国的首都巴黎"
)
# scores[0]['rouge-1']['f']  ~ 0.86
# scores[0]['rouge-l']['f']  ~ 0.86
```

**适用场景**：摘要生成、机器翻译评估。
**局限性**：与人类判断相关性较低——高 ROUGE 不一定代表质量高。

### 4. BLEU Score

机器翻译领域的标准指标，衡量 n-gram 精确率。

```python
from sacrebleu import bleu

score = bleu(
    ["The cat is on the mat"],
    ["There is a cat on the mat"]
)
print(score.score)  # ~53.0（0-100，越高越好）
```

**适用场景**：机器翻译，以及其他需要与参考译文对比的场景。
**局限性**：
- 只衡量精确率，不衡量召回
- 对短句子惩罚（短参考译文更难匹配）
- 与人类判断相关性不高

### 5. BERTScore

利用预训练语言模型（如 BERT）计算语义相似度，而非字面匹配。

```python
from bert_score import score

P, R, F1 = score(
    cand=["巴黎是法国的首都"],
    ref=["法国的首都巴黎"],
    lang="zh"
)
# F1 ~ 0.98（语义几乎相同）
```

**优势**：捕捉语义相似性，比 ROUGE/BLEU 更符合人类判断。
**适用场景**：需要语义评估的所有场景，尤其是开放性生成。
**局限性**：计算成本较高；可能对同义词过度宽容。

### 6. LLM as Judge

用大模型来判断输出质量，是目前最接近人类评估的方法。

```python
def llm_judge(query, response, criteria):
    """
    用 LLM 作为评估器
    """
    prompt = f"""
请评估以下回答的质量。

问题：{query}
回答：{response}

评估标准：{criteria}

请给出 1-5 分的评分，并说明理由。
"""
    result = llm.generate(prompt)
    return parse_score(result)

# 示例评分标准
criteria = """
- 准确性（5分：完全正确，1分：完全错误）
- 完整性（5分：回答完整，1分：遗漏关键信息）
- 清晰度（5分：表达清晰，1分：难以理解）
"""
```

**优势**：能评估无法用规则定义的维度（流畅性、有用性、合理性）。
**挑战**：需要仔细设计的 Prompt 才能保证一致性；LLM 作为评判者可能存在偏好。

## 二、RAG 系统评估指标

RAG（检索增强生成）系统的评估需要同时衡量检索和生成两个部分。

### 1. 检索指标

| 指标 | 说明 | 计算方式 |
|------|------|---------|
| **Hit Rate@K** | Top-K 结果中是否有相关文档 | 有=1，无=0 |
| **MRR@K** | 第一个相关结果的位置倒数均值 | 1/rank 的平均值 |
| **NDCG@K** | 考虑相关度等级的排序质量 | DCG/IDCG |

```python
def hit_rate_at_k(retrieved_docs, relevant_docs, k):
    """Hit Rate@K"""
    retrieved_k = retrieved_docs[:k]
    return 1.0 if any(doc in relevant_docs for doc in retrieved_k) else 0.0

def mrr_at_k(retrieved_docs, relevant_docs, k):
    """MRR@K"""
    for i, doc in enumerate(retrieved_docs[:k], 1):
        if doc in relevant_docs:
            return 1.0 / i
    return 0.0

# 示例
retrieved = ["doc1", "doc2", "doc3"]
relevant = ["doc2"]
hit_rate_at_k(retrieved, relevant, k=3)  # = 1.0
mrr_at_k(retrieved, relevant, k=3)       # = 0.5（doc2在第2位）
```

### 2. 生成指标

| 指标 | 说明 |
|------|------|
| **Faithfulness** | 回答是否忠实于检索到的文档（不编造） |
| **Answer Relevancy** | 回答与问题的相关性 |
| **Context Precision** | 检索到的上下文块的相关度排名 |
| **Context Recall** | 检索到的内容覆盖了多少参考答案 |

这些是 RAGAS 框架的核心指标，我们下一节会详细讲解。

## 三、质量与安全指标

### 1. 毒性（Toxicity）

衡量输出是否包含有害内容（脏话、仇恨言论、暴力等）。

```python
# 使用 Perspective API（Google 提供）
from perspective_api import PerspectiveAPI

client = PerspectiveAPI(api_key="YOUR_KEY")

def toxicity_score(text):
    result = client.analyze(text)
    return result["TOXICITY"]["score"]

# 示例
toxicity_score("你这个白痴")  # ~0.85（高毒性）
toxicity_score("今天天气不错")  # ~0.02（无毒）
```

开源替代方案：Detoxify 库（基于 RoBERTa 微调）。

### 2. 偏见（Bias）

衡量输出是否对特定群体存在不公平偏见。

```python
# 使用 Bbq（Bias Benchmark for QA）
from BBQ import BBQ

bbq = BBQ(model=my_llm)

# 测试多个敏感维度的偏见
bias_results = bbq.evaluate(
    dataset="bbq",
    categories=["gender", "race", "age"]
)
# bias_results["gender"]["score"]  # 越高表示偏见越严重
```

### 3. 幻觉（Hallucination）

衡量模型是否生成了看似正确但实际错误的内容。

**事实核查方法**：
```python
def hallucination_check(answer, claims):
    """
    提取答案中的声明，检查是否与事实一致
    """
    extracted_claims = extract_claims(answer)  # 从答案中提取事实声明
    veracity_scores = []

    for claim in extracted_claims:
        is_verified = fact_check(claim, knowledge_base)
        veracity_scores.append(1.0 if is_verified else 0.0)

    # 幻觉率 = 不实声明 / 总声明
    hallucination_rate = 1 - mean(veracity_scores)
    return hallucination_rate

def extract_claims(text):
    """用 NER 和关系抽取从文本中提取可验证的事实声明"""
    # 简化版：用规则提取数字和实体组合
    import re
    patterns = [
        r"(\w+\s+\d+\s+\w+)",  # "北京 2000 万人"
        r"(\w+\s+\w+\s+是\s+\w+)",  # "巴黎 是 首都"
    ]
    claims = []
    for pattern in patterns:
        claims.extend(re.findall(pattern, text))
    return claims
```

**SelfCheckGPT 方法**：用同一个问题多次采样生成，对比一致性。一致性低的更可能有幻觉。

## 四、效率指标

### 1. 延迟（Latency）

从请求发出到收到响应的时间。

```python
import time

def measure_latency(prompt, iterations=10):
    latencies = []
    for _ in range(iterations):
        start = time.time()
        response = llm.generate(prompt)
        latencies.append(time.time() - start)

    return {
        "mean": mean(latencies),
        "median": median(latencies),
        "p95": percentile(latencies, 95),
        "p99": percentile(latencies, 99),
    }
```

### 2. 成本（Cost）

推理成本通常按 Token 数计费。

```python
def estimate_cost(prompt, response, price_per_1k_input=0.003, price_per_1k_output=0.015):
    input_tokens = count_tokens(prompt)
    output_tokens = count_tokens(response)

    cost = (input_tokens / 1000) * price_per_1k_input
    cost += (output_tokens / 1000) * price_per_1k_output
    return cost

# 示例：GPT-4o mini 价格
estimate_cost(
    prompt="..." * 1000,  # 约 1000 tokens
    response="..." * 500,  # 约 500 tokens
    price_per_1k_input=0.00015,
    price_per_1k_output=0.0006
)
# ≈ $0.00045 per request
```

### 3. 吞吐量（Throughput）

系统每秒能处理的请求数。

```python
from concurrent.futures import ThreadPoolExecutor

def measure_throughput(llm, prompts, max_workers=10):
    start = time.time()
    with ThreadPoolExecutor(max_workers=max_workers) as executor:
        results = list(executor.map(llm.generate, prompts))
    duration = time.time() - start

    return {
        "total_requests": len(prompts),
        "duration_seconds": duration,
        "requests_per_second": len(prompts) / duration,
        "avg_latency": duration / len(prompts)
    }
```

## 五、综合评分体系设计

单一指标不够，需要综合多个指标设计评分体系。

### 雷达图评估

```python
import numpy as np

class AIEvalRadar:
    """雷达图式综合评估"""

    def __init__(self, weights=None):
        self.weights = weights or {
            "accuracy": 0.30,      # 准确性 30%
            "relevance": 0.20,     # 相关性 20%
            "fluency": 0.15,       # 流畅性 15%
            "safety": 0.20,        # 安全性 20%
            "efficiency": 0.15,    # 效率 15%
        }

    def evaluate(self, metrics):
        """
        metrics: dict，各指标的原始分数（0-1）
        返回: (加权总分, 各维度分数)
        """
        scores = {}
        for dimension, weight in self.weights.items():
            score = metrics.get(dimension, 0)
            scores[dimension] = score * weight

        total = sum(scores.values())

        # 归一化
        normalized_scores = {k: v / total * 100 for k, v in scores.items()}

        return total, normalized_scores

# 使用
evaluator = AIEvalRadar()
total, breakdown = evaluator.evaluate({
    "accuracy": 0.85,
    "relevance": 0.92,
    "fluency": 0.88,
    "safety": 0.95,
    "efficiency": 0.78
})
print(f"总分: {total:.2%}")
print(f" breakdown: {breakdown}")
```

### PASS/FAIL 阈值

很多企业场景不需要综合分数，只需要判断是否达到上线标准。

```python
def pass_fail_check(metrics, thresholds):
    """
    判断是否通过上线标准
    """
    results = {}
    for metric, value in metrics.items():
        threshold = thresholds.get(metric, 0)
        results[metric] = {
            "value": value,
            "threshold": threshold,
            "pass": value >= threshold
        }

    all_passed = all(r["pass"] for r in results.values())

    return {
        "overall": "PASS" if all_passed else "FAIL",
        "details": results
    }

# 上线标准
thresholds = {
    "accuracy": 0.85,
    "safety": 0.95,     # 安全性必须 95% 以上
    "latency_p99": 2000,  # P99 延迟不超过 2 秒（毫秒）
    "toxicity": 0.1,    # 毒性不超过 10%
}

result = pass_fail_check(my_metrics, thresholds)
```

## 本节小结

准确性指标：EM、F1、ROUGE 适合有标准答案的场景；BERTScore、LLM as Judge 适合语义评估。

RAG 指标：Hit Rate、MRR、NDCG 衡量检索；Faithfulness、Answer Relevancy 衡量生成。

质量与安全：Toxicity、 Bias、Hallucination 是关键安全指标，需要持续监控。

效率指标：Latency、Cost、Throughput 决定系统能否在生产环境运行。

综合评分：雷达图适合多维度对比，PASS/FAIL 阈值适合上线决策。
