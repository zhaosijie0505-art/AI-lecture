# 第三节 RAG 评估工具 RAGAS

## RAGAS 是什么

**RAGAS**（Retrieval-Augmented Generation Assessment）是目前最流行的 RAG 系统评估框架，专门用于评估 RAG 系统的端到端表现。它由探索大模型领域的学者开发，核心思想是：**RAG 系统的质量由"检索"和"生成"两个部分共同决定，需要分别评估、关联分析。**

RAGAS 的关键优势：
- **无需人工标注参考答案**：通过 LLM 自动生成评估
- **指标设计贴合 RAG 特性**：专门为 RAG 场景设计，而非通用 NLP 指标
- **支持快速集成**：几行代码即可评估现有 RAG 系统

---

## RAGAS 四大核心指标

RAGAS 定义了四个核心指标，从不同维度衡量 RAG 系统：

```
┌─────────────────────────────────────────────────────────┐
│                    RAGAS 四指标                          │
├──────────────────┬──────────────────────────────────────┤
│                  │                                      │
│   检索（Retriever）│            生成（Generator）         │
│                  │                                      │
│  ┌─────────────┐ │  ┌─────────────┐  ┌─────────────┐    │
│  │  Context    │ │  │ Faithfulness│  │Answer       │    │
│  │  Precision  │ │  │  （忠诚度）  │  │Relevancy    │    │
│  │  （上下文   │ │  │  回答是否   │  │  （回答     │    │
│  │   精确度）  │ │  │  忠实于     │  │  相关性）   │    │
│  └─────────────┘ │  │  检索内容） │  └─────────────┘    │
│                  │  └─────────────┘                    │
│  ┌─────────────┐ │  ┌─────────────────────────────┐    │
│  │  Context    │ │  │    Answer Correctness      │    │
│  │  Recall     │ │  │    （回答正确性）           │    │
│  │  （上下文   │ │  │    整体质量，与ground truth  │    │
│  │   召回率）  │ │  │    对比                    │    │
│  └─────────────┘ │  └─────────────────────────────┘    │
│                  │                                      │
└──────────────────┴──────────────────────────────────────┘
```

### 1. Faithfulness（忠诚度）

**衡量回答是否忠实于检索到的上下文**——这是检测"幻觉"的核心指标。

- Faithfulness 高 = 回答中的所有声明都能在上下文中找到依据
- Faithfulness 低 = 回答包含上下文中没有的信息（可能是幻觉）

```python
# RAGAS Faithfulness 示例
from ragas import evaluate
from datasets import Dataset

# 准备测试数据
data = {
    "user_input": ["What did the president say about Ketanji Brown Jackson?"],
    "retrieved_contexts": [["The president said she is one of the most.common ..."]],
    "response": ["The president described her as one of the most brilliant ..."],
    "ground_truth": ["The president said she is one of the most common ..."]
}

dataset = Dataset.from_dict(data)

# 评估
result = evaluate(dataset, metrics=[faithfulness])
print(result)
# faithfulness: 0.92（92%的回答内容可在上下文中找到依据）
```

**计算原理（RAGAS 内部）**：
1. 给 LLM 提供 response 和 retrieved context
2. 让 LLM 提取 response 中的所有声明
3. 让 LLM 判断每个声明是否能从 context 中推导出来
4. Faithfulness = 有依据的声明数 / 总声明数

### 2. Answer Relevancy（回答相关性）

**衡量回答与问题的相关程度**——回答是否真正回应了问题，是否有遗漏或冗余。

- Answer Relevancy 高 = 回答直接切中问题，没有跑题
- Answer Relevancy 低 = 回答跑题、冗余、或遗漏关键点

```python
# RAGAS Answer Relevancy 示例
from ragas.metrics import AnswerRelevancy

# Answer Relevancy 计算
# 通过反向提问：基于回答生成多个问题，看这些生成的问题与原始问题的相似度
# 相似度越高，说明回答越切中要点

# 原始问题："如何学习编程？"
# 回答："学习编程需要先选一门语言，如Python..."
# 生成的反向问题："应该选哪门编程语言？"（与原问题相似）
# 生成的反向问题2："编程入门需要什么？"（与原问题相似）
# 生成的反向问题3："Python适合初学者吗？"（与原问题相关）

# 如果生成的问题都与原问题相似 → Answer Relevancy 高
```

**计算原理**：
1. 基于 response 用 LLM 生成 N 个可能的问题
2. 计算这 N 个问题与原始 question 的余弦相似度
3. 取平均作为 Answer Relevancy 分数

### 3. Context Precision（上下文精确度）

**衡量检索到的上下文块中，高相关内容的排序是否靠前**。

- Context Precision 高 = 最相关的文档排在最前面
- Context Precision 低 = 相关文档被埋在后面，无关文档排在前面

```python
# Context Precision 计算
# 只考虑相关文档在 Top-K 结果中的排名

def context_precision(retrieved_contexts, relevant_context_ids, k=5):
    """
    retrieved_contexts: 检索返回的文档列表
    relevant_context_ids: 实际相关的文档 ID 集合
    k: 只看 Top-K
    """
    top_k = retrieved_contexts[:k]
    relevant_in_top_k = sum(1 for doc in top_k if doc.id in relevant_context_ids)

    # 用精确率的变体：相关文档在 Top-K 中的比例
    precision = relevant_in_top_k / k if k > 0 else 0

    # 考虑排名位置（越靠前权重越高）
    weighted_precision = 0
    for i, doc in enumerate(top_k, 1):
        if doc.id in relevant_context_ids:
            weighted_precision += 1 / i

    return weighted_precision
```

### 4. Context Recall（上下文召回率）

**衡量检索到的内容是否覆盖了回答问题所需的全部信息**。

- Context Recall 高 = 检索到的内容足够完整，能够回答问题
- Context Recall 低 = 检索内容有遗漏，无法完整回答

```python
# Context Recall 计算
# 需要 ground truth（完整答案或相关信息）

def context_recall(retrieved_contexts, ground_truth_contexts):
    """
    retrieved_contexts: 检索到的上下文
    ground_truth_contexts: 完整的相关上下文（理想情况下应该检索到的）
    """
    # 将两者都 tokenize
    retrieved_sentences = set(tokenize(retrieved_contexts))
    gt_sentences = set(tokenize(ground_truth_contexts))

    # 计算有多少 ground truth 句子被检索到了
    overlap = retrieved_sentences & gt_sentences

    return len(overlap) / len(gt_sentences) if gt_sentences else 0

# 示例
ground_truth = "巴黎是法国的首都，法国位于欧洲，人口约6700万"
retrieved = "巴黎是法国的首都"  # 遗漏了"位于欧洲"和"人口6700万"
# context_recall = 1/3 ≈ 33%
```

### 5. 诊断问题

```python
def diagnose_rag_issues(result_df):
    """
    诊断 RAG 系统的主要问题
    """
    issues = []

    # 检查忠诚度问题
    low_faithfulness = result_df[result_df['faithfulness'] < 0.8]
    if len(low_faithfulness) > 0:
        issues.append({
            "type": "HALLUCINATION",
            "count": len(low_faithfulness),
            "description": "这些回答包含上下文中没有的信息（幻觉）",
            "examples": low_faithfulness[["user_input", "response"]].head(3).to_dict()
        })

    # 检查回答相关性问题
    low_relevancy = result_df[result_df['answer_relevancy'] < 0.8]
    if len(low_relevancy) > 0:
        issues.append({
            "type": "IRRELEVANT_ANSWER",
            "count": len(low_relevancy),
            "description": "这些回答没有很好地回应问题",
            "examples": low_relevancy[["user_input", "response"]].head(3).to_dict()
        })

    # 检查检索召回问题
    low_recall = result_df[result_df['context_recall'] < 0.8]
    if len(low_recall) > 0:
        issues.append({
            "type": "INCOMPLETE_RETRIEVAL",
            "count": len(low_recall),
            "description": "检索内容不够完整，无法回答问题",
            "examples": low_recall[["user_input", "retrieved_contexts"]].head(3).to_dict()
        })

    return issues

issues = diagnose_rag_issues(df)
for issue in issues:
    print(f"\n问题类型: {issue['type']}")
    print(f"影响数量: {issue['count']}")
    print(f"描述: {issue['description']}")
```

## 局限性

RAGAS 很优秀，但也有局限：

### 1. 依赖 LLM 作为评估器
RAGAS 用 LLM 评判答案质量，但 LLM 本身可能存在偏见和错误判断。

### 2. 需要 ground truth
Context Recall 的计算需要 ground truth，而 ground truth 通常很难获取。

### 3. 不评估检索的"新鲜度"
RAGAS 假设检索到的文档都是相关的，但不评估文档本身是否是最新的。

### 4. 成本考虑
每个评估数据点需要多次 LLM 调用，大规模评估成本不低。

## 本节小结

RAGAS 四大核心指标：

| 指标 | 衡量内容 | 理想值 | 低分意味着什么 |
|------|---------|--------|--------------|
| Faithfulness | 回答是否基于检索内容 | > 0.9 | 幻觉严重 |
| Answer Relevancy | 回答是否切中问题 | > 0.85 | 跑题或冗余 |
| Context Precision | 相关内容排序是否靠前 | > 0.85 | 排序不合理 |
| Context Recall | 检索内容是否完整 | > 0.85 | 检索有遗漏 |

使用 RAGAS 的完整流程：**准备数据 → 选择指标 → 执行评估 → 分析结果 → 诊断问题 → 优化**。
