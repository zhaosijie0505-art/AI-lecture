# 第四节 主流 Benchmark 详解

## 为什么需要 Benchmark

在比较 AI 模型时，你需要一个"标准考试"来判断谁更强。Benchmark 就是这个标准考试——它提供统一的测试题库、评分标准和排行榜，让不同模型可以在同一条件下比较。

Benchmark 的核心价值：
- **可比性**：在同一标准下比较不同模型
- **可重复性**：任何人都能用同样方法复现结果
- **全面性**：覆盖多个维度，不偏颇单一能力

但 Benchmark 也有局限——它测量的是模型在特定测试集上的表现，不等于在实际应用中的效果。

## LLM 评估 Benchmark 全景

```
学术 Benchmark
├── 通用能力 ───── BIG-bench, HELM, MMLU
├── 推理能力 ───── GSM8K, MATH, ARC
├── 代码能力 ───── HumanEval, MBPP, SantaCoder
├── 中文能力 ───── C-Eval, CMMLU
└── 特定任务 ───── TruthfulQA, RealToxicityPrompts

对话/指令 Benchmark
├── MT-Bench（多轮对话）
├── AlpacaEval（指令遵循）
└── ChatArena（人类偏好）

RAG Benchmark
├── RAGAS（RAG 全链路）
├── Multi-doc QA（多文档问答）
└── Hoops（多跳推理）
```

## MMLU（大规模多任务语言理解）

**MMLU**（Massive Multitask Language Understanding）是目前最流行的 LLM 基准测试之一。

### 覆盖范围

57 个学科，涵盖：
- 基础学科：数学、历史、医学、物理
- 专业领域：法律、伦理、计算机科学、经济学
- 日常知识：常识、公共事务

### 测试格式

全是选择题，四选一：

```
示例问题：
"Which of the following is the largest planet in our solar system?
(A) Earth (B) Mars (C) Jupiter (D) Saturn"

正确答案：C
```

### 评分标准

```
GPT-3（2020）：43.9%
GPT-3.5（2022）：67.0%
GPT-4（2023）：86.4%
Claude 2（2023）：78.0%
Claude 3 Opus（2024）：86.4%
Gemini Ultra（2024）：90.0%
GPT-4o（2024）：88.7%
```

90% 的准确率意味着什么？意味着模型在 57 个学科的选择题上表现得和受过良好教育的普通人差不多。

### 为什么 MMLU 重要

MMLU 之所以重要，是因为：
1. **覆盖广泛**：57 个学科几乎没有其他测试能做到
2. **知识密集**：需要大量的事实知识和基础推理
3. **难度递增**：题目从基础到专业都有

## BIG-bench（Google 的大规模评测）

**BIG-bench**（Beyond the Imitation Game）是由 Google 联合 450 位研究者共同发布的超大规模评测套件，包含 **200+ 任务**。

### 任务类型

| 类型 | 示例任务 | 数量 |
|------|---------|------|
| 推理 | 逻辑推理、数学推导 | 35 |
| 语言理解 | 情感分析、文本分类 | 50+ |
| 知识 | 常识问答、科学知识 | 40+ |
| 代码 | 代码生成、代码解释 | 15 |
| 伦理 | 偏见检测、有害内容识别 | 20+ |
| 多模态 | 图像描述、视觉推理 | 10 |

### 特别任务：Shots Across the Bow

BIG-bench 包含一个有趣的测试：**Shots Across the Bow**——检测模型是否在训练时"见过"测试题。如果一个模型在训练时大量"刷题"（直接记忆答案），在 BIG-bench 上表现会异常好。

### 评估结果示例

```python
# BIG-bench 评分（按任务平均）
PaLM 540B: 58.3 分（相对于人类表现的比例）
GPT-3: 45.3 分
SOTA (2023): 65+ 分

# 人类基准线: 100 分
# 随机猜测: 25 分（选择题）
```

## GSM8K（小学数学应用题）

**GSM8K**（Grade School Math 8K）专门测试模型的数学推理能力。

### 特点

- 8,500 道小学数学应用题
- 需要多步推理才能解答
- 不需要计算器，数字都在可心算范围内
- 每道题有完整的解题步骤（chain-of-thought）

```python
# GSM8K 示例
问题：
"James has 3 packs of baseball cards.
Each pack has 5 cards. He buys 2 more packs.
How many cards does he have in total?"

解题步骤：
- James 原来有 3 × 5 = 15 张卡片
- 又买了 2 × 5 = 10 张卡片
- 总共 15 + 10 = 25 张卡片

正确答案：25
```

### 性能对比

```
GPT-3（few-shot）：约 5%
GPT-3.5 + CoT：约 50%
GPT-4 + CoT：约 90%+
Claude 3 Opus：约 95%+
```

从 5% 到 95%+，这是 Prompt 工程（CoT） + 模型能力提升共同带来的进步。

## TruthfulQA（真实性评估）

**TruthfulQA** 测试模型是否能抵抗"幻觉"，不被错误信息误导。

### 测试方式

问题往往是"陷阱"——有些问题基于常见误解，模型如果"想当然"地回答，就会出错：

```
问题："Which horse is the tallest, a Thoroughbred or a Clydesdale?"
正确答案：骡马赛跑（Thoroughbred 更高）
错误答案："Clydesdale"（Clydesdale 是挽马，更矮）

问题："What is the smallest country in the world?"
正确答案：梵蒂冈（约 0.44 km²）
错误答案：可能有人会猜"摩纳哥"或"马耳他"
```

### 评分标准

- **Truthful**：回答是否与真实世界一致
- **Informative**：回答是否提供了足够的信息

好的回答应该是"真实且有信息量的"（Truthful + Informative）。

### 典型结果

```
人类基线：Truthful 94% / Informative 87%
GPT-3：Truthful 23% / Informative 94%（说实话的能力差）
GPT-4：Truthful 82% / Informative 87%
Claude 2：Truthful 85% / Informative 85%
```

## MT-Bench（多轮对话评估）

**MT-Bench**（Multi-Turn Benchmark）测试模型在多轮复杂对话中的表现。

### 测试设计

- 10 个类别，每个类别 2 道多轮问题（共 80 道）
- 类别：信息提取、逻辑推理、数学、编程、头脑风暴、写作、知识提取、角色扮演、总结、翻译
- 第一轮：直接提问
- 第二轮：追问或转折（如"但是..."、"能否更具体？"）

```python
# MT-Bench 示例（第一轮 + 第二轮）

# 第一轮
问题1: "解释为什么冬天会下雪"
回答1: "因为气温降低，空气中的水蒸气在凝结过程中形成降雪..."

# 第二轮（追问）
追问1: "那你能不能用更生活化的比喻来解释？"
回答2: "想象你在浴室洗热水澡，水蒸气遇到冷的镜子会变成小水珠..."
```

### 评分方式

用 GPT-4 作为 Judge，给回答打 1-10 分。对比两个模型时，让它们回答同一问题，用 Judge 打分比较。

### 典型结果

```
GPT-4: 8.99 分
GPT-3.5: 7.94 分
Claude 2: 8.68 分
Llama-2 70B: 7.12 分
```

## C-Eval 和 CMMLU（中文评测）

对于中文模型，需要专门的中文 Benchmark。

### C-Eval

清华大学发布的**中文基础模型评测基准**，覆盖 52 个学科：

```
C-Eval 覆盖学科（部分）：
├── 基础教育：初中语文、高中数学、高中物理
├── 大学专业：计算机科学、经济学、法学、医学
├── 职业资格：律师、医生、会计
└── 思想教育：马克思主义、毛泽东思想
```

测试格式与 MMLU 类似（选择题），但全是中文语境。

### CMMLU

**CMMLU**（Chinese Massive Multitask Language Understanding）是专门针对中文知识和推理的评测：

```
覆盖领域：67个主题
涵盖：中国文化、历史、地理、成语、风俗
额外覆盖：中医、传统节日、中国特定概念（如"四个自信"）
```

### 为什么需要中文 Benchmark

英文 Benchmark 不等于中文能力：

```
MMLU（英文）：
Claude 3 Opus: 86.4%

C-Eval（中文）：
Claude 3 Opus: ~75%（中文理解能力明显弱于英文）
```

很多中文语境特有的知识（如中国历史、文化、成语），在英文 Benchmark 中无法测试。

## 如何正确使用 Benchmark

### 1. 选对 Benchmark

| 你的需求 | 对应 Benchmark |
|---------|--------------|
| 评估通用知识 | MMLU、C-Eval |
| 评估推理能力 | GSM8K、MATH |
| 评估代码能力 | HumanEval、MBPP |
| 评估中文能力 | C-Eval、CMMLU |
| 评估对话质量 | MT-Bench、AlpacaEval |
| 评估 RAG | RAGAS |
| 评估安全性 | TruthfulQA、RealToxicityPrompts |

### 2. 不要只看单一分数

```python
# 示例：模型综合评估
results = {
    "model_a": {
        "mmlu": 82,      # 知识强
        "gsm8k": 45,     # 推理弱
        "truthfulqa": 75,
    },
    "model_b": {
        "mmlu": 78,      # 知识中等
        "gsm8k": 92,     # 推理强
        "truthfulqa": 80,
    }
}

# model_a 更适合知识密集型任务
# model_b 更适合推理密集型任务
# 没有"最好"的模型，只有"最适合"的模型
```

### 3. Benchmark 分数只是参考

Benchmark 分数高不等于实际应用表现好。Benchmark 只能告诉你"模型在特定测试集上的表现"，无法预测"模型在你真实场景中的表现"。

### 4. 自己构建测试集

对于企业应用，建议在 Benchmark 之外，额外构建**自己的测试集**：

```python
# 企业自定义测试集
custom_test_set = [
    {
        "id": "case_001",
        "category": "产品咨询",
        "question": "你们的产品支持多少并发用户？",
        "ground_truth": "我们的产品支持根据套餐不同，从100并发到无限并发...",
        "criteria": {
            "accuracy": "数值范围正确",
            "completeness": "覆盖所有套餐",
            "clarity": "表达清晰"
        }
    },
    # ... 更多用例
]
```

## 本节小结

主流 Benchmark 有几个类别：

**通用能力**：MMLU（57学科选择题）、BIG-bench（200+任务综合）
**推理能力**：GSM8K（小学数学）、MATH（中学数学竞赛）
**安全性**：TruthfulQA（真实性）、RealToxicityPrompts（毒性检测）
**对话/指令**：MT-Bench（多轮对话）、AlpacaEval（指令遵循）
**中文**：C-Eval（52学科中文）、CMMLU（67主题中文）

Benchmark 是参考，不是绝对标准。选择与你的应用场景最相关的 Benchmark 组合来评估。
