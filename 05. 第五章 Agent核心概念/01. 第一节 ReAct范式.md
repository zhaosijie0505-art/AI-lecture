# 第一节 ReAct 范式

## ReAct 是什么

**ReAct**（Synergizing Reasoning and Acting）是一种让 LLM 能够**同时进行推理和行动**的提示词方法。它由 Google Research 在 2022 年提出，核心思想是：让 AI 在每一步行动后，不仅输出思考（Thought），还要输出下一步行动（Action），以及行动后得到的结果（Observation），形成"想→做→看→想→做→看..."的循环。

### ReAct 的核心循环

```
┌─────────────────────────────────────────────────────────────┐
│                    ReAct 循环                                │
│                                                             │
│   ┌─────────┐                                               │
│   │  Thought │ ← AI 分析当前状态，决定要做什么              │
│   └────┬────┘                                               │
│        ↓                                                    │
│   ┌─────────┐                                               │
│   │  Action │ ← AI 调用工具执行动作                         │
│   └────┬────┘                                               │
│        ↓                                                    │
│   ┌─────────────┐                                           │
│   │ Observation │ ← 工具返回结果，AI 观察到了什么           │
│   └──────┬──────┘                                           │
│          ↓                                                  │
│          └──→ 如果任务完成 → 输出最终答案                   │
│              如果未完成 → 继续循环（回到 Thought）          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### ReAct vs 其他方法对比

| 方法 | 描述 | 特点 |
|------|------|------|
| **Chain-of-Thought** | 纯粹推理，不行动 | 只能解决纯推理问题 |
| **Act-Only** | 直接行动，不推理 | 盲目行动，容易出错 |
| **ReAct** | 边推理边行动 | 可处理需要外部信息的复杂任务 |
| **Plan-then-Execute** | 先规划所有步骤，再执行 | 全局视角，但灵活性差 |

## ReAct 的实现原理

### 提示词结构

```python
# ReAct 提示词模板
REACT_PROMPT = """
你是一个AI助手，需要回答用户的问题。

你可以通过以下工具来帮助你回答问题：
- search(query): 搜索互联网获取信息
- calculate(expression): 执行数学计算
- lookup(keyword): 在知识库中查找信息

回答问题时，请采用以下格式：

问题：{question}

 Thought: 我需要思考当前情况，决定下一步行动
 Action: search/calculate/lookup
 Action Input: "搜索 query 或计算表达式"
 Observation: 工具返回的结果
 ... ( Thought/Action/Action Input/Observation 循环 )
 Thought: 我现在知道答案了
 Answer: 最终答案

现在开始回答问题：

问题：{user_question}
"""

# 示例：用户问 "Elon Musk 的公司有哪些？"
"""
 Thought: 我需要搜索 Elon Musk 相关的公司信息
 Action: search
 Action Input: "Elon Musk companies founded"
 Observation: Elon Musk 创立或领导的公司包括：Tesla, SpaceX, Neuralink, X Corp, xAI, The Boring Company
 Thought: 搜索结果列出了主要公司，我可以给出完整答案了
 Answer: Elon Musk 的主要公司包括：
 - Tesla（电动汽车）
 - SpaceX（航天探索）
 - Neuralink（脑机接口）
 - X Corp / X（社交媒体）
 - xAI（人工智能）
 - The Boring Company（隧道建设）
"""
```

### 实际代码实现

```python
import re
from typing import List, Dict, Any

class ReActAgent:
    """ReAct 模式的简单实现"""

    def __init__(self, llm, tools):
        self.llm = llm
        self.tools = tools  # 工具字典 {tool_name: callable}
        self.prompt_template = REACT_PROMPT

    def run(self, question: str, max_iterations: int = 5) -> str:
        """
        运行 ReAct 循环
        """
        conversation = f"问题：{question}\n\n"

        for iteration in range(max_iterations):
            # 1. 让 LLM 生成 Thought 和 Action
            response = self.llm.generate(conversation + self.prompt_template)

            # 2. 解析响应中的 Thought, Action, Action Input
            thought = self.extractThought(response)
            action = self.extractAction(response)
            action_input = self.extractActionInput(response)

            # 3. 如果没有 Action，说明推理完成
            if not action:
                answer = self.extractAnswer(response)
                return answer

            # 4. 执行 Action
            observation = self.execute_tool(action, action_input)

            # 5. 将结果添加到对话中
            conversation += f"Thought: {thought}\n"
            conversation += f"Action: {action}\n"
            conversation += f"Action Input: {action_input}\n"
            conversation += f"Observation: {observation}\n\n"

        return "达到最大迭代次数，未能得出答案"

    def execute_tool(self, tool_name: str, tool_input: str) -> str:
        """执行工具并返回结果"""
        if tool_name in self.tools:
            try:
                result = self.tools[tool_name](tool_input)
                return str(result)
            except Exception as e:
                return f"工具执行错误: {str(e)}"
        else:
            return f"未知工具: {tool_name}"

    # 解析方法（简化版）
    def extractThought(self, text):
        match = re.search(r'Thought:(.+?)(?:Action:|Answer:)', text, re.DOTALL)
        return match.group(1).strip() if match else ""

    def extractAction(self, text):
        match = re.search(r'Action:\s*(\w+)', text)
        return match.group(1).strip() if match else ""

    def extractActionInput(self, text):
        match = re.search(r'Action Input:\s*(.+)', text)
        return match.group(1).strip() if match else ""

    def extractAnswer(self, text):
        match = re.search(r'Answer:(.+)', text, re.DOTALL)
        return match.group(1).strip() if match else "未找到答案"
```

## ReAct 的优势与局限

### 优势

**1. 可处理需要外部信息的任务**
纯 CoT 只能在训练知识范围内推理；ReAct 可以调用搜索、API 等获取实时信息。

**2. 推理过程可解释**
每一步都有 Thought，可以追踪 AI 的思考过程。

**3. 灵活性高**
遇到新情况可以改变策略，不像 Plan-and-Execute 那样必须按计划执行。

**4. 错误可发现**
通过 Observation 可以知道工具执行结果，如果出错可以及时调整。

### 局限

**1. 迭代次数不确定**
复杂问题可能需要很多步，但 max_iterations 设多少合适是个问题。

**2. 工具描述质量影响很大**
如果工具描述不清晰，AI 可能调用错误的工具。

**3. 短视风险**
每一步都是基于当前 Observation 做决定，可能陷入局部最优。

**4. 上下文长度限制**
多轮对话后，上下文积累很长，可能超过 context window。

## ReAct 的变体

### 1. ReAct + CoT（混合模式）

在 ReAct 的 Thought 中加入 CoT 推理：

```python
"""
Thought: 我需要比较 Tesla 和 Toyota 的市值。
首先我需要搜索 Tesla 当前的市值，用 search 工具。
然后搜索 Toyota 的市值。
最后用 calculate 工具比较两者。
 Action: search
 Action Input: "Tesla market cap 2024"
"""
```

### 2. Streamable ReAct

流式输出，每生成一个 token 就 yield，不等完整 action 再执行：

```python
async def stream_react(question: str):
    """流式 ReAct 实现"""
    conversation = f"问题：{question}\n\n"
    async for token in self.llm.stream_generate(conversation):
        yield token
        # 检测到完整的 Action 后立即执行
        if detect_complete_action(token_buffer):
            action = parse_action(token_buffer)
            result = await execute_tool(action)
            conversation += f"Observation: {result}\n\n"
            token_buffer = ""
```

### 3. Parallel ReAct

在 Thought 阶段，并行调用多个不相关的工具，加速执行：

```python
"""
Thought: 用户同时问了天气和股票，我需要并行查询这两个信息。
这两个查询互相独立，可以同时进行。
 Action: parallel
 Action Input: [
   {"tool": "weather", "params": {"city": "北京"}},
   {"tool": "stock", "params": {"symbol": "AAPL"}}
 ]
 Observation: [天气查询结果, 股票查询结果]
"""
```

## 实践建议

### 1. 工具描述要清晰

```python
# ❌ 模糊的工具描述
tools = [
    {"name": "search", "description": "搜索信息"},
]

# ✅ 清晰的工具描述（包含参数和返回值格式）
tools = [
    {
        "name": "search",
        "description": "搜索互联网获取信息",
        "parameters": {
            "type": "object",
            "properties": {
                "query": {
                    "type": "string",
                    "description": "搜索关键词，建议用英文或通用术语"
                }
            },
            "required": ["query"]
        },
        "output_format": "返回搜索结果列表，每项包含标题、URL和摘要"
    }
]
```

### 2. Thought 引导要明确

```python
# 在 prompt 中加入 Thought 的引导
THOUGHT_GUIDANCE = """
在决定下一步行动之前，先问自己：
1. 用户的问题是什么？我需要什么信息才能回答？
2. 我目前有什么信息？还缺什么？
3. 哪个工具可以提供我需要的信息？
4. 调用这个工具的最佳参数是什么？
"""
```

### 3. 设置合理的迭代上限

```python
# 根据任务复杂度调整
MAX_ITERATIONS = {
    "simple_lookup": 2,      # 简单查询，1-2步
    "multi_step_search": 5,  # 多步搜索，最多5步
    "complex_reasoning": 10, # 复杂推理，最多10步
}
```

## 本节小结

ReAct 的核心是"想→做→看"的循环：
- **Thought**：AI 分析当前状态，决定下一步行动
- **Action**：调用外部工具
- **Observation**：获取工具返回结果
- 循环直到任务完成

ReAct 的优势是灵活、可解释、能获取外部信息，适合需要搜索、API调用等工具的复杂任务。但也存在迭代次数不确定、短视风险等局限。
