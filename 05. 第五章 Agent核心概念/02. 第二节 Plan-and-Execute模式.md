# 第二节 Plan-and-Execute 模式

## 为什么需要 Plan-and-Execute

ReAct 的"边想边做"模式有一个潜在问题：**每一步都基于当前状态做决定，可能在复杂任务中陷入局部最优**。

比如你要写一篇完整的调研报告：
```
ReAct 方式：
Step 1: 搜索 XX 行业概况
Step 2: 搜索 XX 行业市场规模
Step 3: 搜索 XX 行业主要玩家
Step 4: 搜索 XX 行业趋势
Step 5: 开始写报告... （发现前面搜索的不够全面） → 回头补充搜索

问题：前面的搜索没有全局规划，可能遗漏重要信息，或搜索顺序不合理
```

**Plan-and-Execute** 的思路是：**先全局规划，再按计划执行**。

```
ReAct:      想一步做一步 → 做中调整
Plan-and-Execute: 先想好全部步骤 → 按顺序执行 → 执行中可以修正计划
```

## Plan-and-Execute 的核心架构

```
┌─────────────────────────────────────────────────────────────┐
│                 Plan-and-Execute 架构                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │              Planner（规划器）                        │  │
│   │  输入：用户复杂任务                                  │  │
│   │  输出：执行计划（步骤列表）                          │  │
│   └──────────────────────┬──────────────────────────────┘  │
│                          ↓                                  │
│   ┌─────────────────────────────────────────────────────┐  │
│   │              Executor（执行器）                      │  │
│   │  按顺序执行每个步骤                                  │  │
│   │  收集每个步骤的执行结果                              │  │
│   │  将结果汇总给下一步                                  │  │
│   └──────────────────────┬──────────────────────────────┘  │
│                          ↓                                  │
│   ┌─────────────────────────────────────────────────────┐  │
│   │              Synthesizer（综合器）                   │  │
│   │  将所有步骤的执行结果综合成最终答案                  │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Plan-and-Execute vs ReAct

| 维度 | Plan-and-Execute | ReAct |
|------|------------------|-------|
| **决策时机** | 执行前一次性规划 | 每步动态决策 |
| **全局视角** | 有（先看全貌） | 无（只看眼前） |
| **灵活性** | 低（按计划执行） | 高（随时调整） |
| **适合任务** | 步骤固定、可预见的任务 | 探索性强、可能随时变化的任务 |
| **失败代价** | 计划出错则步步错 | 单步失败影响小 |

## 实现 Plan-and-Execute

### 核心代码

```python
from typing import List, Dict, Any, Callable
import json

class PlanAndExecuteAgent:
    """
    Plan-and-Execute 模式的 Agent
    """

    def __init__(self, llm, tools: Dict[str, Callable], planner_prompt=None):
        self.llm = llm
        self.tools = tools
        self.planner_prompt = planner_prompt or DEFAULT_PLANNER_PROMPT
        self.executor_prompt = DEFAULT_EXECUTOR_PROMPT

    def run(self, task: str) -> str:
        """
        主流程：规划 → 执行 → 综合
        """
        # Step 1: 生成执行计划
        plan = self.create_plan(task)
        print(f"📋 生成计划，共 {len(plan['steps'])} 个步骤")
        for i, step in enumerate(plan['steps'], 1):
            print(f"  {i}. {step}")

        # Step 2: 按顺序执行每个步骤
        step_results = []
        for i, step in enumerate(plan['steps'], 1):
            print(f"\n🔄 执行步骤 {i}: {step}")
            result = self.execute_step(step, step_results)
            step_results.append({
                "step": step,
                "result": result,
                "step_number": i
            })
            print(f"  ✅ 完成: {result[:100]}...")

        # Step 3: 综合结果
        final_answer = self.synthesize(task, step_results)
        return final_answer

    def create_plan(self, task: str) -> Dict[str, Any]:
        """
        生成执行计划
        """
        prompt = f"""
{task}

请将上述任务分解成具体的执行步骤。
每个步骤应该：
1. 有明确的目标
2. 可以通过单个工具或简单操作完成
3. 按逻辑顺序排列

请以 JSON 格式返回：
{{
  "steps": ["步骤1描述", "步骤2描述", ...],
  "reasoning": "制定此计划的理由"
}}
"""
        response = self.llm.generate(prompt)
        # 解析 JSON 响应
        try:
            return json.loads(response)
        except:
            # 如果不是 JSON，尝试从文本中提取
            return self._parse_plan_from_text(response)

    def execute_step(self, step: str, previous_results: List[Dict]) -> str:
        """
        执行单个步骤
        """
        # 构建上下文（包含前面步骤的结果）
        context = self._build_context(step, previous_results)

        prompt = f"""
{self.executor_prompt}

当前任务：{step}

上下文信息（前面步骤的结果）：
{context}

请决定使用哪个工具来完成当前任务，并给出：
1. 使用的工具名称
2. 工具的输入参数
3. 期望的输出格式
"""
        response = self.llm.generate(prompt)
        tool_call = self._parse_tool_call(response)

        if tool_call:
            result = self.tools[tool_call['name']](**tool_call['params'])
            return str(result)
        else:
            return response  # 直接用 LLM 生成的内容作为结果

    def synthesize(self, task: str, step_results: List[Dict]) -> str:
        """
        综合所有结果
        """
        results_text = "\n".join([
            f"步骤 {r['step_number']}（{r['step']}）结果：{r['result']}"
            for r in step_results
        ])

        prompt = f"""
原始任务：{task}

执行结果汇总：
{results_text}

请根据上述执行结果，给出完整的最终答案。
"""
        return self.llm.generate(prompt)

    def _build_context(self, current_step: str, previous_results: List[Dict]) -> str:
        """构建执行上下文"""
        if not previous_results:
            return "无（这是第一个步骤）"

        context = "前面步骤的结果：\n"
        for r in previous_results:
            context += f"- 步骤{r['step_number']}：{r['result']}\n"
        return context

    def _parse_tool_call(self, response: str) -> Dict[str, Any]:
        """从响应中解析工具调用"""
        import re
        tool_match = re.search(r'Tool:\s*(\w+)', response)
        params_match = re.search(r'Params:\s*(.+)', response)

        if tool_match:
            return {
                "name": tool_match.group(1),
                "params": json.loads(params_match.group(1)) if params_match else {}
            }
        return None
```

## Plan-and-Execute 的变体

### 1. 并行执行优化

某些步骤之间没有依赖关系，可以并行执行：

```python
def build_execution_graph(steps: List[Dict]) -> List[List[int]]:
    """
    分析步骤间的依赖关系，返回可并行执行的批次
    """
    # 简化：假设所有步骤按顺序有依赖
    # 实际实现需要 LLM 分析每一步需要前面哪些步骤的结果

    dependencies = {
        0: [],           # 第一步：无依赖
        1: [0],          # 第二步：依赖第一步
        2: [0],          # 第三步：也依赖第一步（可以并行）
        3: [1, 2],       # 第四步：依赖第二和第三步
        4: [3],          # 第五步：依赖第四步
    }

    # 返回批次：[[0], [1,2], [3], [4]]
    # 第二批 [1,2] 可以并行执行
```

### 2. 动态计划调整

Plan-and-Execute 也需要在执行中调整计划：

```python
def execute_with_replan(self, task: str, max_replans: int = 2):
    """
    支持在执行中重新规划的 Plan-and-Execute
    """
    plan = self.create_plan(task)

    for replan_round in range(max_replans + 1):
        try:
            results = self.execute_plan(plan)
            return self.synthesize(task, results)
        except PlanExecutionError as e:
            # 执行失败，尝试重新规划
            print(f"⚠️ 执行失败: {e}，尝试重新规划...")
            plan = self.create_plan(task, context=f"之前的计划执行失败: {e}")
```

## 最佳实践

### 什么时候用 Plan-and-Execute

```
适合 Plan-and-Execute 的场景：
✓ 任务可以预先拆解成明确步骤
✓ 步骤之间有明确的依赖关系
✓ 需要全局视角（先看全貌再行动）
✓ 步骤相对固定，不会频繁变化

不适合的场景：
✗ 探索性任务（不知道需要多少步）
✗ 用户随时可能打断或改变需求
✗ 步骤间的依赖关系不明确
```

### 步骤粒度控制

```python
# 步骤粒度太粗
["写一篇报告"]  # 太粗：无法追踪执行过程

# 步骤粒度太细
["搜索第一段要用的数据", "搜索第二段要用的数据", ...]  # 太细：计划变得繁琐

# 合适的粒度
["搜索市场规模数据", "搜索竞争格局数据", "搜索政策环境",
 "搜索发展趋势", "撰写报告"]  # 合理：每个步骤可独立执行
```

## 本节小结

Plan-and-Execute 的核心是先规划后执行：

- **Planner**：将复杂任务分解成步骤列表
- **Executor**：按顺序执行每个步骤
- **Synthesizer**：汇总所有结果成最终答案

与 ReAct 的对比：
- ReAct：灵活、动态，适合探索性任务
- Plan-and-Execute：全局视角、按部就班，适合可预见的任务

两者各有优势，实际应用中可以根据任务特点选择或结合使用。
