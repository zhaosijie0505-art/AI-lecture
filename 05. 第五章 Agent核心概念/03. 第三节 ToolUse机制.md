# 第三节 Tool Use 机制

## 为什么需要 Tool Use

LLM 的知识有截止日期，无法回答实时问题，也无法执行真实操作。**Tool Use（工具使用）** 让 LLM 能够调用外部工具来扩展其能力边界。

```
LLM 能力扩展：
┌─────────────────────────────────────────────┐
│  LLM 本身的能力                              │
│  · 文本生成                                 │
│  · 推理分析                                 │
│  · 知识问答（截止到训练日期）               │
└─────────────────────────────────────────────┘
                 ↓ 通过 Tool Use 扩展
┌─────────────────────────────────────────────┐
│  工具扩展的能力                              │
│  · 实时搜索（获取最新信息）                 │
│  · API 调用（执行真实操作）                 │
│  · 数据库查询（访问私有数据）               │
│  · 代码执行（计算、编程）                   │
│  · 文件操作（读写文档）                     │
│  · 发送消息（通知、邮件）                   │
└─────────────────────────────────────────────┘
```

## Tool Use 的工作原理

### 核心概念

Tool Use 包含三个关键组件：

```
Tool Use 三组件：
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  1. Tool Definition（工具定义）                              │
│     ━━━━━━━━━━━━━━━━━━━━━━━                                 │
│     向 LLM 描述：                                            │
│     - 工具名称（name）                                      │
│     - 工具功能（description）                                │
│     - 参数格式（parameters schema）                          │
│                                                             │
│  2. Tool Selection（工具选择）                              │
│     ━━━━━━━━━━━━━━━━━━━━━━━                                 │
│     LLM 根据用户问题决定：                                   │
│     - 是否需要调用工具                                       │
│     - 调用哪个工具                                           │
│     - 传入什么参数                                           │
│                                                             │
│  3. Tool Execution（工具执行）                               │
│     ━━━━━━━━━━━━━━━━━━━━━━━                                 │
│     执行工具并返回结果：                                     │
│     - 调用外部系统（搜索/API/数据库）                        │
│     - 将结果格式化为文本                                     │
│     - 返回给 LLM 继续处理                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 工具描述（Tool Description）设计

工具描述是 Tool Use 效果的关键。描述不好，LLM 就不知道什么时候该用什么工具。

### 设计原则

```python
# 好的工具描述示例

# ❌ 模糊描述
{
    "name": "search",
    "description": "Search for information"
}

# ✅ 清晰描述
{
    "name": "search",
    "description": """
搜索互联网获取实时信息。

适用场景：
- 查询当前新闻、事件
- 查询实时价格、统计数据
- 查询天气、分数等实时信息

不适用场景：
- 专业知识问题（用 lookup）
- 计算问题（用 python）

返回格式：
[
  {
    "title": "结果标题",
    "url": "来源链接",
    "snippet": "内容摘要"
  },
  ...
]
""",
    "parameters": {
        "type": "object",
        "properties": {
            "query": {
                "type": "string",
                "description": "搜索关键词。尽量使用简洁的关键词，避免完整句子。例：'北京天气' 而非 '请帮我查一下北京今天的天气怎么样'"
            },
            "max_results": {
                "type": "integer",
                "description": "最多返回的结果数（默认5条）"
            }
        },
        "required": ["query"]
    }
}
```

## 工具调用的实现

### 基于 OpenAI API 的完整实现

```python
from openai import OpenAI
import json

client = OpenAI()

class ToolAgent:
    def __init__(self):
        self.tools = self._define_tools()
        self.available_tools = {
            "search": self._do_search,
            "calculate": self._do_calculate,
            "python": self._do_python,
        }

    def _define_tools(self):
        return [
            {
                "type": "function",
                "function": {
                    "name": "search",
                    "description": "搜索互联网获取实时信息",
                    "parameters": {
                        "type": "object",
                        "properties": {
                            "query": {"type": "string", "description": "搜索关键词"}
                        },
                        "required": ["query"]
                    }
                }
            },
            # ... 其他工具定义
        ]

    def chat(self, user_message: str, max_turns: int = 10) -> str:
        """主对话循环"""
        messages = [{"role": "user", "content": user_message}]

        for turn in range(max_turns):
            # 发送请求，包含工具定义
            response = client.chat.completions.create(
                model="gpt-4o",
                messages=messages,
                tools=self.tools,
                tool_choice="auto"  # 让 LLM 决定是否调用工具
            )

            assistant_msg = response.choices[0].message
            messages.append(assistant_msg)

            # 如果没有 tool_calls，说明 LLM 直接回答了
            if not assistant_msg.tool_calls:
                return assistant_msg.content

            # 执行工具调用
            for tool_call in assistant_msg.tool_calls:
                tool_name = tool_call.function.name
                tool_args = json.loads(tool_call.function.arguments)

                print(f"🔧 调用工具: {tool_name}({tool_args})")

                # 执行工具
                if tool_name in self.available_tools:
                    tool_result = self.available_tools[tool_name](**tool_args)
                else:
                    tool_result = f"错误：未知工具 {tool_name}"

                # 添加工具结果到消息
                messages.append({
                    "role": "tool",
                    "tool_call_id": tool_call.id,
                    "content": str(tool_result)
                })

        return "达到最大轮次限制"

    # 工具实现
    def _do_search(self, query: str) -> str:
        # 调用搜索 API
        results = search_api.query(query, max_results=5)
        return json.dumps(results)

    def _do_calculate(self, expression: str) -> str:
        result = eval(expression)
        return str(result)

    def _do_python(self, code: str) -> str:
        # 使用 subprocess 执行更安全
        import subprocess
        result = subprocess.run(
            ["python", "-c", code],
            capture_output=True,
            text=True,
            timeout=10
        )
        return result.stdout or result.stderr
```

## Tool Use 的安全考虑

### 1. 工具调用权限控制

```python
class SecureToolAgent:
    """
    带权限控制的 Tool Agent
    """

    def __init__(self, user_role: str):
        self.user_role = user_role
        # 定义角色权限
        self.role_permissions = {
            "admin": ["search", "calculate", "python", "read_file", "write_file", "delete_file"],
            "user": ["search", "calculate", "read_file"],
            "guest": ["search"]
        }

    def execute_tool(self, tool_name: str, **kwargs) -> str:
        """执行工具，带权限检查"""
        allowed_tools = self.role_permissions.get(self.user_role, [])

        if tool_name not in allowed_tools:
            return f"错误：用户角色 '{self.user_role}' 无权调用工具 '{tool_name}'"

        # 检查具体参数的约束
        if tool_name == "read_file":
            return self._secure_read_file(kwargs["path"])
        elif tool_name == "write_file":
            return self._secure_write_file(kwargs["path"], kwargs["content"])

        return self._execute_tool(tool_name, **kwargs)

    def _secure_read_file(self, path: str) -> str:
        """限制只能读取特定目录"""
        import os
        allowed_dirs = ["/data/projects", "/tmp/uploads"]

        abs_path = os.path.abspath(path)
        if not any(abs_path.startswith(d) for d in allowed_dirs):
            return f"错误：禁止访问路径 {path}"

        if not os.path.exists(path):
            return f"错误：文件不存在"

        with open(path) as f:
            return f.read()[:10000]  # 限制返回大小
```

## 本节小结

Tool Use 让 LLM 能够调用外部工具来扩展能力：

- **搜索工具**：获取实时信息
- **计算工具**：执行数学计算和代码
- **API 工具**：调用外部系统
- **数据库工具**：查询私有数据
- **文件工具**：读写文件

Tool Use 的三个关键组件：
1. **Tool Definition**：清晰描述工具名称、功能、参数格式
2. **Tool Selection**：LLM 根据问题决定调用哪个工具
3. **Tool Execution**：执行工具并返回结果

工具描述设计要点：清晰的功能说明、适用/不适用场景、返回格式、示例。
