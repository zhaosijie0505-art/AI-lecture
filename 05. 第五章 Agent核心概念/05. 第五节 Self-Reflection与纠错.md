# 第五节 Self-Reflection 与纠错

## 为什么需要 Self-Reflection

没有反思能力的 Agent 就像一个从不复盘的人——同样的错误会重复犯，无法从失败中学习。

```
没有 Self-Reflection 的 Agent：
用户: 翻译 "The quick brown fox"
Agent: 快速的棕色狐狸  ✅ 第一次对
用户: 翻译 "The quick brown fox jumps over"
Agent: 缓慢的棕色狐狸跳跃  ❌ 错了，但 Agent 不会发现

有 Self-Reflection 的 Agent：
Agent: "Jumps over" 意思是"跳跃越过"
反思: "等等，我之前把 'quick' 翻译成'快速的'，现在又翻译成'缓慢的'？
       这是矛盾的，我需要重新检查。"
修正: 应该是"敏捷的棕色狐狸跳跃过"
```

Self-Reflection 让 Agent 能够：
- **发现错误**：在生成答案后主动检查
- **纠正错误**：识别到问题后主动修正
- **学习改进**：从反馈中提取经验教训

## Self-Reflection 的核心机制

```
┌─────────────────────────────────────────────────────────────┐
│                 Self-Reflection 循环                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌──────────────┐                                          │
│   │    Act       │  ← 执行动作，生成响应                    │
│   └──────┬───────┘                                          │
│          ↓                                                  │
│   ┌──────────────┐                                          │
│   │   Observe    │  ← 观察结果                              │
│   └──────┬───────┘                                          │
│          ↓                                                  │
│   ┌──────────────┐                                          │
│   │   Reflect    │  ← 反思：哪里做得好？哪里做得不好？       │
│   └──────┬───────┘                                          │
│          ↓                                                  │
│   ┌──────────────┐                                          │
│   │   Plan       │  ← 计划：下一步该怎么做？                │
│   └──────┬───────┘                                          │
│          ↓                                                  │
│          ├─→ 如果结果满意 → 结束                            │
│          └─→ 如果有问题 → 回到 Act，继续修正                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Self-Reflection 的类型

**1. 结果验证型（Output Verification）**
检查生成的内容是否符合预期：
- 格式是否正确？
- 是否回答了用户的问题？
- 是否有事实错误？

**2. 过程回溯型（Process Tracing）**
回溯推理过程，找出错误环节：
- 哪一步推理出了问题？
- 工具调用的参数是否正确？
- 遗漏了什么信息？

**3. 对比学习型（Comparative Learning）**
从成功和失败的案例中学习：
- 这次为什么成功/失败？
- 和之前的案例有什么异同？
- 下次遇到类似情况该怎么做？

## 实现 Self-Reflection

### 1. 基础版：输出验证

```python
class ReflectiveAgent:
    """
    带自我反思能力的 Agent
    """

    def __init__(self, llm, tools):
        self.llm = llm
        self.tools = tools
        self.max_reflections = 3

    def generate_with_reflection(self, user_message: str) -> str:
        """
        带自我反思的生成流程
        """
        # Step 1: 初始生成
        response = self._generate_initial(user_message)

        # Step 2: 自我反思
        for reflection_round in range(self.max_reflections):
            reflection = self._reflect(user_message, response)

            # Step 3: 检查是否需要修正
            if reflection["needs_correction"]:
                print(f"🔄 第 {reflection_round + 1} 轮反思：发现 {reflection['issues']}")
                response = self._revise(user_message, response, reflection)
            else:
                # 不需要修正，结束
                print("✅ 反思通过，无需修正")
                break

        return response

    def _generate_initial(self, user_message: str) -> str:
        """初始生成"""
        return self.llm.generate(f"用户问题: {user_message}\n\n请给出回答:")

    def _reflect(self, user_message: str, response: str) -> dict:
        """
        自我反思：检查回答是否正确
        """
        reflection_prompt = f"""
请检查以下回答的质量：

用户问题: {user_message}
回答: {response}

请检查：
1. 回答是否直接回应了用户的问题？
2. 回答中是否有事实错误或逻辑漏洞？
3. 回答格式是否合适？
4. 回答是否完整？

请返回 JSON 格式：
{{
  "needs_correction": true/false,
  "issues": ["问题1", "问题2", ...],
  "quality_score": 1-10,
  "strengths": ["优点1", ...],
  "suggestions": ["改进建议1", ...]
}}
"""
        result = self.llm.generate(reflection_prompt)
        return json.loads(result)

    def _revise(self, user_message: str, old_response: str, reflection: dict) -> str:
        """
        根据反思结果修正回答
        """
        revision_prompt = f"""
用户问题: {user_message}
原回答: {old_response}

反思发现的问题：
{chr(10).join(f"- {issue}" for issue in reflection['issues'])}

改进建议：
{chr(10).join(f"- {s}" for s in reflection['suggestions'])}

请根据以上反馈，修正回答。
"""
        return self.llm.generate(revision_prompt)
```

### 2. 进阶版：工具调用反思

```python
class ToolReflectiveAgent:
    """
    带工具调用反思的 Agent
    """

    def __init__(self, llm, tools):
        self.llm = llm
        self.tools = tools
        self.execution_trace = []  # 记录执行过程

    def run_with_reflection(self, task: str) -> str:
        """带反思的工具调用循环"""
        self.execution_trace = []

        for round in range(5):
            # 1. 决定下一步行动
            action = self._decide_action(task)

            # 2. 执行
            if action["type"] == "tool":
                result = self._execute_tool(action["tool"], action["params"])
            else:
                result = self.llm.generate(action["content"])

            # 3. 记录执行
            self.execution_trace.append({
                "round": round,
                "action": action,
                "result": str(result)[:500]
            })

            # 4. 反思：检查当前状态是否满意
            reflection = self._reflect_on_trace(task)
            if reflection["task_complete"]:
                return reflection["final_answer"]

            if reflection["should_retry"]:
                print(f"🔄 反思：需要重试 - {reflection['reason']}")
                continue

        return "达到最大轮次"

    def _decide_action(self, task: str) -> dict:
        """决定下一步行动"""
        trace_text = "\n".join([
            f"Round {t['round']}: {t['action']} → {t['result'][:100]}"
            for t in self.execution_trace
        ])

        prompt = f"""
任务: {task}

执行历史:
{trace_text or "（暂无历史）"}

基于以上信息，决定下一步行动：
- 如果需要工具，使用格式: {{"type": "tool", "tool": "工具名", "params": {{}}}}
- 如果可以直接回答，使用格式: {{"type": "answer", "content": "你的回答"}}
"""
        result = self.llm.generate(prompt)
        return json.loads(result)

    def _reflect_on_trace(self, task: str) -> dict:
        """反思执行历史"""
        trace_text = "\n".join([
            f"Round {t['round']}: {t['action']} → {t['result']}"
            for t in self.execution_trace
        ])

        prompt = f"""
任务: {task}

执行历史:
{trace_text}

请判断：
1. 任务是否已经完成？
2. 最后的结果是否正确、完整？
3. 是否需要继续执行更多步骤？

返回 JSON：
{{
  "task_complete": true/false,
  "final_answer": "最终答案（如果完成）",
  "should_retry": true/false,
  "reason": "如果需要重试，说明原因"
}}
"""
        result = self.llm.generate(prompt)
        return json.loads(result)
```

### 3. Self-Correction 策略

```python
class SelfCorrector:
    """
    错误检测与纠正
    """

    # 常见的错误类型及检测方法
    ERROR_TYPES = {
        "factual_error": {
            "detector": "fact_checker",
            "corrector": "revise_with_facts"
        },
        "format_error": {
            "detector": "format_validator",
            "corrector": "reformat_output"
        },
        "inconsistency": {
            "detector": "consistency_checker",
            "corrector": "resolve_inconsistency"
        },
        "incomplete": {
            "detector": "completeness_checker",
            "corrector": "expand_content"
        }
    }

    def detect_and_correct(self, output: str, context: dict) -> str:
        """检测并纠正错误"""
        errors = []

        # 检测各类错误
        for error_type, handlers in self.ERROR_TYPES.items():
            detector = getattr(self, handlers["detector"])
            error = detector(output, context)
            if error:
                errors.append((error_type, error))

        # 如果没有错误，直接返回
        if not errors:
            return output

        # 纠正错误
        corrected = output
        for error_type, error_detail in errors:
            corrector_name = self.ERROR_TYPES[error_type]["corrector"]
            corrector = getattr(self, corrector_name)
            corrected = corrector(corrected, error_detail)

        return corrected

    def fact_checker(self, output: str, context: dict) -> dict:
        """检测事实错误"""
        # 提取输出中的声明
        claims = self._extract_claims(output)

        # 检查每个声明
        factual_errors = []
        for claim in claims:
            if not self._verify_fact(claim, context.get("knowledge_base")):
                factual_errors.append(claim)

        return {"type": "factual_error", "errors": factual_errors} if factual_errors else None

    def consistency_checker(self, output: str, context: dict) -> dict:
        """检测不一致性"""
        # 检查输出内部的一致性
        inconsistencies = []

        # 前后矛盾检测
        sentences = output.split("。")
        for i in range(len(sentences)):
            for j in range(i + 1, len(sentences)):
                if self._contradicts(sentences[i], sentences[j]):
                    inconsistencies.append(f"{sentences[i]} vs {sentences[j]}")

        return {"type": "inconsistency", "errors": inconsistencies} if inconsistencies else None

    def _contradicts(self, s1: str, s2: str) -> bool:
        """检测两个句子是否矛盾"""
        # 简化版：检测反义词
        contradictions = [
            ("是", "不是"), ("有", "没有"), ("可以", "不能"),
            ("对", "错"), ("真", "假"), ("快", "慢")
        ]

        for pos, neg in contradictions:
            if pos in s1 and neg in s2:
                return True
            if neg in s1 and pos in s2:
                return True

        return False

    def completeness_checker(self, output: str, context: dict) -> dict:
        """检测是否完整"""
        required_elements = context.get("required_elements", [])

        missing = []
        for element in required_elements:
            if element not in output:
                missing.append(element)

        return {"type": "incomplete", "missing": missing} if missing else None
```

## 实践案例

### 案例：代码生成后的自我检查

```python
def generate_code_with_reflection(requirement: str, llm) -> str:
    """
    生成代码后进行自我检查和修正
    """
    # Step 1: 生成代码
    code = llm.generate(f"请生成满足以下需求的代码：\n{requirement}")

    # Step 2: 自我检查
    review_prompt = f"""
请审查以下代码是否满足需求：

需求：{requirement}

代码：
```python
{code}
```

请检查：
1. 代码是否能正确运行？
2. 是否满足所有需求？
3. 是否有潜在的 bug 或边界情况未处理？
4. 代码风格是否一致？

返回 JSON：
{{
  "is_correct": true/false,
  "issues": ["问题1", ...],
  "revised_code": "修正后的代码（如果需要）"
}}
"""
    review = json.loads(llm.generate(review_prompt))

    if not review["is_correct"]:
        print(f"发现 {len(review['issues'])} 个问题，开始修正...")
        return review["revised_code"]

    return code
```

### 案例：RAG 回答的幻觉检测

```python
def rag_with_hallucination_check(query: str, retrieved_contexts: list, llm) -> str:
    """
    RAG 生成后检测幻觉
    """
    # Step 1: 生成初始回答
    answer = llm.generate(f"基于以下上下文回答问题：\n问题：{query}\n上下文：{retrieved_contexts}")

    # Step 2: 幻觉检测
    check_prompt = f"""
请检查以下回答是否忠实于给定的上下文。

上下文：
{retrieved_contexts}

回答：
{answer}

请检查回答中的每个声明：
1. 哪些声明可以直接从上下文中推导出来？
2. 哪些声明是上下文中没有的（可能是幻觉）？
3. 整体 faithfulness 评分（1-5）？

返回 JSON：
{{
  "faithful_claims": ["声明1", ...],
  "unfaithful_claims": ["疑似幻觉的声明1", ...],
  "faithfulness_score": 1-5,
  "revised_answer": "修正后的回答（如果有问题）"
}}
"""
    check = json.loads(llm.generate(check_prompt))

    if check["faithfulness_score"] < 3:
        print(f"⚠️ 检测到幻觉风险：{check['unfaithful_claims']}")
        return check["revised_answer"]

    return answer
```

## 本节小结

Self-Reflection 让 Agent 能够主动发现和纠正错误：

- **结果验证型**：检查输出是否正确
- **过程回溯型**：找出推理过程中的问题
- **对比学习型**：从经验中学习

实现关键：
- 生成 → 反思 → 修正 → 再反思 → ... 直到满意或达到上限
- 记录执行历史（trace），用于反思和调试
- 针对不同错误类型使用不同的检测和纠正策略

Self-Reflection 是构建可靠 Agent 系统的重要一环——让 Agent 不仅仅生成答案，还能验证和改进自己的答案。
