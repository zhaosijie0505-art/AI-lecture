# 第三节 文档分块策略

## 为什么需要文档分块

RAG 的检索单元是"文档块"。如果块太大，里面混入了无关内容，检索精度会下降；如果块太小，上下文不完整，生成效果也会受影响。

```
分块策略的影响：

分块过大（4000+ 字符）：
❌ 问题：包含太多无关内容，检索到的是整个段落
❌ 问题：超出 LLM context 限制，无法注入太多块
✅ 优点：上下文完整

分块过小（100 字符）：
❌ 问题：上下文碎片化，缺少背景信息
❌ 问题：检索命中率高但精确率低
✅ 优点：检索精度高（针对具体问题）

合适的分块（500-1000 字符）：
✅ 上下文相对完整
✅ 检索精度和召回率平衡
✅ 适合 LLM 处理
```

## 分块策略详解

### 1. 固定大小分块（Fixed Size Chunking）

最简单的方法，按字符数或 Token 数切分：

```python
from langchain.text_splitter import CharacterTextSplitter

# 按字符数分块
splitter = CharacterTextSplitter(
    chunk_size=1000,       # 每块 1000 字符
    chunk_overlap=200,     # 相邻块重叠 200 字符
    separator="\n"         # 尽量在换行处分割
)

chunks = splitter.split_text(long_document)
```

**优点**：简单、可控
**缺点**：可能切断句子、段落

### 2. 递归字符分块（Recursive Character Splitting）

按优先级尝试不同的分隔符：

```python
from langchain.text_splitter import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,
    chunk_overlap=200,
    # 按优先级尝试分割符
    separators=["\n\n", "\n", "。", "！", "？", " ", ""]
)

chunks = splitter.split_documents(documents)
```

**分割符优先级**：
1. `\n\n` — 段落（优先保持段落完整）
2. `\n` — 换行
3. `。！？` — 句子结束标点
4. ` ` — 单词边界

### 3. 语义分块（Semantic Chunking）

按语义相似度自动决定分块位置：

```python
def semantic_chunking(text: str, threshold: float = 0.7) -> list:
    """
    语义分块：将语义相似的内容聚合在一起
    """
    sentences = split_into_sentences(text)  # 拆成句子
    embeddings = [embed(sentence) for sentence in sentences]

    chunks = []
    current_chunk = [sentences[0]]

    for i in range(1, len(sentences)):
        # 计算与上一个句子的相似度
        similarity = cosine_similarity(embeddings[i], embeddings[i-1])

        if similarity > threshold:
            # 语义相似，加入当前块
            current_chunk.append(sentences[i])
        else:
            # 语义不相似，开始新块
            chunks.append("".join(current_chunk))
            current_chunk = [sentences[i]]

    # 最后一个块
    if current_chunk:
        chunks.append("".join(current_chunk))

    return chunks
```

### 4. 按文档结构分块

利用 Markdown、HTML 的标题层级结构：

```python
from langchain.document_loaders import NotionLoader

def split_by_headings(markdown_text: str) -> list:
    """
    按 Markdown 标题分块
    """
    import re

    # 匹配所有标题
    heading_pattern = r'^(#{1,6})\s+(.+)$'
    lines = markdown_text.split('\n')

    chunks = []
    current_content = []
    current_level = 0

    for line in lines:
        match = re.match(heading_pattern, line)
        if match:
            # 遇到新标题，保存之前的内容
            if current_content:
                chunks.append('\n'.join(current_content))
                current_content = []

            current_level = len(match.group(1))
            current_content.append(line)
        else:
            current_content.append(line)

    # 最后一个块
    if current_content:
        chunks.append('\n'.join(current_content))

    return chunks
```

## Overlap 的作用

相邻块之间保留重叠区域，确保跨块信息不丢失：

```
文档内容：
[段落1：机器学习的定义] [段落2：机器学习的类型] [段落3：机器学习的应用]

重叠设计（overlap=200字符）：
块1：[段落1：机器学习的定义.............]
块2：[...................段落2：机器学习的类型.........]
块3：[...................段落3：机器学习的应用]

优点：
- 问"机器学习有哪些应用"时，块2包含了足够的上下文
- 块边界不会切断重要信息
```

```python
# Overlap 设置建议
config = {
    # 块大小
    "chunk_size": 1000,      # 字符数或 Token 数

    # 重叠大小（通常为 chunk_size 的 10-20%）
    "chunk_overlap": 200,    # 保留 20% 重叠

    # 重叠太小：关键信息可能被切断
    # 重叠太大：大量重复存储，检索效率降低
}
```

## 不同内容类型的分块策略

### 1. 结构性文档（Markdown/Notion/Confluence）

```python
# 策略：保留标题层级结构
def split_structured_doc(content: str, min_chunk_size: int = 300) -> list:
    """按标题结构分块，保持层级关系"""

    # 用正则提取所有标题及其层级
    sections = re.split(r'(?=^#{1,6}\s+)', content, flags=re.MULTILINE)

    chunks = []
    current_chunk = []

    for section in sections:
        if not section.strip():
            continue

        # 检查当前 section 加入后是否会超出大小限制
        potential_size = sum(len(c) for c in current_chunk) + len(section)

        if potential_size > max_chunk_size and current_chunk:
            # 保存当前 chunk，开始新的
            chunks.append('\n'.join(current_chunk))
            # 保留最后一个 section 作为新 chunk 的开头（overlap）
            current_chunk = [current_chunk[-1], section]
        else:
            current_chunk.append(section)

    # 最后一块
    if current_chunk:
        chunks.append('\n'.join(current_chunk))

    return [c for c in chunks if len(c) > min_chunk_size]
```

### 2. 代码文档

```python
# 策略：按函数/类分块，保持代码结构完整
def split_code(code: str, language: str = "python") -> list:
    """按函数/类分块代码"""

    import re

    if language == "python":
        # 匹配函数和类定义
        pattern = r'(^(?:def|class)\s+\w+.*?(?=\n(?:def|class)|$))'
    elif language == "javascript":
        pattern = r'(^(?:function|const|let|class)\s+\w+.*?(?=\n(?:function|const|let|class)|$))'
    else:
        # 通用：按行数分块
        return [code[i:i+chunk_size] for i in range(0, len(code), chunk_size)]

    matches = re.findall(pattern, code, re.MULTILINE | re.DOTALL)
    return [m.strip() for m in matches if len(m.strip()) > 100]
```

### 3. 表格数据

```python
# 策略：表格整体作为一块，或按行拆分
def split_table(table_content: str, max_rows: int = 10) -> list:
    """按行分块表格，同时保留表头"""

    lines = table_content.strip().split('\n')
    header = lines[0]  # 表头
    rows = lines[1:]   # 数据行

    chunks = []
    for i in range(0, len(rows), max_rows):
        chunk_rows = rows[i:i+max_rows]
        # 每块都包含表头
        chunk = header + '\n' + '\n'.join(chunk_rows)
        chunks.append(chunk)

    return chunks
```

## 分块大小选择指南

### 经验法则

```
选择分块大小的经验：

短内容（100-300 Token）：FAQ、简短回答、单一问题
  → 适合精确匹配

中等内容（300-1000 Token）：文章段落、产品描述
  → 适合大多数 RAG 场景（推荐）

长内容（1000-2000 Token）：长篇分析、完整文档
  → 上下文丰富但可能稀释相关性

超长内容（2000+ Token）：完整报告、书籍章节
  → 需要配合 parent-document retrieval
```

### 按 Token 而非字符估算

```python
import tiktoken

def count_tokens(text: str, model: str = "gpt-4") -> int:
    """计算文本的 Token 数"""
    encoding = tiktoken.encoding_for_model(model)
    return len(encoding.encode(text))

# 经验规则：1 Token ≈ 4 字符（英文）/ 2 字符（中文）
# GPT-4 context = 128K tokens ≈ 50万字符（英文）/ 25万字符（中文）

def suggest_chunk_size(content_type: str) -> dict:
    """根据内容类型建议分块参数"""
    recommendations = {
        "faq": {"chunk_size": 300, "overlap": 50},
        "article": {"chunk_size": 1000, "overlap": 200},
        "report": {"chunk_size": 1500, "overlap": 300},
        "code": {"chunk_size": 800, "overlap": 100},
        "manual": {"chunk_size": 1500, "overlap": 300},
    }
    return recommendations.get(content_type, {"chunk_size": 1000, "overlap": 200})
```

## 高级分块策略

### Parent Document Retrieval

保留两级分块：小子块用于检索，大父块用于生成：

```
检索层：小块（300字符），精确匹配
生成层：大块（1500字符），完整上下文

查询流程：
1. 用小块检索，找到相关小块
2. 找到小块所属的大块
3. 用大块作为上下文生成答案
```

```python
class ParentDocumentRetriever:
    """父文档检索器"""

    def __init__(self, vector_store, parent_chunk_size=1500, child_chunk_size=300):
        self.vector_store = vector_store
        self.parent_size = parent_chunk_size
        self.child_size = child_chunk_size

    def index_documents(self, documents: list):
        """索引文档，建立父子关系"""

        # 1. 创建父块（大块）
        parent_splitter = RecursiveCharacterTextSplitter(
            chunk_size=self.parent_size
        )
        parent_docs = parent_splitter.split_documents(documents)

        # 2. 创建子块（小块）
        child_splitter = RecursiveCharacterTextSplitter(
            chunk_size=self.child_size
        )
        child_docs = child_splitter.split_documents(documents)

        # 3. 建立父子映射
        self.parent_map = {}
        for i, parent in enumerate(parent_docs):
            parent_id = f"parent_{i}"
            self.parent_map[parent_id] = parent

        # 4. 只索引子块
        self.vector_store.add_documents(child_docs)

    def retrieve(self, query: str, k: int = 5) -> list:
        """检索相关父文档"""
        # 1. 检索子文档
        child_results = self.vector_store.similarity_search(query, k=k)

        # 2. 获取父文档
        parent_docs = []
        for child in child_results:
            parent_id = child.metadata.get("parent_id")
            if parent_id and parent_id in self.parent_map:
                parent_docs.append(self.parent_map[parent_id])

        # 去重
        return list({doc.page_content: doc for doc in parent_docs}.values())
```

## 本节小结

文档分块是 RAG 效果的关键环节。

**分块策略**：固定大小（简单）、递归字符（常用）、语义分块（精准）、按结构分块（保持完整性）。

**Overlap**：建议为 chunk_size 的 10-20%，防止边界切断重要信息。

**分块大小**：FAQ 用 300 token，文章用 1000 token，长文档用 1500-2000 token。

**特殊内容**：代码按函数分块，表格保留表头，PDF 善用结构。

**高级策略**：Parent Document Retrieval 用小块检索、大块生成，兼顾精确性和完整性。
