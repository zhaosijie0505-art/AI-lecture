# 第七章：Agent 开发框架

![cover](image/agent-frameworks-cover.png)

**框架，让开发 AI 应用像搭积木一样简单。**

有了 Agent 的核心概念（ReAct、Tool Use、Memory 等），我们可以手写一个简单的 Agent。但如果要构建生产级别的 AI 应用，手写所有组件的效率太低了——你需要考虑错误处理、重试机制、Streaming、Callback、持久化等大量工程问题。

这就是 Agent 开发框架的价值：**把常见的 Agent 组件封装成可复用的模块，让开发者专注业务逻辑，而不是基础设施**。

## 主流框架对比

| 框架 | 定位 | 优势 | 劣势 | 适合场景 |
|------|------|------|------|---------|
| **LangChain** | 全功能框架 | 功能全面、生态丰富 | 复杂、学习曲线陡 | 复杂 RAG、Agent |
| **LlamaIndex** | 数据优先 | RAG 专用、数据连接强 | Agent 能力弱 | RAG 场景 |
| **AutoGen** | 多 Agent | Agent 协作、对话式 | 定制性差 | 多 Agent 协作 |
| **CrewAI** | 多 Agent | 角色分工清晰 | 新兴生态弱 | 多角色 Agent |
| **Semantic Kernel** | 企业微软系 | 与 Azure 集成好 | 微软生态绑定 | 企业微软用户 |
| **LangGraph** | 图结构 | 状态管理强 | 上手复杂 | 复杂工作流 |
