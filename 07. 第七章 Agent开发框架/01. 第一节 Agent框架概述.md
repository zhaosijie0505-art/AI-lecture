# 第一节 Agent 框架概述

## 为什么需要 Agent 框架

手写一个简单的 Agent 只需要几十行代码。但要把 Agent 应用到生产环境，你需要考虑：

```
手写 Agent 缺失的能力：

❌ 错误处理：工具超时、API 限流、模型返回格式错误
❌ 重试机制：失败后自动重试
❌ Streaming：生成内容实时流式返回
❌ Callback：每个步骤完成后执行回调（如记录日志）
❌ 持久化：对话历史、Agent 状态的持久化
❌ 多模态：图像、音频输入输出
❌ 监控：Token 用量、延迟、错误率监控
❌ 可观测性：Trace、Span 追踪

→ 这些问题框架都帮你解决了
```

框架把 AI 应用开发中的**通用能力**封装成可复用的组件，让开发者专注**业务逻辑**。

## 框架的核心概念

### 1. Chain（链）

Chain 是把多个组件串联起来的机制：

```python
# Chain 示例：LLM + Prompt + Output Parser
from langchain_openai import ChatOpenAI
from langchain.prompts import PromptTemplate
from langchain.output_parsers import StrOutputParser

# 定义组件
llm = ChatOpenAI(model="gpt-4o")
prompt = PromptTemplate.from_template("用{language}实现{function}")
parser = StrOutputParser()

# 组合成 Chain
chain = prompt | llm | parser

# 调用
result = chain.invoke({
    "language": "Python",
    "function": "快速排序"
})
```

### 2. Agent（智能体）

Agent 是能够自主决策并执行动作的实体：

```python
# Agent 示例：带工具调用能力的 Agent
from langchain.agents import AgentExecutor, create_openai_functions_agent
from langchain.tools import Tool

# 定义工具
tools = [
    Tool(
        name="search",
        func=search_function,
        description="搜索互联网获取信息"
    ),
    Tool(
        name="calculator",
        func=calculate_function,
        description="执行数学计算"
    )
]

# 创建 Agent
agent = create_openai_functions_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools)

# 运行
result = agent_executor.invoke({"input": "帮我查一下今天北京的天气，然后计算华氏度"})
```

### 3. Tool（工具）

Tool 是 Agent 可以调用的外部能力：

```python
# Tool 定义方式
from langchain.tools import tool

# 方式1：使用 @tool 装饰器
@tool
def get_weather(city: str) -> str:
    """获取城市天气"""
    return f"{city}今天晴天，25度"

# 方式2：使用 Tool 类
from langchain.tools import Tool
get_weather_tool = Tool(
    name="weather",
    func=get_weather,
    description="""获取指定城市的当前天气。
    输入：城市名称（字符串）
    输出：天气描述和温度"""
)

# 方式3：StructuredTool（支持复杂参数）
from langchain.tools import StructuredTool
def create_order(item: str, quantity: int, price: float) -> dict:
    """创建订单"""
    return {"order_id": "12345", "item": item, "quantity": quantity}

order_tool = StructuredTool.from_function(
    func=create_order,
    name="create_order",
    description="创建新订单",
    args_schema=OrderInputSchema
)
```

### 4. Callback（回调）

Callback 用于在 Chain/Agent 执行过程中插入自定义逻辑：

```python
from langchain.callbacks import CallbackManager
from langchain.callbacks.base import BaseCallbackHandler
from langchain.schema import AgentAction, AgentFinish

class MyCallbackHandler(BaseCallbackHandler):
    """自定义回调处理器"""

    def on_agent_action(self, action: AgentAction, **kwargs):
        """Agent 执行动作前调用"""
        print(f"🤖 Agent 正在执行: {action.tool}")
        print(f"   参数: {action.tool_input}")

    def on_agent_finish(self, finish: AgentFinish, **kwargs):
        """Agent 完成时调用"""
        print(f"✅ Agent 完成任务")
        print(f"   输出: {finish.return_values['output']}")

    def on_llm_new_token(self, token: str, **kwargs):
        """LLM 生成新 token 时调用（Streaming）"""
        print(token, end="", flush=True)

# 使用 Callback
callback_manager = CallbackManager([MyCallbackHandler()])
agent_executor = AgentExecutor(
    agent=agent,
    tools=tools,
    callback_manager=callback_manager
)
```

## LangChain vs LlamaIndex

这是最常见的疑问：**我该用 LangChain 还是 LlamaIndex？**

```
LangChain：
定位："构建 LLM 应用的框架"
核心能力：Prompt 管理、Chain、Agent、Tool、Memory
优势：功能全面、灵活性高、生态丰富
适合：复杂 Agent、复杂工作流、需要高度定制

LlamaIndex：
定位："构建 RAG 应用的框架"
核心能力：数据连接、索引、检索、Query Engine
优势：RAG 专用、数据处理能力强、简单易用
适合：知识库问答、需要接入多种数据源
```

**实际上**：两者不是互斥的，可以结合使用：

```python
# 典型组合：LlamaIndex 做 RAG + LangChain 做 Agent

# 1. 用 LlamaIndex 构建检索器
from llama_index import VectorStoreIndex, SimpleDirectoryReader

documents = SimpleDirectoryReader("./data").load_data()
index = VectorStoreIndex.from_documents(documents)
retriever = index.as_retriever()

# 2. 用 LangChain 包装成 Tool
from langchain.tools import tool

@tool
def rag_search(query: str) -> str:
    """基于企业知识库搜索相关信息"""
    results = retriever.retrieve(query)
    return "\n".join([r.text for r in results])

# 3. 用 LangChain 构建 Agent
from langchain.agents import create_openai_functions_agent
agent = create_openai_functions_agent(llm, [rag_search], prompt)
```

## 各框架对比

### 1. LangChain

```
优点：
✓ 功能最全面（Prompt、Chain、Agent、Memory、Tool 全覆盖）
✓ 生态最丰富（100+ 的集成）
✓ 灵活性高（可以深度定制）
✓ 社区活跃（文档丰富、示例多）

缺点：
✗ 学习曲线陡（概念多、API 复杂）
✗ 变化快（版本迭代快，API 不稳定）
✗ 调试困难（Chain 执行不透明）

适合：需要深度定制的复杂 Agent 项目
```

### 2. LlamaIndex

```
优点：
✓ RAG 专用，数据连接能力最强
✓ 上手简单，文档清晰
✓ 索引和检索性能好
✓ 与 LangChain 可以无缝集成

缺点：
✗ Agent 能力不如 LangChain
✗ 定制性稍差

适合：知识库问答、数据密集型应用
```

### 3. AutoGen（微软）

```
优点：
✓ 多 Agent 协作能力强
✓ 对话式交互自然
✓ 与微软生态集成好

缺点：
✗ 定制性差（适合场景有限）
✗ 文档质量一般
✗ 新兴框架，稳定性待验证

适合：多 Agent 对话协作场景
```

### 4. CrewAI

```python
# CrewAI 角色分工示例
from crewai import Agent, Task, Crew

# 定义不同角色的 Agent
researcher = Agent(
    role="研究员",
    goal="找到关于{topic}的最新信息",
    backstory="你是一名专业的市场研究员",
    tools=[search_tool]
)

writer = Agent(
    role="内容编辑",
    goal="将研究结果整理成文章",
    backstory="你是一名资深编辑，擅长撰写清晰易懂的文章",
    tools=[]
)

# 创建任务
research_task = Task(
    description="研究{topic}的现状和趋势",
    agent=researcher
)

write_task = Task(
    description="撰写一篇关于{topic}的文章",
    agent=writer
)

# 创建 Crew（Agent 团队）
crew = Crew(agents=[researcher, writer], tasks=[research_task, write_task])
result = crew.kickoff()
```

### 5. Semantic Kernel（微软）

```
优点：
✓ 与 Azure OpenAI 集成最紧密
✓ 企业级稳定性
✓ 支持 C#/Python/Java 多语言
✓ Planning 能力强

缺点：
✗ 微软生态绑定
✗ 学习资料不如 LangChain 多

适合：已使用 Azure 的企业用户
```

## 如何选择框架

### 选择决策树

```
你的场景是什么？
├── RAG + 知识库问答
│   └──→ LlamaIndex（首选）或 LangChain + LlamaIndex
│
├── 单 Agent + 工具调用
│   └──→ LangChain（功能全）
│
├── 多 Agent 协作
│   ├── 简单对话协作 → AutoGen / CrewAI
│   └── 复杂工作流 → LangGraph
│
├── 微软 Azure 生态
│   └──→ Semantic Kernel
│
└── 需要高度定制
    └──→ LangChain + LangGraph
```

### 渐进式学习路径

```
第一阶段：LlamaIndex（上手简单）
→ 学习 RAG 基础，理解索引和检索

第二阶段：LangChain（功能全面）
→ 学习 Chain、Prompt、Tool、Agent

第三阶段：LangGraph（复杂工作流）
→ 学习状态管理、图结构工作流

第四阶段：按需选择
→ AutoGen（多 Agent 协作）
→ Semantic Kernel（Azure 集成）
```

## 本节小结

Agent 框架解决了 AI 应用开发中的通用工程问题：错误处理、重试、Streaming、Callback、持久化等。

**核心概念**：
- **Chain**：串联多个组件
- **Agent**：自主决策执行
- **Tool**：Agent 可调用的外部能力
- **Callback**：执行过程中的钩子

**框架对比**：
- **LangChain**：功能最全面，适合复杂定制
- **LlamaIndex**：RAG 专用，简单易用
- **AutoGen/CrewAI**：多 Agent 协作
- **Semantic Kernel**：Azure 企业用户

**实际上**：各框架不是互斥的，LlamaIndex + LangChain 是常见组合。
