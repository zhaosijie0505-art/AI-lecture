# 第三节 LlamaIndex 深度指南

## LlamaIndex 与 LangChain 的区别

```
LangChain：通用 LLM 应用框架
  → 功能全面：Prompt、Chain、Agent、Tool、Memory
  → 灵活性高，但 RAG 专用功能不够深入

LlamaIndex：数据优先的 RAG 框架
  → 数据连接器丰富（PDF、Word、Notion、SQL 等）
  → 索引类型多样（向量、列表、树形、知识图谱）
  → RAG 专用，简单高效
```

**选择建议**：
- 纯 RAG 场景：LlamaIndex 更简单
- 复杂 Agent + RAG：LlamaIndex + LangChain 结合
- 复杂工作流：LangChain / LangGraph

---

## 核心概念

```
LlamaIndex 核心概念：

Data Connectors（数据连接器）
  ↓ 从各种数据源加载数据
Documents（文档）
  ↓ 分割成节点
Nodes（节点）
  ↓ 建立索引
Index（索引）
  ↓ 查询
Query Engine（查询引擎）
  ↓ 返回响应
Response（响应）
```

## 数据加载

```python
from llama_index import SimpleDirectoryReader

# 加载目录下的所有文件
documents = SimpleDirectoryReader("./data").load_data()

# 只加载特定类型
documents = SimpleDirectoryReader(
    "./data",
    required_exts=[".pdf", ".docx"]  # 只加载 PDF 和 Word
).load_data()

# 加载时指定元数据
documents = SimpleDirectoryReader(
    "./data",
    filename_as_id=True  # 用文件名作为文档 ID
).load_data()

print(f"加载了 {len(documents)} 个文档")
```

### 节点（Node）

```python
from llama_index import Document
from llama_index.node_parser import SimpleNodeParser

# 创建文档
doc = Document(text="长文本内容...")

# 解析成节点
parser = SimpleNodeParser.from_defaults(
    chunk_size=1000,
    chunk_overlap=200
)

nodes = parser.get_nodes_from_documents([doc])

# 每个节点包含：
# - text：节点内容
# - metadata：元数据（继承自文档 + 添加节点级别信息）
# - relationships：与其他节点的关系
```

## 索引类型

### 1. VectorStoreIndex（向量索引）

最常用的索引类型：

```python
from llama_index import VectorStoreIndex

# 从文档创建索引
index = VectorStoreIndex.from_documents(documents)

# 保存索引到磁盘
index.save_to_disk("./index.json")

# 加载索引
from llama_index import load_index_from_storage
index = load_index_from_storage(storage_context=storage_context)
```

### 2. SummaryIndex（列表索引）

按原始顺序存储，适合需要遍历全部文档的场景：

```python
from llama_index import SummaryIndex

index = SummaryIndex.from_documents(documents)
```

### 3. TreeIndex（树形索引）

按层级组织，适合总结和摘要类任务：

```python
from llama_index import TreeIndex

index = TreeIndex.from_documents(documents)
```

## 查询引擎

### 基础查询

```python
# 获取查询引擎
query_engine = index.as_query_engine(
    similarity_top_k=3,     # 返回 top-3 相关文档
    response_mode="compact" # compact: 压缩上下文后生成
)

# 查询
response = query_engine.query("你们公司的年假政策是什么？")
print(response.response)  # 回答内容
print(response.source_nodes)  # 使用的文档节点
```

### 查询模式

```python
# 1. 检索模式（只检索，不生成）
retriever = index.as_retriever(
    similarity_top_k=5,
    search_mode="default"  # 或 "mmr"（最大边际相关）
)

nodes = retriever.retrieve("查询内容")

# 2. 生成模式（检索 + 生成）
query_engine = index.as_query_engine(
    response_mode="compact"  # compact/simple/refine/tree_summarize
)

# response_mode 选项：
# - compact：压缩上下文后一次性生成
# - simple：简单直接生成
# - refine：先生成草稿，再用后续上下文精炼
# - tree_summarize：用树形结构总结
```

### Retriever 进阶

```python
# 自动检索（Auto-Retrieval）
# 根据查询内容，LLM 自动决定检索什么

from llama_index import VectorStoreIndex
from llama_index.retrievers import BM25Retriever, QueryFusionRetriever

# 1. 向量检索
vector_retriever = index.as_retriever(similarity_top_k=5)

# 2. BM25 关键词检索
bm25_retriever = BM25Retriever.from_defaults(
    index=index,
    similarity_top_k=5
)

# 3. 混合检索（融合向量和 BM25）
fusion_retriever = QueryFusionRetriever(
    retrievers=[vector_retriever, bm25_retriever],
    mode="reciprocal_rerank",  # 或 "dist_based_score"
    top_k=5
)

# 使用融合检索器
query_engine = index.as_query_engine(retriever=fusion_retriever)
```

## 自定义 QueryEngine

```python
from llama_index import QueryEngine, Response
from llama_index.retrievers import BaseRetriever

class CustomQueryEngine(QueryEngine):
    """自定义查询引擎"""

    def __init__(self, retriever: BaseRetriever, llm):
        self.retriever = retriever
        self.llm = llm

    def query(self, query_str: str) -> Response:
        # 1. 检索相关节点
        nodes = self.retriever.retrieve(query_str)

        # 2. 构建上下文
        context = "\n\n".join([node.text for node in nodes])

        # 3. 生成回答
        prompt = f"""基于以下资料回答问题。

资料：
{context}

问题：{query_str}

回答："""

        response_text = self.llm.complete(prompt)

        return Response(
            response=response_text,
            source_nodes=nodes
        )

# 使用
query_engine = CustomQueryEngine(
    retriever=index.as_retriever(),
    llm=llm
)
```

## 与 LangChain 集成

```python
from llama_index import VectorStoreIndex
from langchain.tools import tool

# 1. 用 LlamaIndex 构建索引
index = VectorStoreIndex.from_documents(documents)
retriever = index.as_retriever()

# 2. 用 LlamaIndex 的检索结果
@tool
def rag_search(query: str) -> str:
    """基于企业知识库搜索相关信息"""
    nodes = retriever.retrieve(query)
    return "\n\n".join([
        f"[来源{i+1}] {node.text}"
        for i, node in enumerate(nodes)
    ])

rag_search.name = "knowledge_base_search"
rag_search.description = """
搜索企业内部知识库中的相关信息。

输入：用户的问题
输出：相关文档片段，每段标注来源
"""

# 3. 用 LangChain Agent 调用
from langchain.agents import create_openai_functions_agent

agent = create_openai_functions_agent(
    llm=llm,
    tools=[rag_search],
    prompt=prompt
)

executor = AgentExecutor(agent=agent, tools=[rag_search])
result = executor.invoke({"input": "如何申请年假？"})
```

## 本节小结

**LlamaIndex 核心概念**：
- **Data Connectors**：从各种数据源加载（Notion/Google/Confluence/SQL）
- **Documents/Nodes**：文档及其分割后的节点
- **Index**：向量索引、列表索引、树形索引
- **Query Engine**：检索 + 生成的查询接口

**索引类型**：
- **VectorStoreIndex**：向量相似度检索（最常用）
- **SummaryIndex**：原始列表
- **TreeIndex**：树形层级组织

**查询模式**：
- compact（压缩上下文）
- refine（逐步精炼）
- tree_summarize（树形总结）

**与 LangChain 集成**：LlamaIndex 做 RAG 检索 + LangChain 做 Agent 决策。
