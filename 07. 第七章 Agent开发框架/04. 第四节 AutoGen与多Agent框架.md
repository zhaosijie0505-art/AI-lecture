# 第四节 AutoGen 与多 Agent 框架

## 为什么需要多 Agent 框架

单 Agent 的局限：

```
单 Agent 的问题：

1. 能力单一
   → 一个 Agent 无法同时擅长写作、编程、数学

2. 角色混淆
   → 让 Agent 既当研究员又当编辑，输出容易混乱

3. 协作困难
   → 复杂任务需要多个专业角色的配合

→ 多 Agent 框架解决这些问题
```

多 Agent 的核心思想：**让不同的 Agent 扮演不同的专业角色，通过协作完成复杂任务**。

---

## AutoGen 概述

AutoGen 是微软开源的多 Agent 框架，核心概念：

```
AutoGen 核心概念：

Agent（智能体）：
  → 可以是 LLM、人类或工具
  → 每个 Agent 有特定的角色和能力

Group Chat（群聊）：
  → 多个 Agent 在同一个对话中协作
  → Agent 之间可以相互通信

Manager（管理器）：
  → 协调 Agent 之间的通信
  → 决定下一个由哪个 Agent 发言
```

### 基础 Agent

```python
import autogen
from autogen import AssistantAgent, UserProxyAgent

# 配置
config_list = [
    {
        "model": "gpt-4o",
        "api_key": "your-api-key"
    }
]

# 创建 Assistant Agent
assistant = AssistantAgent(
    name="assistant",
    llm_config={
        "config_list": config_list,
        "temperature": 0.7,
    },
    system_message="你是一个专业的AI助手。"
)

# 创建 User Proxy Agent（代表人类）
user_proxy = UserProxyAgent(
    name="user_proxy",
    human_input_mode="NEVER",  # NEVER/TERMINATE/CONTINUOUS
    max_consecutive_auto_reply=10,
    code_execution_config={"work_dir": "coding"}
)

# 开始对话
user_proxy.initiate_chat(
    assistant,
    message="帮我用 Python 写一个快速排序算法"
)
```

### Tool Use

```python
import autogen

# 定义工具
@autogen.register_function
def get_weather(city: str) -> str:
    """获取城市天气"""
    return f"{city}今天晴天，25度"

# 创建带工具的 Agent
assistant = AssistantAgent(
    name="assistant",
    llm_config={
        "config_list": config_list,
        "tools": [get_weather]
    }
)

# 使用工具
user_proxy.initiate_chat(
    assistant,
    message="纽约今天天气怎么样？"
)
```

## 多 Agent 协作

### Group Chat（群聊）

```python
import autogen
from autogen import AssistantAgent, UserProxyAgent, GroupChat, GroupChatManager

# 创建多个 Agent
writer = AssistantAgent(
    name="writer",
    llm_config={"config_list": config_list},
    system_message="你是一名专业作家，擅长撰写清晰、有吸引力的文章。"
)

editor = AssistantAgent(
    name="editor",
    llm_config={"config_list": config_list},
    system_message="你是一名资深编辑，负责审核文章质量和准确性。"
)

researcher = AssistantAgent(
    name="researcher",
    llm_config={"config_list": config_list},
    system_message="你是一名研究员，负责查找和验证事实信息。"
)

# 创建群聊
groupchat = GroupChat(
    agents=[writer, editor, researcher],
    messages=[],
    max_round=10
)

# 创建管理器
manager = GroupChatManager(
    groupchat=groupchat,
    llm_config={"config_list": config_list}
)

# 启动群聊
user_proxy.initiate_chat(
    manager,
    message="写一篇关于人工智能发展历程的文章"
)

# 群聊会自动协调：
# 1. 研究员负责查找资料
# 2. 作家负责撰写
# 3. 编辑负责审核
# 4. 多轮迭代直到完成
```

### Sequential Chat（顺序协作）

```python
# 顺序执行：完成一个再下一个
user_proxy = UserProxyAgent(name="user_proxy", human_input_mode="NEVER")

# 第一步：研究员
researcher = AssistantAgent(name="researcher", llm_config={"config_list": config_list})
research_result = user_proxy.initiate_chat(
    researcher,
    message="研究人工智能在医疗领域的应用现状"
)
# → 保存 research_result

# 第二步：作家（基于研究结果）
writer = AssistantAgent(name="writer", llm_config={"config_list": config_list})
article = user_proxy.initiate_chat(
    writer,
    message=f"基于以下研究结果撰写文章：\n{research_result.summary}"
)
# → 保存 article

# 第三步：编辑（基于文章）
editor = AssistantAgent(name="editor", llm_config={"config_list": config_list})
final_article = user_proxy.initiate_chat(
    editor,
    message=f"审核以下文章：\n{article.summary}"
)
```

## CrewAI（另一个多 Agent 框架）

```bash
pip install crewai
```

### 基础用法

```python
from crewai import Agent, Task, Crew

# 创建 Agent（定义角色）
researcher = Agent(
    role="市场研究员",
    goal="收集并分析市场相关信息",
    backstory="""你是一名有10年经验的市场研究员，
    专长于数据分析和市场趋势判断。""",
    tools=[search_tool, calculator]  # 工具
)

writer = Agent(
    role="内容编辑",
    goal="将复杂信息转化为易于理解的文字",
    backstory="""你是一名资深编辑，擅长将技术内容
    转化为通俗易懂的文章。""",
    verbose=True
)

# 创建 Task（定义任务）
research_task = Task(
    description="研究竞品A和竞品B的功能和定价",
    agent=researcher,
    expected_output="竞品对比分析报告"
)

write_task = Task(
    description="基于竞品分析，撰写一篇营销文章",
    agent=writer,
    expected_output="一篇1000字的营销文章"
)

# 创建 Crew（Agent 团队）
crew = Crew(
    agents=[researcher, writer],
    tasks=[research_task, write_task],
    process="sequential",  # sequential（顺序）或 hierarchical（层级）
    verbose=2
)

# 启动
result = crew.kickoff()
print(result)
```

### Hierarchical Process（层级协作）

```python
# 层级模式：有一个 Manager Agent 协调
crew = Crew(
    agents=[researcher, writer, designer, manager],
    tasks=[...],
    process="hierarchical",
    manager_agent=manager  # 指定管理器
)
# Manager 决定：
# 1. 哪些任务分配给哪个 Agent
# 2. Agent 的执行顺序
# 3. 如何合并结果
```

## 多 Agent 协作模式

### 1. 路由器模式

```python
"""
路由器模式：入口 Agent 根据请求类型分发到专业 Agent

用户 → 路由器Agent → 分发到 → 写作Agent/编程Agent/分析Agent
"""
class RouterAgent:
    def route(self, query: str) -> str:
        if any(kw in query for kw in ["写", "文章", "内容"]):
            return "writer"
        elif any(kw in query for kw in ["代码", "编程", "函数"]):
            return "coder"
        elif any(kw in query for kw in ["分析", "数据", "统计"]):
            return "analyst"
        return "general"
```

### 2. 审查者模式

```python
"""
审查者模式：一个 Agent 执行，另一个 Agent 审查

执行Agent → 审查Agent → 通过? → 继续/重试
"""
class ReviewerAgent:
    def review(self, output: str, criteria: list) -> dict:
        # 检查输出是否符合标准
        issues = []
        for criterion in criteria:
            if not self.check(output, criterion):
                issues.append(criterion)

        return {
            "pass": len(issues) == 0,
            "issues": issues
        }
```

### 3. 上下文继承模式

```python
"""
上下文继承：子 Agent 继承父 Agent 的上下文

ManagerAgent
  ├── Agent A（继承完整上下文）
  ├── Agent B（继承完整上下文）
  └── Agent C（继承完整上下文）

→ 所有子 Agent 都知道完整背景，协作更顺畅
```

## 本节小结

**AutoGen 核心概念**：
- **AssistantAgent**：执行具体任务
- **UserProxyAgent**：代表人类用户
- **GroupChat**：多 Agent 群聊协作
- **GroupChatManager**：协调多 Agent 通信

**CrewAI 核心概念**：
- **Agent**：角色定义（role/goal/backstory/tools）
- **Task**：任务定义（description/expected_output）
- **Crew**：Agent 团队（agents/tasks/process）

**多 Agent 协作模式**：
- **Sequential**：顺序执行（一个完成后下一个）
- **Group Chat**：群聊协作（自由讨论）
- **Hierarchical**：层级管理（Manager 协调）
- **Router**：路由器（按类型分发）
