# 第五节 企业级框架选型

## 选型决策框架

选择 Agent 框架不是看哪个"最好"，而是看哪个**最适合你的项目**。

```
选型决策树：

1. 你的主要场景是什么？
   ├── RAG/知识库 → LlamaIndex
   ├── 单 Agent + 工具 → LangChain
   ├── 多 Agent 协作 → AutoGen / CrewAI
   └── 复杂工作流 → LangGraph / Semantic Kernel

2. 你的技术栈是什么？
   ├── Python 生态 → 任意框架
   ├── 微软/Azure 生态 → Semantic Kernel
   └── 多语言需求 → Semantic Kernel（C#/Python/Java）

3. 你的团队熟悉度？
   ├── 团队学习能力强 → LangChain（功能全）
   └── 快速交付 → LlamaIndex / CrewAI

4. 生产环境要求？
   ├── 高稳定性 → Semantic Kernel（微软支持）
   ├── 快速迭代 → LangChain（社区活跃）
   └── 云原生 → Pinecone + LangChain
```

## Semantic Kernel 深度解析

### 为什么企业应该关注 Semantic Kernel

```
Semantic Kernel 的优势：

1. 微软背书，企业级稳定性
   → 有正式的企业支持和服务协议
   → 版本稳定性高，Breaking Changes 少

2. 与 Azure OpenAI 深度集成
   → 一键部署到 Azure
   → 原生支持 Azure AD 认证
   → 与 Azure Cognitive Services 无缝集成

3. 跨语言支持
   → C#（主语言）、Python、Java
   → 可以在不同项目间共享技能（Skill）

4. Planning 能力强
   → 内置 Planner 可以自动分解任务
```

### Semantic Kernel Hello World

```python
from semantic_kernel import Kernel
from semantic_kernel.connectors.ai.open_ai import OpenAIChatCompletion

# 创建 Kernel
kernel = Kernel()

# 添加 LLM 服务
kernel.add_service(
    OpenAIChatCompletion(
        service_id="my-gpt4",
        api_key="your-api-key",
        model_id="gpt-4o"
    )
)

# 定义 Skill（相当于 Tool）
from semantic_kernel import sk_function

class MathSkill:
    @sk_function(description="执行数学计算")
    def calculate(self, expression: str) -> str:
        result = eval(expression)
        return str(result)

# 注册 Skill
math_skill = MathSkill()
kernel.import_skill(math_skill, skill_name="math")

# 创建并执行 Pipeline
from semantic_kernel import PipelinePromptTemplate, PromptTemplateConfig

prompt_template = PromptTemplateConfig.from_completion_template(
    "{{$input}}"
)

pipeline = kernel.add_function(
    function_name="my_pipeline",
    plugin_name="my_plugin",
    prompt_template_config=prompt_template
)

result = kernel.run(pipeline, input_str="2 + 2")
```

## LangGraph（复杂工作流）

当 LangChain 的 Chain 不够用时，用 LangGraph：

```python
from langgraph.graph import StateGraph, END

# 定义状态
class AgentState(TypedDict):
    messages: list
    next_action: str

# 创建图
graph = StateGraph(AgentState)

# 添加节点
graph.add_node("researcher", research_agent)
graph.add_node("writer", write_agent)
graph.add_node("editor", editor_agent)

# 添加边
graph.add_edge("researcher", "writer")
graph.add_edge("writer", "editor")
graph.add_edge("editor", END)

# 设置入口
graph.set_entry_point("researcher")

# 编译
app = graph.compile()

# 运行
result = app.invoke({
    "messages": [{"role": "user", "content": "写一篇AI文章"}]
})
```

## 实践建议

### 1. 从简单开始

```
学习路径建议：

第1周：用 LlamaIndex 搭一个 RAG 问答
  → 理解 Embedding、索引、检索的基础

第2周：用 LangChain 实现带工具调用的 Agent
  → 理解 Tool、Agent、Chain 的协作

第3周：尝试 CrewAI 或 AutoGen 的多 Agent 协作
  → 理解多 Agent 协作模式

第4周：根据项目需求选择框架，正式开发
```

### 2. 不要过度设计

```python
# ❌ 过度设计：为了用框架而用框架
crew = Crew(
    agents=[researcher, writer, editor, reviewer, publisher],
    tasks=[...],
    process="hierarchical"
)

# ✅ 简单够用：单一 Agent 就能完成
agent = create_openai_functions_agent(llm, [search_tool, write_tool], prompt)
```

### 3. 保持灵活性

```python
# ❌ 框架绑定：代码直接依赖 LangChain
chain = LLMChain(llm=llm, prompt=prompt)

# ✅ 框架抽象：定义抽象接口，具体实现可替换
from abc import ABC

class LLMChain(ABC):
    def invoke(self, input: dict) -> str:
        pass

class LangChainLLMChain(LLMChain):
    def __init__(self, llm, prompt):
        self.chain = LLMChain(llm=llm, prompt=prompt)

    def invoke(self, input: dict) -> str:
        return self.chain.run(**input)
```
