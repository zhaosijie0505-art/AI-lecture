# 第八章：RAG 进阶

![cover](image/rag-advanced-cover.png)

**从"能跑"到"跑好"，RAG 高级技巧实战。**

基础 RAG 搭起来简单，但真正在生产环境中，你会发现一堆问题：
- 复杂问题答不好（多跳推理）
- 上下文太长、回答跑偏
- 文档太多找不到相关的内容
- 检索结果质量不稳定

这一章，我们来解决这些问题——从 Query Decomposition 到 Agentic RAG，把 RAG 推进到生产级别。

## 进阶技术概览

```
基础 RAG 的局限：

1. 简单检索
   简单问题能答，复杂问题容易答偏

2. 单次检索
   一次检索不到相关内容就失败了

3. 无状态
   不理解对话历史和上下文

→ 进阶 RAG 的解决方案：

1. Query 改写
   HyDE / Query Expansion / Query Decomposition

2. 链式检索
   Forward/Backward / Chain of Thought Retrieval

3. Agentic RAG
   Agent 自主决定检索策略、多次检索迭代
```
