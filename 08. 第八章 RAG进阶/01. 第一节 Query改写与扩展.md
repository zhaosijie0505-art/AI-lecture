# 第一节 Query 改写与扩展

## 为什么需要 Query 改写

用户输入的问题，往往不是检索的最佳表达：

```
用户问题 vs 检索需求：

用户问题："张三在哪个公司工作？"
→ 直接检索"张三"可能找到多个同名的人

理想检索："张三 + [具体项目/部门/城市]"
→ 但用户没有提供这些限定信息

用户问题："这本书讲了什么？"
→ "这本书"是代词，指代不明

理想检索：[书名] + 主要内容/主题

→ 需要用 LLM 理解用户意图，改写成更好的检索 query
```

## 1. HyDE（Hypothetical Document Embeddings）

### 核心思想

让 LLM 先生成一个"假设答案"，然后用这个答案去找真实文档：

```
传统检索：用户问题 → 直接检索
HyDE：用户问题 → LLM 生成假设答案 → 用假设答案检索

原理：假设答案可能包含正确的关键词和语义结构，
     用它检索比直接检索用户口语化的问题更有效
```

### 实现

```python
from langchain_openai import ChatOpenAI
from langchain.chains import LLMChain
from langchain.prompts import PromptTemplate

llm = ChatOpenAI(model="gpt-4o", temperature=0.7)

hyde_prompt = PromptTemplate.from_template("""
请针对以下用户问题，生成一个简短的回答。
这个回答可以是假设的，不需要完全准确，但应该包含可能的关键词和概念。

用户问题：{question}

假设回答：（用一段话回答这个问题，50-100字）
""")

hyde_chain = LLMChain(llm=llm, prompt=hyde_prompt)

def hyde_retrieve(question: str, vector_store, k: int = 5) -> list:
    """
    HyDE 检索流程
    1. 用 LLM 生成假设回答
    2. 用假设回答的 embedding 检索
    """
    # Step 1: 生成假设回答
    hypothetical_answer = hyde_chain.run(question)

    # Step 2: 用假设回答检索
    results = vector_store.similarity_search(hypothetical_answer, k=k)

    return results

# 示例
# 用户问："特斯拉的年化营收是多少？"
# 假设回答可能是："特斯拉（Tesla）是一家电动汽车制造商，年营收约800亿美元..."
# 用这个答案去检索，可能找到更相关的财务文档
```

### HyDE 变体

```python
# 变体1：生成多个假设答案，取最相关的
def hyde_multi_retrieve(question: str, vector_store, k: int = 5, n_hypotheses: int = 3):
    """生成多个假设答案，选择检索结果最一致的那个"""

    # 生成多个假设答案
    hypothetical_answers = []
    for _ in range(n_hypotheses):
        answer = hyde_chain.run(question)
        hypothetical_answers.append(answer)

    # 检索每个假设答案
    all_results = []
    for hyp_answer in hypothetical_answers:
        results = vector_store.similarity_search(hyp_answer, k=k)
        all_results.extend(results)

    # 去重并按出现频率排序
    seen = set()
    deduped = []
    for result in all_results:
        if result.page_content not in seen:
            seen.add(result.page_content)
            deduped.append(result)

    return deduped[:k]

# 变体2：结合原始问题
def hyde_hybrid_retrieve(question: str, vector_store, k: int = 5):
    """同时用原始问题和假设答案检索"""

    # 原始问题检索
    original_results = vector_store.similarity_search(question, k=k)

    # 假设答案检索
    hypothetical_answer = hyde_chain.run(question)
    hypothetical_results = vector_store.similarity_search(hypothetical_answer, k=k)

    # 合并结果（原始问题权重更高）
    combined = {}
    for doc, score in original_results:
        combined[doc.page_content] = ("original", score * 1.5)

    for doc, score in hypothetical_results:
        if doc.page_content in combined:
            combined[doc.page_content] = ("combined", combined[doc.page_content][1] + score)
        else:
            combined[doc.page_content] = ("hypothetical", score)

    # 排序
    sorted_results = sorted(combined.items(), key=lambda x: x[1][1], reverse=True)
    return [(doc, score) for doc, (_, score) in sorted_results[:k]]
```

## 2. Query Expansion（查询扩展）

### 核心思想

将一个查询扩展成多个等效的表述，从不同角度检索：

```
用户查询："公司年假政策"

扩展后：
1. "公司年假政策"（原始）
2. "年假有多少天"
3. "请假制度规定"
4. "带薪休假天数"

→ 分别检索，合并去重
→ 避免单一查询遗漏重要信息
```

### 实现

```python
from langchain_openai import ChatOpenAI
from langchain.prompts import PromptTemplate

llm = ChatOpenAI(model="gpt-4o", temperature=0.7)

expansion_prompt = PromptTemplate.from_template("""
用户查询：{query}

请生成3-5个等效的查询，从不同角度描述同样的信息需求。

要求：
- 保持原意，但使用不同的表述方式
- 可以增加或减少细节
- 每个查询一行

返回格式：
1. 查询1
2. 查询2
3. 查询3
""")

expansion_chain = LLMChain(llm=llm, prompt=expansion_prompt)

def expand_query(query: str) -> list:
    """扩展查询"""
    result = expansion_chain.run(query)

    # 解析成列表
    queries = [q.strip() for q in result.split('\n') if q.strip()]
    queries.insert(0, query)  # 保留原始查询

    return queries

def multi_query_retrieve(
    query: str,
    vector_store,
    embedding_model,
    k: int = 5,
    alpha: float = 0.5
) -> list:
    """
    多查询检索

    1. 扩展查询为多个子查询
    2. 并行检索
    3. RRF（Rocket Ranking Fusion）合并结果
    """
    # Step 1: 扩展查询
    expanded_queries = expand_query(query)
    print(f"扩展后查询数：{len(expanded_queries)}")
    for q in expanded_queries:
        print(f"  - {q}")

    # Step 2: 并行检索每个查询
    all_results = []
    for q in expanded_queries:
        results = vector_store.similarity_search(q, k=k)
        all_results.append(results)

    # Step 3: RRF 合并
    # RRF(k) = Σ 1/(k + rank(d in query_i))
    # k 通常取 60
    rrf_scores = {}
    rrf_k = 60

    for query_results in all_results:
        for rank, (doc, score) in enumerate(query_results, 1):
            doc_key = doc.page_content
            if doc_key not in rrf_scores:
                rrf_scores[doc_key] = {"doc": doc, "score": 0}
            rrf_scores[doc_key]["score"] += 1 / (rrf_k + rank)

    # 排序
    sorted_results = sorted(rrf_scores.values(), key=lambda x: x["score"], reverse=True)
    return [(item["doc"], item["score"]) for item in sorted_results[:k]]
```

## 3. Query Decomposition（查询分解）

### 核心思想

将复杂问题分解成多个简单问题，分别检索再合并：

```
用户问题："特斯拉的市值和比亚迪比怎么样？"

分解后：
1. "特斯拉市值"
2. "比亚迪市值"

分别检索 → 分别获取答案 → 合并生成对比回答

用户问题："这本书的作者是哪里人？主要讨论了什么主题？"

分解后：
1. "这本书的作者"
2. "这本书的主题"

分别检索 → 合并信息 → 完整回答
```

### 实现

```python
decomposition_prompt = PromptTemplate.from_template("""
用户问题：{question}

如果这个问题涉及多个子问题，请将其分解。

判断：
- 如果问题只有一个焦点，直接返回原问题
- 如果问题有多个方面/问题，分解成多个子问题

返回格式：
- 如果是单一问题：直接返回原问题
- 如果是多部分问题：
  Q1: 第一个子问题
  Q2: 第二个子问题
""")

decompose_chain = LLMChain(llm=llm, prompt=decomposition_prompt)

def decompose_query(question: str) -> list:
    """分解复杂问题"""
    result = decompose_chain.run(question).strip()

    # 检查是否分解
    if result.startswith("Q1:") or result.startswith("1."):
        # 解析分解后的子问题
        queries = []
        for line in result.split('\n'):
            line = line.strip()
            if line.startswith("Q1:") or line.startswith("1."):
                queries.append(line[3:].strip() if line.startswith("Q") else line[2:].strip())
            elif line.startswith("Q2:") or line.startswith("2."):
                queries.append(line[3:].strip() if line.startswith("Q") else line[2:].strip())
            elif line.startswith("Q3:") or line.startswith("3."):
                queries.append(line[3:].strip() if line.startswith("Q") else line[2:].strip())
        return queries if queries else [question]
    else:
        return [question]

def decompose_retrieve(question: str, vector_store, k: int = 5) -> dict:
    """
    分解问题，检索每个子问题，合并结果
    """
    # Step 1: 分解问题
    sub_questions = decompose_query(question)
    print(f"分解为 {len(sub_questions)} 个子问题：{sub_questions}")

    # Step 2: 检索每个子问题
    sub_results = {}
    for sq in sub_questions:
        results = vector_store.similarity_search(sq, k=k)
        sub_results[sq] = results

    # Step 3: 合并结果
    # 策略1：收集所有相关文档，统一去重
    all_docs = {}
    for sq, results in sub_results.items():
        for doc, score in results:
            if doc.page_content not in all_docs:
                all_docs[doc.page_content] = {"doc": doc, "score": score, "sources": [sq]}
            else:
                all_docs[doc.page_content]["sources"].append(sq)

    # 按来源数量和分数排序
    sorted_docs = sorted(
        all_docs.values(),
        key=lambda x: (len(x["sources"]), x["score"]),
        reverse=True
    )

    return {
        "sub_questions": sub_questions,
        "sub_results": sub_results,
        "merged_results": [(d["doc"], d["score"]) for d in sorted_docs]
    }

def decompose_and_answer(question: str, vector_store, llm, k: int = 5) -> str:
    """
    完整流程：分解 → 检索 → 合并 → 回答
    """
    # 分解检索
    result = decompose_retrieve(question, vector_store, k=k)

    # 构建上下文（标注来源）
    context_parts = []
    for i, (doc, score) in enumerate(result["merged_results"][:k], 1):
        # 找出这个文档来自哪个子问题
        sources = result["merged_results"][i-1][0].metadata.get("sources", [])
        context_parts.append(f"[来源{i}] {doc.page_content}")

    context = "\n\n".join(context_parts)

    # 生成回答
    prompt = f"""
基于以下资料回答问题。

问题：{question}

涉及的子问题：{"、".join(result["sub_questions"])}

资料：
{context}

回答要求：
1. 先回答每个子问题
2. 再综合成完整答案
"""
    answer = llm.invoke(prompt)
    return answer.content
```

## 4. Step-Back Prompting（退后一步）

### 核心思想

将问题泛化，先问一个更 general 的问题，找到基础概念，再 narrow down：

```
原始问题："为什么碳酸锂价格下跌会导致电动车成本下降？"

退后一步："碳酸锂在电动车中起什么作用？"
→ 找到背景知识后，再 narrow 到具体问题
```

### 实现

```python
stepback_prompt = PromptTemplate.from_template("""
原问题：{question}

请将这个问题泛化，提出一个更 basic、更 high-level 的问题，
回答这个 high-level 问题有助于回答原问题。

要求：
- 泛化到概念、原理、定义层面
- 50字以内

返回格式：
退后一步的问题：[你的问题]
""")

stepback_chain = LLMChain(llm=llm, prompt=stepback_prompt)

def stepback_retrieve(question: str, vector_store, k: int = 5) -> list:
    """
    Step-Back 检索

    1. 提出泛化问题
    2. 检索泛化问题（找背景知识）
    3. 检索原始问题
    4. 合并结果
    """
    # Step 1: 生成退后一步的问题
    stepback_result = stepback_chain.run(question)
    stepback_question = stepback_result.split("：")[-1].strip()

    print(f"退后一步：{stepback_question}")

    # Step 2: 检索泛化问题
    stepback_results = vector_store.similarity_search(stepback_question, k=k)

    # Step 3: 检索原始问题
    original_results = vector_store.similarity_search(question, k=k)

    # Step 4: 合并（泛化问题结果权重略低）
    all_docs = {}
    for doc, score in stepback_results:
        all_docs[doc.page_content] = {"doc": doc, "score": score * 0.8}

    for doc, score in original_results:
        if doc.page_content in all_docs:
            all_docs[doc.page_content]["score"] += score
        else:
            all_docs[doc.page_content] = {"doc": doc, "score": score}

    sorted_docs = sorted(all_docs.values(), key=lambda x: x["score"], reverse=True)
    return [(item["doc"], item["score"]) for item in sorted_docs[:k]]
```

## 本节小结

**Query 改写的四种方法**：

| 方法 | 思想 | 适用场景 |
|------|------|---------|
| **HyDE** | 用 LLM 生成假设答案，再检索 | 语义模糊、口语化问题 |
| **Query Expansion** | 扩展成多角度查询 | 单一查询容易遗漏的场景 |
| **Query Decomposition** | 分解成多个子问题 | 复杂多部分问题 |
| **Step-Back** | 泛化到概念层面再 narrow | 需要背景知识的复杂因果问题 |

**实战技巧**：
- HyDE：temperature 调高（0.7-1.0）产生多样性假设
- Expansion：3-5 个扩展查询，用 RRF 合并
- Decomposition：先判断是否需要分解，避免过度工程
- Step-Back：适合学术/技术类需要原理背景的问题
