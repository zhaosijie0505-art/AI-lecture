# 第三节 Agentic RAG

## 什么是 Agentic RAG

传统 RAG 是"固定流程"：

```
用户问题 → 一次检索 → 生成回答
```

Agentic RAG 是"智能决策"：

```
用户问题 → Agent 判断：
  ├─ 需要检索？
  │  └─ 用什么检索策略？
  │     ├─ 直接检索
  │     ├─ 分解后检索
  │     └─ 多次迭代检索
  ├─ 已有足够信息？→ 生成回答
  └─ 需要工具？
     └─ 调用计算器/搜索等
```

**Agentic RAG 的核心**：Agent 作为 RAG 的"大脑"，自主决定检索策略、迭代次数、工具调用。

### 1. 定义检索工具

```python
from langchain.tools import tool, StructuredTool
from langchain_openai import ChatOpenAI
from langchain.prompts import ChatPromptTemplate
from langchain.agents import AgentExecutor, create_openai_functions_agent
from langchain.schema import Document

# 向量检索工具
@tool
def vector_search(query: str, top_k: int = 5) -> str:
    """使用向量相似度搜索相关文档

    Args:
        query: 搜索 query
        top_k: 返回结果数量

    Returns:
        相关文档列表
    """
    results = vector_store.similarity_search(query, k=top_k)

    if not results:
        return "未找到相关文档"

    return "\n\n".join([
        f"[文档{i+1}] (相似度: {results[i].metadata.get('score', 'N/A')})\n{results[i].page_content}"
        for i in range(len(results))
    ])

# 关键词检索工具
@tool
def keyword_search(query: str, top_k: int = 5) -> str:
    """使用关键词搜索相关文档（BM25）

    Args:
        query: 搜索 query
        top_k: 返回结果数量

    Returns:
        相关文档列表
    """
    # BM25 实现
    results = bm25_retriever.retrieve(query, k=top_k)

    if not results:
        return "未找到相关文档"

    return "\n\n".join([
        f"[文档{i+1}]\n{results[i].page_content}"
        for i in range(len(results))
    ])

# 混合搜索工具
@tool
def hybrid_search(query: str, top_k: int = 5) -> str:
    """混合搜索：结合向量和关键词搜索

    Args:
        query: 搜索 query
        top_k: 返回结果数量

    Returns:
        相关文档列表，按相关性排序
    """
    results = hybrid_retriever.retrieve(query, k=top_k)

    if not results:
        return "未找到相关文档"

    return "\n\n".join([
        f"[文档{i+1}] (相似度: {score:.2f})\n{doc.page_content}"
        for i, (doc, score) in enumerate(results)
    ])

# 查询分解工具
@tool
def decompose_query(question: str) -> str:
    """将复杂问题分解成多个简单问题

    Args:
        question: 复杂问题

    Returns:
        分解后的子问题列表
    """
    prompt = f"""
将以下问题分解成多个可以独立回答的子问题。

问题：{question}

分解原则：
- 每个子问题应该可以从单个检索中回答
- 子问题之间相互独立

返回格式：
1. 子问题1
2. 子问题2
...
"""

    response = llm.invoke(prompt)
    return response.content

# 总结上下文工具
@tool
def summarize_context(context: str, question: str) -> str:
    """总结检索到的上下文，提取与问题相关的关键信息

    Args:
        context: 检索到的上下文
        question: 用户问题

    Returns:
        总结后的关键信息
    """
    prompt = f"""
基于以下上下文，总结回答这个问题所需的关键信息。

问题：{question}

上下文：
{context}

只返回与问题直接相关的内容，简洁明了。
"""

    response = llm.invoke(prompt)
    return response.content

# 工具列表
tools = [
    vector_search,
    keyword_search,
    hybrid_search,
    decompose_query,
    summarize_context
]
```

### 2. 定义 Agent

```python
# 系统提示词
system_prompt = """你是一个智能 RAG 助手。你的任务是根据用户问题，从知识库中检索相关信息并回答。

你有以下工具可用：
- vector_search: 向量检索，适合语义相似的问题
- keyword_search: 关键词检索，适合有明确关键词的问题
- hybrid_search: 混合检索，结合向量和关键词
- decompose_query: 分解复杂问题为多个子问题
- summarize_context: 总结上下文

决策流程：
1. 分析用户问题，判断需要什么类型的检索
2. 如果问题复杂，先分解
3. 执行检索
4. 评估检索结果是否足够回答问题
5. 如果不够，尝试其他检索策略
6. 生成最终回答

重要原则：
- 只基于检索到的信息回答，不要编造
- 如果找不到相关信息，明确告知
- 在回答中标注信息来源
"""

# 创建 Agent
prompt = ChatPromptTemplate.from_messages([
    ("system", system_prompt),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}")
])

agent = create_openai_functions_agent(llm, tools, prompt)
agent_executor = AgentExecutor(
    agent=agent,
    tools=tools,
    verbose=True,
    max_iterations=5,  # 限制最大迭代次数
    handle_parsing_errors=True
)
```

### 3. 自定义 Agentic RAG 类

```python
from typing import List, Optional, Dict
from dataclasses import dataclass

@dataclass
class RetrievalStep:
    """单次检索步骤"""
    step: int
    action: str
    query: str
    results_count: int
    summary: str

class AgenticRAG:
    """Agent 驱动的 RAG 系统"""

    def __init__(
        self,
        vector_store,
        bm25_retriever=None,
        hybrid_retriever=None,
        llm=None,
        max_iterations: int = 5
    ):
        self.vector_store = vector_store
        self.bm25_retriever = bm25_retriever
        self.hybrid_retriever = hybrid_retriever
        self.llm = llm or ChatOpenAI(model="gpt-4o", temperature=0)
        self.max_iterations = max_iterations
        self.tools = self._create_tools()
        self.retrieval_history: List[RetrievalStep] = []

    def _create_tools(self):
        """创建工具列表"""
        return [
            self._create_vector_search_tool(),
            self._create_keyword_search_tool(),
            self._create_hybrid_search_tool(),
            self._create_decompose_tool(),
            self._create_summarize_tool()
        ]

    def _create_vector_search_tool(self):
        @tool
        def vector_search(query: str, top_k: int = 5) -> str:
            """向量检索"""
            results = self.vector_store.similarity_search(query, k=top_k)
            return self._format_results(results)

        return vector_search

    def _create_keyword_search_tool(self):
        @tool
        def keyword_search(query: str, top_k: int = 5) -> str:
            """BM25 关键词检索"""
            if not self.bm25_retriever:
                return "BM25 检索器未初始化"
            results = self.bm25_retriever.retrieve(query, k=top_k)
            return self._format_results(results)

        return keyword_search

    def _create_hybrid_search_tool(self):
        @tool
        def hybrid_search(query: str, top_k: int = 5) -> str:
            """混合检索"""
            if not self.hybrid_retriever:
                return "混合检索器未初始化"
            results = self.hybrid_retriever.retrieve(query, k=top_k)
            return self._format_results([r[0] for r in results])

        return hybrid_search

    def _create_decompose_tool(self):
        @tool
        def decompose_query(question: str) -> str:
            """问题分解"""
            prompt = f"将问题分解：{question}"
            return self.llm.invoke(prompt).content

        return decompose_query

    def _create_summarize_tool(self):
        @tool
        def summarize_context(context: str, question: str) -> str:
            """总结上下文"""
            prompt = f"基于'{question}'总结：{context}"
            return self.llm.invoke(prompt).content

        return summarize_context

    def _format_results(self, results: List) -> str:
        """格式化检索结果"""
        if not results:
            return "未找到相关结果"

        return "\n\n".join([
            f"[{i+1}] {r.page_content if hasattr(r, 'page_content') else r}"
            for i, r in enumerate(results[:5])
        ])

    def query(self, question: str, return_history: bool = False) -> Dict:
        """
        Agentic RAG 查询

        Returns:
            {"answer": str, "history": [RetrievalStep, ...]}
        """
        self.retrieval_history = []

        # 创建 Agent
        prompt = ChatPromptTemplate.from_messages([
            ("system", self._get_system_prompt()),
            ("human", "{input}"),
            ("placeholder", "{agent_scratchpad}")
        ])

        agent = create_openai_functions_agent(self.llm, self.tools, prompt)
        executor = AgentExecutor(
            agent=agent,
            tools=self.tools,
            verbose=False,
            max_iterations=self.max_iterations
        )

        # 执行
        result = executor.invoke({"input": question})

        if return_history:
            return {
                "answer": result["output"],
                "history": self.retrieval_history
            }
        return {"answer": result["output"]}

    def _get_system_prompt(self) -> str:
        """获取系统提示词"""
        return """你是一个智能 RAG 助手。有多种检索工具可用。

工具说明：
- vector_search: 向量检索（语义相似）
- keyword_search: BM25 关键词检索（精确匹配）
- hybrid_search: 混合检索（结合两者）
- decompose_query: 将复杂问题分解
- summarize_context: 总结上下文

决策指南：
- 问题明确、有关键词 → keyword_search 或 hybrid_search
- 问题语义模糊 → vector_search
- 问题复杂多部分 → 先 decompose_query，再分别检索
- 检索结果太多太杂 → summarize_context 后再检索

流程：
1. 选择合适的检索工具
2. 执行检索
3. 评估结果是否相关
4. 如需要，尝试其他策略
5. 总结答案

注意：
- 只用检索到的内容回答
- 标注来源
- 不确定时明确说明"""
```

## Self-RAG 介绍

Self-RAG 是另一种 Agentic RAG 方案，由 Salesforce 提出：

```
Self-RAG 核心思想：

检索 → 判断是否需要检索（是的标记：[检索]）
     → 生成内容（标记：[相关]、[支持]、[无据]）

生成时决定是否需要检索，生成后评估是否被支持
```

```python
# Self-RAG 的实现思路
class SelfRAG:
    """Self-RAG 简化实现"""

    def __init__(self, retriever, llm):
        self.retriever = retriever
        self.llm = llm

    def generate_with_rag(self, question: str) -> str:
        """
        Self-RAG 流程
        1. 判断是否需要检索
        2. 如果需要，检索相关文档
        3. 生成内容，评估每个声明是否被支持
        """
        # Step 1: 判断是否需要检索
        should_retrieve = self._decide_retrieve(question)

        context = ""
        if should_retrieve:
            # Step 2: 检索
            docs = self.retriever.retrieve(question, k=5)
            context = "\n".join([doc.content for doc in docs])

        # Step 3: 带 [检索] 标记的生成
        response = self._generate_with_markers(question, context)

        # Step 4: 评估生成的内容
        # （实际实现更复杂，这里简化）
        return response

    def _decide_retrieve(self, question: str) -> bool:
        """判断是否需要检索"""
        prompt = f"""
问题：{question}

回答：是否需要从外部知识库检索信息来回答这个问题？
只需要回答"是"或"否"。
"""
        response = self.llm.invoke(prompt)
        return "是" in response.content[:5]

    def _generate_with_markers(self, question: str, context: str) -> str:
        """带标记的生成"""
        prompt = f"""
基于以下信息回答问题。

信息：{context}

问题：{question}

要求：
- 如果使用信息中的内容，在后面加 [检索]
- 如果是自己的推理，加 [推理]

回答：
"""
        return self.llm.invoke(prompt).content
```

## 本节小结

**Agentic RAG 的核心**：用 Agent 作为"大脑"，自主决定：
- 是否需要检索
- 用什么检索策略
- 检索多少次
- 何时停止

**工具集**：
- 向量检索（语义相似）
- 关键词检索（BM25，精确匹配）
- 混合检索（结合两者）
- 问题分解（复杂问题拆解）
- 上下文总结（去冗余）

**关键设计**：
- max_iterations 限制迭代次数，防止无限循环
- 记录检索历史，避免重复检索
- 多种检索策略可选，Agent 自主选择

**Self-RAG**：在生成内容中嵌入 [检索]/[推理] 标记，评估每个声明的可信度。
