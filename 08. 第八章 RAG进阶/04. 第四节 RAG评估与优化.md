# 第四节 RAG 评估与优化

## 为什么 RAG 需要评估

RAG 系统有多个环节，每个环节都可能出问题：

```
RAG Pipeline 各环节的潜在问题：

1. 文档解析
   ❌ PDF 解析乱码、结构丢失
   ❌ Markdown 标题层级错误

2. 分块
   ❌ chunk_size 太大（引入无关噪声）
   ❌ chunk_size 太小（丢失上下文）
   ❌ overlap 设置不当（边界切割）

3. Embedding
   ❌ 中文 embedding 质量差
   ❌ 模型与业务场景不匹配

4. 检索
   ❌ 检索不到相关内容（召回率低）
   ❌ 检索到太多无关内容（精确率低）
   ❌ 排序不合理

5. 生成
   ❌ 幻觉（生成内容与检索结果不符）
   ❌ 回答不完整
   ❌ 上下文丢失（上下文太长被截断）
```

**没有评估就没有优化**——你需要指标来指导调优方向。

## 评估指标体系

### 1. RAGAs（主流 RAG 评估框架）

RAGAs（RAG Assessment）是最常用的 RAG 评估框架：

| 指标 | 衡量什么 | 计算方式 |
|------|---------|---------|
| **Faithfulness** | 生成内容是否忠实于检索内容 | LLM 判断生成中有多少声明被检索支持 |
| **Answer Relevance** | 回答与问题的相关性 | LLM 判断回答对问题的覆盖程度 |
| **Context Precision** | 检索结果的相关性排序 | 相关文档是否排在前面 |
| **Context Recall** | 检索结果是否包含答案 | 检索内容是否涵盖回答所需信息 |
| **Context Relevancy** | 检索内容是否都相关 | 检索内容中有多大比例与问题相关 |

### 3. 传统 IR 指标

除了 LLM 评估，还可以用传统信息检索指标：

```python
from sklearn.metrics import precision_score, recall_score
import numpy as np

def calculate_hit_rate(retriever, questions: list, relevant_docs: list) -> float:
    """
    命中率（Hit Rate）

    衡量：正确的文档在 Top-K 结果中的比例

    适用场景：有明确的"正确答案文档"可对比
    """
    hits = 0
    for question, relevant in zip(questions, relevant_docs):
        results = retriever.retrieve(question, k=5)
        result_contents = [doc.page_content for doc in results]

        # 检查相关文档是否在结果中
        if any(rel in result for rel in relevant):
            hits += 1

    return hits / len(questions)

def calculate_mrr(retriever, questions: list, relevant_docs: list) -> float:
    """
    MRR（Mean Reciprocal Rank）

    衡量：正确文档的平均排名（越靠前越好）
    """
    reciprocal_ranks = []

    for question, relevant in zip(questions, relevant_docs):
        results = retriever.retrieve(question, k=10)
        result_contents = [doc.page_content for doc in results]

        # 找第一个相关文档的排名
        for rank, content in enumerate(result_contents, 1):
            if any(rel in content for rel in relevant):
                reciprocal_ranks.append(1 / rank)
                break
        else:
            reciprocal_ranks.append(0)

    return np.mean(reciprocal_ranks)

def calculate_ndcg(retriever, questions: list, relevance_scores: list) -> float:
    """
    NDCG（Normalized Discounted Cumulative Gain）

    衡量：考虑相关度等级的排序质量
    适用场景：文档相关性有不同等级（完全不相关/部分相关/高度相关）
    """
    ndcg_scores = []

    for question, rel_scores in zip(questions, relevance_scores):
        results = retriever.retrieve(question, k=10)

        # 计算 DCG
        dcg = 0
        for i, (doc, rel) in enumerate(zip(results, rel_scores[:10])):
            dcg += (2 ** rel - 1) / np.log2(i + 2)

        # 计算 IDCG（理想排序）
        ideal_rel = sorted(rel_scores, reverse=True)[:10]
        idcg = sum((2 ** rel - 1) / np.log2(i + 2) for i, rel in enumerate(ideal_rel))

        ndcg = dcg / idcg if idcg > 0 else 0
        ndcg_scores.append(ndcg)

    return np.mean(ndcg_scores)
```

### 3. 上下文太长被截断

**问题表现**：关键信息在上下文中，但 LLM 看不到。

**解决方案**：

```python
# 方案1：上下文压缩
from llama_index.postprocessor import LongContextReorder

# 长上下文重排序：把相关的内容移到前面
postprocessor = LongContextReorder()
reranked_nodes = postprocessor.postprocess_nodes(nodes)

# 方案2：分层检索（Parent Document）
# 大块检索引流，小块内容详细
parent_retriever = ParentDocumentRetriever(
    child_retriever=vector_retriever,
    parent_size=2000,    # 父文档大小
    child_size=500       # 子文档大小
)

# 检索时用子文档匹配，拿到父文档上下文
# 既保证了细粒度匹配，又不会丢失上下文

# 方案3：摘要压缩
def compress_context(docs: list, question: str, llm, max_tokens: int = 4000) -> str:
    """压缩上下文，保留与问题相关的部分"""

    compressed = []
    total_tokens = 0

    for doc in docs:
        extract_prompt = f"""
从以下内容中提取与问题相关的部分。

问题：{question}

内容：
{doc.page_content}

只返回相关部分，不要添加解释。
"""
        relevant = llm.invoke(extract_prompt)

        # 检查 token 限制
        if total_tokens + len(relevant) > max_tokens:
            break

        compressed.append(relevant)
        total_tokens += len(relevant)

    return "\n\n".join(compressed)
```

## 完整评估 Pipeline

```python
class RAGEvaluator:
    """RAG 系统评估器"""

    def __init__(self, rag_pipeline, eval_dataset: list):
        self.rag = rag_pipeline
        self.eval_dataset = eval_dataset

    def evaluate(self) -> dict:
        """
        完整评估
        """
        results = []

        for sample in tqdm(self.eval_dataset):
            question = sample["question"]

            # 执行 RAG
            rag_result = self.rag.query(question)

            # 评估各指标
            evaluation = {
                "question": question,
                "rag_answer": rag_result["answer"],
                "ground_truth": sample["ground_truth_answer"],
                "contexts": rag_result.get("contexts", [])
            }

            # Faithfulness
            evaluation["faithfulness"] = self._evaluate_faithfulness(
                question, rag_result["answer"], rag_result.get("contexts", [])
            )

            # Answer Relevance
            evaluation["answer_relevance"] = self._evaluate_answer_relevance(
                question, rag_result["answer"]
            )

            # Context Precision
            evaluation["context_precision"] = self._evaluate_context_precision(
                sample.get("relevant_docs", []),
                rag_result.get("contexts", [])
            )

            results.append(evaluation)

        # 汇总
        return self._aggregate_results(results)

    def _evaluate_faithfulness(self, question, answer, contexts) -> float:
        """评估忠实度"""
        # 使用 LLM 评估
        prompt = f"""
判断以下回答是否基于给定的参考资料。

参考资料：
{contexts}

回答：
{answer}

只回答 0-1 之间的小数，0 表示完全不忠实，1 表示完全忠实。
"""
        response = self.llm.invoke(prompt)
        try:
            return float(response.content.strip())
        except:
            return 0.5

    def _evaluate_answer_relevance(self, question, answer) -> float:
        """评估回答相关性"""
        prompt = f"""
评估回答与问题的相关性。

问题：{question}
回答：{answer}

只回答 0-1 之间的小数，0 表示完全不相关，1 表示完全相关。
"""
        response = self.llm.invoke(prompt)
        try:
            return float(response.content.strip())
        except:
            return 0.5

    def _evaluate_context_precision(self, relevant_docs, retrieved_docs) -> float:
        """评估上下文精确率"""
        if not relevant_docs or not retrieved_docs:
            return 0.0

        # 计算相关文档在检索结果中的比例
        relevant_set = set(relevant_docs)
        retrieved_set = set(retrieved_docs)

        hits = len(relevant_set & retrieved_set)
        return hits / len(relevant_set)

    def _aggregate_results(self, results: list) -> dict:
        """汇总结果"""
        import numpy as np

        return {
            "avg_faithfulness": np.mean([r["faithfulness"] for r in results]),
            "avg_answer_relevance": np.mean([r["answer_relevance"] for r in results]),
            "avg_context_precision": np.mean([r["context_precision"] for r in results]),
            "total_samples": len(results),
            "detailed_results": results
        }
```

## 本节小结

**评估指标**：
- **RAGAs**：Faithfulness、Answer Relevance、Context Precision、Context Recall
- **传统 IR**：Hit Rate、MRR、NDCG

**常见问题与解决**：

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 检索召回率低 | Embedding 不匹配、chunk 不合理 | 换中文 embedding、增加 overlap、混合检索 |
| 生成幻觉 | Prompt 不够约束 | 强化 Prompt、引入自我验证 |
| 上下文截断 | 上下文太长 | 上下文压缩、Parent Document、摘要 |

**优化闭环**：评估 → 定位问题 → 调整 → 再评估
