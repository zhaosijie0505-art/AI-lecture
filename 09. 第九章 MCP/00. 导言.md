# 第九章：MCP（模型上下文协议）

![cover](image/mcp-cover.png)

**让 AI 连接一切的标准协议。**

你可能遇到过这个问题：想给 AI 配一个自定义工具，需要写一堆胶水代码。不同的 AI 平台、不同的工具，接口都不一样。

**MCP（Model Context Protocol）** 就是来解决这个问题的——一个通用的标准协议，让 AI 模型可以标准化地连接外部工具、数据源和文件系统。无论你用 Claude、GPT 还是国产模型，只要支持 MCP，就能用同一套接口接入各种工具。

## 为什么需要 MCP

```
传统方式：每个 AI 应用都要单独对接

AI 应用 A → 文件系统 API
AI 应用 A → GitHub API
AI 应用 A → Slack API

AI 应用 B → 文件系统 API（又要写一遍）
AI 应用 B → GitHub API（重新写）
AI 应用 B → ...

→ 代码重复，维护成本高
→ 每个 AI 平台都要单独适配
```

```
MCP：统一协议，一次接入，处处运行

          ┌─────────────┐
          │   MCP Host  │  ← Claude Desktop / Cursor / VS Code
          └──────┬──────┘
                 │ MCP 协议（统一接口）
          ┌──────┴──────┐
          │ MCP Client  │
          └──────┬──────┘
                 │
    ┌────────────┼────────────┐
    ↓            ↓            ↓
┌────────┐ ┌────────┐ ┌────────┐
│ 文件系统 │ │ GitHub │ │ Slack  │
│ Server  │ │ Server │ │ Server │
└────────┘ └────────┘ └────────┘

→ 一次开发，到处运行
→ 不同 AI 应用可以复用同一套 MCP 工具
```

---

## MCP vs 传统方案

| 维度 | 传统 API | MCP |
|------|---------|-----|
| **接入方式** | 每个应用独立开发 | 标准化协议，统一接入 |
| **工具复用** | 难以复用 | 一次开发，多次复用 |
| **AI 平台** | 需针对每个平台适配 | 协议通用，天然跨平台 |
| **接口规范** | 各家不同 | 统一的数据格式和调用方式 |
| **安全模型** | 应用自行实现 | 内置安全模型（用户确认、资源隔离） |
