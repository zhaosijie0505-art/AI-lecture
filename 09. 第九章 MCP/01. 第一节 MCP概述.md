# 第一节 MCP 概述

## MCP 是什么

**MCP（Model Context Protocol）** 是 Anthropic 于 2024 年底开源的标准协议，用于连接 AI 模型与外部工具、数据源和系统。

```
MCP 定位：

AI 模型 ←→ MCP ←→ 外部世界
（理解任务）（标准协议）（工具/数据/文件）

类似于：
- USB 的角色：统一接口，连接各种外设
- 浏览器的角色：统一协议，访问各种网站
```

### 官方定义

MCP 是一个开放协议，用于为 AI 助手提供上下文接入能力。它定义了：
- **工具（Tools）**：AI 可以调用的函数
- **资源（Resources）**：AI 可以读取的数据
- **提示（Prompts）**：可复用的 Prompt 模板

---

## 为什么需要 MCP

### 痛点 1：每个 AI 应用都要单独开发工具集成

```
没有 MCP 时：

Claude Desktop → 需要单独开发文件系统集成
Claude API → 需要单独开发
Cursor → 需要单独开发
VS Code Copilot → 需要单独开发

→ 同样的工具，每个平台都要开发一次
→ 开发成本高，维护负担重
```

### 痛点 2：工具接口不统一

```
传统方式：

AI 应用 → 工具 A（API 1）
AI 应用 → 工具 B（API 2）
AI 应用 → 工具 C（API 3）

每个工具的：
- 认证方式不同
- 数据格式不同
- 错误处理不同
- 调用方式不同

→ 开发复杂，容易出错
```

### MCP 的解决方案

```
MCP 统一协议：

                    ┌─────────────┐
                    │   AI 模型   │
                    └──────┬──────┘
                           │ MCP 协议
                    ┌──────┴──────┐
                    │  MCP Host   │  ← Claude Desktop / Cursor
                    └──────┬──────┘
                           │
        ┌──────────────────┼──────────────────┐
        ↓                  ↓                  ↓
  ┌──────────┐       ┌──────────┐       ┌──────────┐
  │  文件    │       │  GitHub  │       │ 数据库    │
  │  系统    │       │  Server  │       │  Server  │
  └──────────┘       └──────────┘       └──────────┘

优势：
1. 一次开发，多平台复用
2. 统一的数据格式和调用方式
3. 内置安全模型
```

## MCP 核心概念

### 1. MCP Host（宿主）

运行 AI 的应用程序：
- Claude Desktop
- Cursor
- VS Code（通过扩展）
- 自定义应用

Host 负责：
- 与用户交互
- 管理 MCP Client
- 处理安全确认

### 2. MCP Client（客户端）

嵌入在 Host 中的客户端，负责：
- 与 MCP Server 建立连接
- 发送请求和接收响应
- 管理会话状态

### 3. MCP Server（服务器）

提供特定能力的独立进程：
- 文件系统 Server
- GitHub Server
- 数据库 Server

每个 Server 提供一组 **Tools**、**Resources**、**Prompts**。

### 4. Tools（工具）

AI 可以调用的函数：

```json
{
  "name": "read_file",
  "description": "读取文件内容",
  "inputSchema": {
    "type": "object",
    "properties": {
      "path": {
        "type": "string",
        "description": "文件路径"
      }
    },
    "required": ["path"]
  }
}
```

### 5. Resources（资源）

AI 可以读取的数据：

```json
{
  "uri": "file:///Users/demo/documents/report.pdf",
  "name": "report.pdf",
  "mimeType": "application/pdf"
}
```

### 6. Prompts（提示模板）

可复用的 Prompt 模板：

```json
{
  "name": "code_review",
  "description": "代码审查模板",
  "arguments": [
    {"name": "language", "description": "编程语言"}
  ],
  "template": "请审查以下 ${language} 代码...\n${code}"
}
```

## MCP 与传统 API 的区别

### API 对比

| 维度 | REST API | MCP |
|------|----------|-----|
| **用途** | 通用的数据传输 | 专门为 AI 工具设计 |
| **接口规范** | 各自定义，不统一 | 协议统一规范 |
| **调用方式** | HTTP 请求 | 通过 MCP 协议 |
| **AI 感知** | AI 不知道接口，需要手动调用 | AI 自动发现和调用工具 |
| **上下文注入** | 手动处理 | 协议内置 |
| **安全模型** | 应用自行实现 | 内置用户确认 |

### MCP 的优势

```
1. AI 原生（AI-native）
   - AI 自动发现可用工具
   - 自动构建调用参数
   - 自动处理响应

2. 标准化
   - 统一的接口定义
   - 统一的数据格式
   - 统一的错误处理

3. 安全
   - 用户确认机制
   - 资源访问控制
   - 沙盒隔离

4. 可复用
   - 一次开发，多平台使用
   - 社区共享 Server
```

## MCP 的使用场景

```
1. 文件系统操作
   → 读取/写入本地文件
   → 搜索文件内容
   → 管理目录

2. GitHub 集成
   → 查看 Issue 和 PR
   → 创建 Issue 和评论
   → 管理代码仓库

3. 数据库查询
   → 执行 SQL 查询
   → 获取数据库结构
   → 数据分析

4. API 集成
   → 调用外部 API
   → 获取第三方数据
   → 集成各种服务

5. 开发工具
   → 代码搜索
   → 代码审查
   → 自动化测试

6. 消息通知
   → 发送 Slack 消息
   → 发送 Email
   → 发送钉钉/飞书消息
```

## 本节小结

**MCP（Model Context Protocol）** 是连接 AI 与外部世界的标准协议。

**核心架构**：
- **MCP Host**：AI 应用（Claude Desktop、Cursor）
- **MCP Client**：嵌入 Host 的客户端
- **MCP Server**：提供能力的独立服务

**三大核心概念**：
- **Tools**：AI 可调用的函数
- **Resources**：AI 可读取的数据
- **Prompts**：可复用的模板

**解决的问题**：
- 工具重复开发
- 接口不统一
- 缺乏标准化
