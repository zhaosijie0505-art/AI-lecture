## 通信流程

### 1. 连接建立

```
Step 1: Host 启动
  → 加载已配置的 MCP Servers
  → 建立 MCP Client 连接

Step 2: Client 连接 Server
  → 通过 stdio / HTTP / WebSocket 建立连接
  → 交换协议版本和支持的能力

Step 3: 能力发现
  Client ← Server
  → list_tools：获取可用工具列表
  → list_resources：获取可用资源列表
  → list_prompts：获取可用 Prompt 模板
```

### 2. 工具调用流程

```
用户：帮我读取 /Users/demo/readme.md

┌──────────────┐      ┌──────────────┐      ┌──────────────┐
│   AI Model   │      │  MCP Client  │      │  MCP Server  │
└──────┬───────┘      └──────┬───────┘      └──────┬───────┘
       │                     │                     │
       │  决定调用 read_file  │                     │
       │  工具               │                     │
       │────────────────────→│                     │
       │                     │  构造调用请求         │
       │                     │────────────────────→│
       │                     │                     │ 执行文件读取
       │                     │                     │←─────────────
       │                     │                     │
       │                     │   返回文件内容       │
       │                     │←────────────────────│
       │   返回文件内容       │                     │
       │←────────────────────│                     │
       │                     │                     │
```

## 传输层（Transport）

MCP 支持多种传输层：

### 1. Stdio（标准输入/输出）

默认传输方式，适合本地进程：

```json
// Claude Desktop 配置
{
  "mcpServers": {
    "filesystem": {
      "command": "node",
      "args": ["/path/to/server/dist/index.js"],
      "env": {
        "API_KEY": "xxx"
      }
    }
  }
}
```

**特点**：
- 进程间通信，通过 stdin/stdout
- 简单，适合本地
- 自动重启（进程崩溃时）

### 2. HTTP + SSE

适合远程 Server：

```python
# 服务端
from mcp.server import Server
from mcp.server.sse import SseServerTransport
import uvicorn

app = Server(
    name="my-server",
    version="1.0.0"
)

# 配置 SSE 传输
transport = SseServerTransport("/messages")
app.run(transport=transport)

# 客户端连接
# POST /messages - 发送请求
# GET /events - 接收 Server-Sent Events
```

### 3. WebSocket

适合双向实时通信：

```python
# WebSocket 传输
from mcp.server.websocket import WebSocketServer

server = Server(name="my-server")
ws_server = WebSocketServer(server, host="0.0.0.0", port=8080)
ws_server.run()
```

## 安全模型

MCP 内置多层安全机制：

### 1. 用户确认机制

```
用户：删除 /tmp/test.txt

Client → Server：调用 delete_file
Server → Client：需要用户确认
Client → UI：弹出确认对话框
用户：确认/取消

→ 只有用户确认后才执行
```

### 2. 资源隔离

```
不同 Server 只能访问自己的资源：
- 文件系统 Server：只能访问配置的目录
- GitHub Server：只能访问授权的仓库
- 数据库 Server：只能访问授权的数据库
```

### 3. 权限控制

```json
// Claude Desktop 配置 - 细粒度权限
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem"],
      "allowedDirectories": [
        "/Users/demo/projects",
        "/Users/demo/documents"
      ]
    }
  }
}
```

### 4. 敏感信息处理

```python
# Server 端：敏感信息不返回给 Client
@server.list_tools()
async def list_tools():
    return [
        Tool(
            name="api_call",
            description="调用外部 API",  # 不暴露 API Key
            inputSchema={...}
        )
    ]

@server.call_tool()
async def call_tool(name: str, arguments: dict):
    # 在 Server 内部处理 API Key
    # 只返回业务数据
    result = call_external_api(
        api_key=os.getenv("API_KEY"),  # 内部使用，不暴露
        **arguments
    )
    return result
```

## Server 开发模板

```python
from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp.types import Tool, TextContent
from pydantic import AnyUrl
import asyncio

# 创建 Server
app = Server("my-mcp-server")

# 注册工具
@app.list_tools()
async def list_tools() -> list[Tool]:
    """列出所有可用工具"""
    return [
        Tool(
            name="greet",
            description="打招呼",
            inputSchema={
                "type": "object",
                "properties": {
                    "name": {"type": "string", "description": "名字"}
                },
                "required": ["name"]
            }
        ),
        Tool(
            name="calculate",
            description="计算",
            inputSchema={
                "type": "object",
                "properties": {
                    "expression": {"type": "string", "description": "表达式"}
                },
                "required": ["expression"]
            }
        )
    ]

# 处理工具调用
@app.call_tool()
async def call_tool(name: str, arguments: dict) -> list[TextContent]:
    """处理工具调用"""
    if name == "greet":
        return [TextContent(
            type="text",
            text=f"你好，{arguments['name']}！"
        )]
    elif name == "calculate":
        result = eval(arguments["expression"])
        return [TextContent(
            type="text",
            text=f"计算结果：{result}"
        )]
    else:
        raise ValueError(f"Unknown tool: {name}")

# 主函数
async def main():
    async with stdio_server() as (read_stream, write_stream):
        await app.run(
            read_stream,
            write_stream,
            app.create_initialization_options()
        )

if __name__ == "__main__":
    asyncio.run(main())
```

## 本节小结

**MCP 架构**：
- **Host**：运行 AI 的应用（Claude Desktop、Cursor）
- **Client**：嵌入 Host 的客户端，负责与 Server 通信
- **Server**：提供能力的服务（文件系统、GitHub 等）

**通信流程**：
- 连接建立 → 能力发现 → 工具调用 → 返回结果

**传输层**：
- Stdio（默认，本地进程）
- HTTP + SSE（远程）
- WebSocket（双向实时）

**安全模型**：
- 用户确认机制
- 资源隔离
- 细粒度权限控制
- 敏感信息不暴露
