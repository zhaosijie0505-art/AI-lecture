# 第五节 构建你的第一个 Skill

## 动手实践

前面四节我们学了概念、价值、案例和技术原理，这一节我们真正动手，从零开始创建一个 Skill。

我们以一个实际场景为例：**创建一个"数据分析报告"Skill**，用于生成标准化的数据分析报告。

---

## 准备工作

### 需要的环境

- Claude.ai 账号（需要在 claude.ai/settings/capabilities 启用 Skills 功能）
- 基本的 YAML 语法知识
- 你所在领域的专业知识

### Skill 创建的基本流程

```
确定场景 → 提炼流程 → 编写 Skill → 测试验证 → 迭代优化
```

---

## Step 1：确定场景

创建 Skill 的第一步是**明确你要解决什么问题**。

问自己几个问题：
- 这个 Skill 要帮我做什么？
- 我现在做这件事的标准流程是什么？
- 我希望 AI 输出达到什么质量标准？
- 这件事有哪些关键步骤和规范？

对于"数据分析报告"Skill，答案大概是：

**目标**：生成结构清晰、分析深入、格式规范的数据分析报告

**当前流程**：
1. 理解数据背景和业务问题
2. 数据清洗和预处理
3. 描述性统计和趋势分析
4. 找出关键发现和异常
5. 形成结论和建议
6. 按公司模板输出报告

**质量标准**：
- 报告结构：背景 → 方法 → 分析 → 结论 → 建议
- 数据必须有来源标注
- 图表必须有清晰的标题和说明
- 结论必须有数据支撑

---

## Step 2：编写 Skill

### Skill 的基本格式

```yaml
name: data-analysis-report
description: >
  专业的中国市场数据分析报告生成。
  适用于销售数据、运营数据、市场调研数据的分析。
  当用户提供数据并要求分析时自动触发。
version: "1.0"
author: your-name

instructions:
  role: |
    你是一位专业的数据分析师，擅长用数据讲故事。
    你有丰富的中国市场行业分析经验，熟悉常用的分析方法。
    你的分析报告以结构清晰、数据扎实、观点明确著称。

  procedure: |
    按以下步骤进行数据分析报告生成：

    1. 理解背景
       - 明确业务问题是什么
       - 确认分析的目标和范围
       - 识别关键指标（KPI）

    2. 数据处理
       - 检查数据完整性和准确性
       - 识别并处理异常值
       - 必要时进行数据标准化

    3. 分析方法
       - 描述性统计：均值、中位数、分布情况
       - 趋势分析：时间序列变化、同比环比
       - 分类分析：按维度拆解（地区、产品、客户）
       - 相关性分析：指标之间的关系

    4. 关键发现
       - 从数据中提炼 3-5 个核心发现
       - 每个发现必须有数据支撑
       - 区分事实陈述和推断

    5. 结论与建议
       - 结论必须直接回应业务问题
       - 建议必须有可操作性
       - 区分短期和长期建议

  output_format: |
    报告必须包含以下部分：

    # 一、执行摘要（100字以内）
    一句话概括核心发现和建议方向。

    # 二、背景与目标
    说明分析的业务背景、分析目标和关键问题。

    # 三、数据概况
    描述数据来源、时间范围、样本量等基本信息。

    # 四、分析方法
    说明使用的分析方法和选择依据。

    # 五、核心发现（重点章节）
    用数据讲故事。每个发现：
    - 标题：有洞察力的发现概括
    - 数据支撑：具体数字和对比
    - 业务含义：这个数据说明什么

    # 六、结论
    直接回应业务问题，简洁明确。

    # 七、建议
    分短期和长期，建议必须有可操作性。

    # 八、数据附录（如有）
    详细数据表格、补充分析。

  quality_standards: |
    - 数据必须标注来源
    - 图表必须有标题和说明文字
    - 避免空洞的套话，每句话要有数据支撑
    - 专业但不晦涩，让决策者能看懂
    - 适当使用对比（环比、同比、行业基准）

  examples:
    - input: |
        分析这份月度销售数据，找出下滑原因。
        [数据表格]
      output: |
        完整的结构化分析报告，包含以上所有章节。

    - input: |
        竞品市场占有率分析。
      output: |
        市场分析报告，重点在竞争格局和趋势。
```

### 关键部分说明

**name**：Skill 的唯一标识符，用 kebab-case（短横线命名）。

**description**：这是自动匹配的关键。用自然语言描述清楚 Skill 做什么、适用于什么场景、什么时候触发。写清楚这个描述，隐式触发才可能生效。

**instructions.role**：定义 AI 的身份和专业背景。这不是空话，它会显著影响 AI 的分析视角和表达方式。

**instructions.procedure**：具体的操作步骤。写清楚先做什么、后做什么，决策点怎么判断。

**instructions.output_format**：输出规范。这保证报告的结构是一致的。

**instructions.quality_standards**：质量要求。写清楚什么算"做对了"，什么算"没做好"。

**examples**：正例示例。有时候一个示例胜过一千字的描述。

## Step 3：安装和测试

### 在 Claude.ai 中安装 Skill

1. 打开 claude.ai/settings/capabilities
2. 找到 Skills 部分
3. 选择"Create Skill"或"Import Skill"
4. 粘贴或上传你的 Skill YAML 定义
5. 保存并启用

### 测试你的 Skill

创建几个不同类型的测试任务：

**测试 1：标准场景**
```
请分析这份 Q3 销售数据，找出增长和下滑的因素。
```

**测试 2：边界情况**
```
这些数据有缺失值和异常值，请先处理再分析。
```

**测试 3：不同数据类型**
```
请分析这份用户调研数据，总结用户行为特征。
```

每次测试后检查：
- 报告结构是否符合你的 output_format？
- 内容质量是否达到 quality_standards？
- 有什么需要调整的地方？

---

## Step 4：迭代优化

Skill 很少能一次写完美。根据测试结果持续优化：

### 常见问题及解决

**问题 1：输出结构不稳定**
- 原因：output_format 描述不够具体
- 解决：在 output_format 中用更明确的标记，比如"必须包含"、"禁止缺少"

**问题 2：分析深度不够**
- 原因：procedure 缺少分析框架指导
- 解决：增加分析方法论的内容，或提供更多 examples

**问题 3：语言风格不符**
- 原因：role 定义不够具体
- 解决：详细描述期望的语言风格，提供正反例

**问题 4：参数处理有问题**
- 原因：变量定义不清晰
- 解决：确保每个参数都有明确的说明和使用示例

## 进阶技巧

### 1. 使用变量

让 Skill 适应不同场景：

```yaml
variables:
  report_type: 报告类型（月度/季度/年度）
  industry: 行业背景（互联网/金融/零售）
  audience: 受众（管理层/运营团队/投资人）

instructions:
  procedure: |
    生成{{report_type}}数据分析报告，
    重点关注{{industry}}行业的关键指标，
    结论部分要能让{{audience}}直接使用。
```

### 2. 条件分支

同一个 Skill 处理不同情况：

```yaml
conditions: |
  当用户提到"下滑"或"下降"时，先做归因分析；
  当用户提到"增长"或"扩张"时，重点分析驱动因素；
  当用户提到"对比"时，增加竞品对比分析章节。
```

### 3. Chain（链式调用）

复杂任务分解为多个 Skill：

```yaml
requires:
  - data-cleaning        # 先清洗数据
  - statistical-analysis # 再做统计分析
  - report-writing       # 最后生成报告
```

这三个 Skill 可以独立存在和测试，组合起来处理完整的分析流程。

## 常见坑和避坑指南

| 坑 | 问题 | 避坑方法 |
|----|------|---------|
| 描述太长 | 模型无法准确理解触发条件 | 描述控制在 2-3 句话 |
| 流程太细 | 执行僵化，缺乏灵活性 | 只定义关键步骤，给模型留空间 |
| 标准太模糊 | 输出质量不稳定 | 具体化每个标准，提供正反例 |
| 变量太多 | 使用门槛高 | 只保留必要的变量，其他用默认值 |
| 一次写完 | 肯定有遗漏 | 先写最小版本，快速测试，快速迭代 |

---

## 小结

构建第一个 Skill 的完整流程：

```
1. 确定场景：明确要解决什么问题
2. 提炼流程：把你的专业知识变成步骤
3. 编写 Skill：用 YAML 格式定义结构
4. 测试验证：用不同场景测试效果
5. 迭代优化：根据问题持续改进
```

重要的是**开始动手**，不要等完美再动手。先写一个能用的版本，测试、改进、再测试。Skills 是在实践中打磨出来的。

## 本章总结

本章我们从五个维度完整学习了 Skills：

| 维度 | 核心要点 |
|------|---------|
| **本质** | 专业知识、流程和最佳实践的封装，不是 Tool/Agent/RAG |
| **价值** | 一致性、知识封装、跨平台复用、组合使用 |
| **案例** | Figma、Box、Notion、Canva 等企业的真实落地效果 |
| **原理** | 触发→注入→执行的技术流程，Stack 组合机制 |
| **实践** | 从场景确定到迭代优化的完整构建流程 |

Skills 代表了 AI 应用的一个新阶段：**从"让 AI 做事"到"让 AI 按照我的标准做事"**。掌握 Skills，就掌握了把专业能力规模化的钥匙。
