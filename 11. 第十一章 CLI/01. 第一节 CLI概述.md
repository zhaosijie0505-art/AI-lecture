# 第一节 CLI 概述

## CLI 是什么

**CLI（Command-Line Interface）** 是通过文本命令与程序交互的界面。与 GUI（图形界面）用鼠标点击不同，CLI 通过键盘输入命令，程序通过文本回显输出结果。

```bash
# CLI 示例：文件操作
$ ls -la /Users/demo/projects
total 64
drwxr-xr-x  12 mlmap  staff  384 May 16 10:30 .
drwxr-xr-x   9 mlmap  staff  288 May 16 09:15 ..
-rw-r--r--   1 mlmap  staff  220 May 16 10:30 README.md
drwxr-xr-x   8 mlmap  staff  256 May 16 10:30 src

# GUI 操作：打开 Finder，点击文件夹
# CLI 操作：输入 ls 命令

# 本质相同：查看目录内容
# 区别：CLI 可编程、可组合、可远程、可自动化
```

### 核心概念

```
CLI 核心组件：

┌─────────────────────────────────────────┐
│              CLI 程序                    │
├─────────────────────────────────────────┤
│                                         │
│  命令（Command）                         │
│  $ gh pr create --title "..."           │
│      │                                  │
│      ├── gh（程序名）                   │
│      ├── pr（子命令）                   │
│      ├── create（操作）                 │
│      └── --title "..."（参数）          │
│                                         │
│  选项（Options）                        │
│  --title, --body, --base                │
│  -t, -b, -B（短选项）                   │
│                                         │
│  参数（Arguments）                      │
│  位置参数，不需要选项名                 │
│                                         │
│  子命令（Subcommands）                  │
│  gh pr create, gh repo clone           │
│  （类似 GUI 的菜单层级）                 │
│                                         │
└─────────────────────────────────────────┘
```

## CLI vs GUI

| 维度 | CLI | GUI |
|------|-----|-----|
| **操作方式** | 键盘命令 | 鼠标点击 |
| **学习曲线** | 陡峭（需要记忆命令） | 平缓（直观可视化） |
| **自动化能力** | 强（脚本化、管道组合） | 弱（依赖 UI 自动化工具） |
| **远程操作** | 原生支持（SSH） | 需要远程桌面 |
| **出错处理** | 精确的错误信息 | 弹窗，不精确 |
| **测试** | 简单（输入/输出验证） | 困难（像素级比对） |
| **AI 友好度** | 极高（文本 in/out） | 低（需要视觉理解） |
| **可组合性** | 管道 `A | B | C` | 难组合 |

### 为什么 AI 更擅长 CLI

```
GUI 的问题（对 AI 来说）：

1. 视觉渲染不稳定
   - 同一页面，窗口大小变化，元素位置就变
   - AI 需要理解和操作的是"逻辑"不是"像素"

2. 状态难以感知
   - 下拉菜单是否展开？
   - 按钮是否可点击？
   - 需要额外的界面状态查询

3. 跨平台差异大
   - macOS、Windows、Linux 的 GUI 完全不同
   - AI 需要针对每个平台单独适配

CLI 的优势（对 AI 来说）：

1. 输入输出都是文本
   - AI 的输入是什么，输出就是什么
   - 不需要渲染、理解像素

2. 确定性输出
   - 同样的命令，同样的输出
   - 不存在"看起来一样但实际不同"

3. 管道组合天然模块化
   - A | B | C
   - AI 只需要理解每个命令，不需要理解组合逻辑

4. 错误信息结构化
   - exit code: 0 成功，非 0 失败
   - stderr: 错误详情
   - 解析和处理都很直接
```

## AI 时代的 CLI

### 传统 CLI 的局限性

```
传统 CLI 的问题：

1. 需要记忆大量命令
   $ git rebase -i HEAD~3
   $ docker ps -a --format "{{.Names}}"
   $ kubectl get pods -n production -o jsonpath='{.items[*].metadata.name}'
   → 命令太长，参数太多，记忆成本高

2. 输出难以解析
   $ git log --oneline --graph
   → 纯文本，难以程序化提取结构化数据

3. 不支持自然语言
   → 用户必须知道具体命令，无法说"帮我部署"
```

### AI + CLI 的解决

```
AI + CLI 的组合：

用户（自然语言）：
  "帮我把这个 feature 分支 rebase 到 main，然后推送到远程"

AI 解析 → 生成命令序列：
  $ git checkout feature/login
  $ git fetch origin
  $ git rebase origin/main
  $ git push --force-with-lease

AI 执行 → 解析输出 → 反馈结果：
  ✓ Rebase 成功，无冲突
  ✓ 已推送到 origin/feature/login

→ 用户不需要记忆任何命令
→ AI 负责理解意图、生成命令、处理错误
```

## Shell 和 Bash 基础

### 常用 Shell 命令

```bash
# 文件操作
ls -la                    # 列出目录（含隐藏文件）
cd /path/to/dir           # 切换目录
pwd                       # 显示当前目录
mkdir -p a/b/c            # 递归创建目录
rm -rf path               # 强制删除
cp -r src dest             # 递归复制
mv old new                # 重命名/移动
touch file.txt            # 创建空文件

# 文件查看
cat file.txt              # 查看全部
head -n 20 file.txt       # 前20行
tail -n 100 file.txt      # 后100行
grep "pattern" file.txt    # 搜索
wc -l file.txt            # 行数统计

# 管道和组合
cat file.txt | grep "pattern" | sort | uniq
find . -name "*.py" | xargs grep "func"

# 环境变量
export VAR=value
echo $VAR

# 条件执行
command1 && command2       # command1 成功才执行 command2
command1 || command2       # command1 失败才执行 command2
command1 | command2       # command1 的输出作为 command2 的输入
```

### 退出码（Exit Code）

```bash
# 退出码约定
0    = 成功
1    = 一般错误
2    = shell 内置错误
127  = 命令未找到
130  = Ctrl+C 中断

# 检查退出码
$ command
$ echo $?   # 打印上一个命令的退出码

# 条件判断
if command; then
    echo "Success"
else
    echo "Failed"
fi
```

### Shell 脚本基础

```bash
#!/bin/bash
# first_script.sh

# 变量
NAME="World"
echo "Hello, $NAME"

# 条件
if [ "$1" == "dev" ]; then
    ENV="development"
elif [ "$1" == "prod" ]; then
    ENV="production"
else
    echo "Usage: $0 {dev|prod}"
    exit 1
fi

# 循环
for file in *.txt; do
    echo "Processing: $file"
done

# 函数
function deploy() {
    local env=$1
    echo "Deploying to $env..."
}

deploy "production"
```

## CLI 的 AI 应用场景

```
1. Shell Agent
   用户：帮我清理这个项目的 node_modules
   AI：识别 OS（Linux/macOS）→ 生成清理命令 → 执行 → 反馈结果

2. 代码部署
   AI：分析项目结构 → 生成 Dockerfile → 构建 → 推送 → SSH 部署

3. 数据处理管道
   用户：分析这个 CSV 文件，找出销量最高的产品
   AI：写 Python 脚本 → CLI 执行 → 解析结果 → 生成报告

4. 数据库操作
   用户：查询昨天注册的用户数量
   AI：生成 SQL → 通过 CLI 执行 → 格式化输出

5. Git 操作
   用户：帮我把这个 PR 的所有 commit squash 成一个
   AI：生成交互式 rebase 命令 → 执行 → 处理冲突
```

## 本节小结

**CLI（命令行界面）** 通过文本命令与程序交互，是 AI Agent 最天然的协作伙伴。

**为什么 AI 擅长 CLI**：
- 输入输出都是文本，AI 天然理解
- 输出确定性高，解析可靠
- 管道组合，模块化清晰
- 退出码机制，错误处理明确

**CLI 基础概念**：
- 命令、选项、参数、子命令
- 管道组合（`|`）
- 退出码（0=成功，非0=失败）
- Shell 脚本（变量、条件、循环、函数）
