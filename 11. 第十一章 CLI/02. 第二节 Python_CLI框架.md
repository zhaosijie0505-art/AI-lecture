# 第二节 Python CLI 框架

## 三大框架对比

Python CLI 生态有三个主流框架，按出现时间排序：

| 框架 | 发布时间 | 基于 | 特点 |
|------|---------|------|------|
| **Argparse** | Python 3.2+ | 标准库 | 语法繁琐，但无需安装 |
| **Click** | 2014年 | 无 | Decorator 风格，命令组合好 |
| **Typer** | 2019年 | Click | 类型安全，Auto-completion 强 |

```
选择建议：
- 快速脚本 + 不想装依赖 → Argparse
- 生产级 CLI + 需要命令组合 → Click
- 现代 Python 项目 + 类型提示 → Typer（推荐）
```

---

## Argparse（标准库）

### 基础用法

```python
import argparse

# 创建解析器
parser = argparse.ArgumentParser(
    description="文件处理工具",
    formatter_class=argparse.RawDescriptionHelpFormatter
)

# 添加参数
parser.add_argument("path", help="文件路径")
parser.add_argument("-o", "--output", help="输出路径")
parser.add_argument("-n", "--lines", type=int, default=10,
                    help="显示行数（默认10）")
parser.add_argument("-v", "--verbose", action="store_true",
                    help="详细输出")
parser.add_argument("--version", action="version", version="1.0.0")

# 解析参数
args = parser.parse_args()

# 使用
if args.verbose:
    print(f"处理文件: {args.path}")

with open(args.path) as f:
    lines = f.readlines()[:args.lines]
    print("".join(lines))
```

```bash
# 运行
$ python tool.py /path/to/file.txt -n 20 -v
$ python tool.py --help
```

### 子命令

```python
import argparse

parser = argparse.ArgumentParser()
subparsers = parser.add_subparsers(dest="command", help="子命令")

# 子命令：deploy
deploy_parser = subparsers.add_parser("deploy", help="部署应用")
deploy_parser.add_argument("--env", choices=["dev", "staging", "prod"],
                           required=True)
deploy_parser.add_argument("--name", required=True)

# 子命令：rollback
rollback_parser = subparsers.add_parser("rollback", help="回滚版本")
rollback_parser.add_argument("--env", required=True)
rollback_parser.add_argument("--version")

args = parser.parse_args()

if args.command == "deploy":
    print(f"部署 {args.name} 到 {args.env}")
elif args.command == "rollback":
    print(f"回滚 {args.env} 到 {args.version or '上一个版本'}")
```

### 优缺点

```
✓ Argparse 优点：
  - 标准库，无需安装
  - 文档完善，资料多
  - 足够简单，小脚本够用

✗ Argparse 缺点：
  - 语法啰嗦（特别是必填参数）
  - 类型安全差（str/int 不检查）
  - 子命令实现繁琐
  - 不支持命令组合
  - 默认不生成 shell 自动补全
```

## Click（生产级框架）

### 基础命令

```python
import click

@click.command()
@click.argument("path")
@click.option("-n", "--lines", default=10, help="显示行数")
@click.option("-v", "--verbose", is_flag=True, help="详细输出")
def view(path, lines, verbose):
    """查看文件内容"""
    if verbose:
        click.echo(f"读取文件: {path}")

    with open(path) as f:
        for i, line in enumerate(f):
            if i >= lines:
                break
            click.echo(line.rstrip())

if __name__ == "__main__":
    view()
```

```bash
$ python tool.py /path/to/file.txt -n 20
$ python tool.py --help
```

### 命令组合（Group）

```python
import click

@click.group()
def cli():
    """文件处理工具"""
    pass

@cli.command()
@click.argument("path")
def view(path):
    """查看文件"""
    with open(path) as f:
        click.echo(f.read())

@cli.command()
@click.argument("src")
@click.argument("dest")
def copy(src, dest):
    """复制文件"""
    import shutil
    shutil.copy2(src, dest)
    click.echo(f"已复制: {src} → {dest}")

@cli.command()
@click.argument("path")
@click.option("-r", "--recursive", is_flag=True, help="递归删除")
def delete(path, recursive):
    """删除文件"""
    import os
    if os.path.isdir(path):
        if recursive:
            shutil.rmtree(path)
            click.echo(f"已递归删除目录: {path}")
        else:
            click.echo("目录需要 -r 参数才能删除", err=True)
    else:
        os.remove(path)
        click.echo(f"已删除文件: {path}")

if __name__ == "__main__":
    cli()
```

```bash
$ python tool.py view /path/to/file.txt
$ python tool.py copy a.txt b.txt
$ python tool.py delete /path/to/file.txt -r
```

### 带参数的命令

```python
import click

@click.command()
@click.argument("name")
@click.argument("age", type=int)
@click.option("--email", default="", help="邮箱")
@click.option("--phone", default="", help="电话")
@click.option("--tags", multiple=True, help="标签（可多次指定）")
def create_user(name, age, email, phone, tags):
    """创建用户"""
    click.echo(f"创建用户: {name}, {age}岁")
    if email:
        click.echo(f"  邮箱: {email}")
    if phone:
        click.echo(f"  电话: {phone}")
    if tags:
        click.echo(f"  标签: {', '.join(tags)}")

# 运行
# python create_user.py alice 25 --email alice@example.com --tags developer --tags python
# 输出：
# 创建用户: alice, 25岁
#   邮箱: alice@example.com
#   标签: developer, python
```

### 提示和交互

```python
import click

@click.command()
def init():
    """初始化项目"""
    name = click.prompt("项目名称", default="my-project")
    framework = click.prompt("框架", type=click.Choice(["FastAPI", "Django", "Flask"]))
    db = click.confirm("是否启用数据库？")

    click.echo(f"\n项目配置：")
    click.echo(f"  名称: {name}")
    click.echo(f"  框架: {framework}")
    click.echo(f"  数据库: {'是' if db else '否'}")

    if click.confirm("确认创建？"):
        click.echo("正在创建...")
    else:
        click.echo("取消创建")
```

### 优缺点

```
✓ Click 优点：
  - Decorator 风格，代码清晰
  - 命令组合（group）强大
  - 内置进度条、颜色输出、提示
  - 生态丰富（插件多）
  - 社区认可度高（被大量项目使用）

✗ Click 缺点：
  - 还是不够类型安全
  - 类型提示支持有限
  - Auto-completion 需要额外配置
```

## Typer（现代推荐）

### 基础用法

```python
import typer
from typing import Optional

app = typer.Typer()

@app.command()
def view(
    path: str,
    lines: int = typer.Option(10, "--lines", "-n", help="显示行数"),
    verbose: bool = typer.Option(False, "--verbose", "-v", help="详细输出")
):
    """查看文件内容"""
    if verbose:
        typer.echo(f"读取文件: {path}")

    with open(path) as f:
        for i, line in enumerate(f):
            if i >= lines:
                break
            typer.echo(line.rstrip())

if __name__ == "__main__":
    app()
```

```bash
$ python tool.py view /path/to/file.txt --lines 20
$ python tool.py --help
```

### 类型安全和自动补全

```python
import typer
from typing import Annotated

app = typer.Typer()

@app.command()
def create(
    name: str = typer.Argument(..., help="用户名"),
    age: int = typer.Argument(..., ge=0, le=150, help="年龄"),
    email: Optional[str] = typer.Option(None, "--email", "-e", help="邮箱"),
    tags: list[str] = typer.Option([], "--tag", "-t", help="标签"),
):
    """创建用户"""
    typer.echo(f"创建用户: {name}, {age}岁")
    if email:
        typer.echo(f"  邮箱: {email}")
    if tags:
        typer.echo(f"  标签: {', '.join(tags)}")

# Typer 自动：
# - 参数类型检查（age 必须是 0-150）
# - Shell 自动补全（需要运行 typer install-completion）
# - 友好的错误提示
```

### 子命令

```python
import typer

app = typer.Typer()

@app.command()
def deploy(
    name: str,
    env: str = typer.Option(..., "--env", "-e",
                            case_sensitive=False,
                            help="环境 dev/staging/prod"),
    image: Optional[str] = None,
):
    """部署应用到指定环境"""
    typer.echo(f"部署 {name} 到 {env}")

@app.command()
def rollback(
    env: str,
    version: Optional[str] = None,
):
    """回滚到指定版本"""
    v = version or "上一个版本"
    typer.echo(f"回滚 {env} 到 {v}")

@app.command()
def status(env: str):
    """查看应用状态"""
    typer.echo(f"环境 {env} 运行正常")

if __name__ == "__main__":
    app()
```

```bash
$ python app.py deploy my-app --env prod
$ python app.py rollback --env staging
$ python app.py status --env dev
$ python app.py --help
```

### 与 FastAPI 生态结合

```python
# 一个项目，两个入口
# cli.py: 命令行
# api.py: HTTP API

# 共享业务逻辑
from models import User
from services import create_user as svc_create_user

# CLI
import typer
app = typer.Typer()

@app.command()
def create(name: str, email: str):
    user = svc_create_user(name, email)
    typer.echo(f"创建用户: {user.id}")

# API
from fastapi import FastAPI
api = FastAPI()

@api.post("/users")
def create(name: str, email: str):
    user = svc_create_user(name, email)
    return {"id": user.id}

# 运行
# CLI: python cli.py create alice alice@example.com
# API: uvicorn api:app
```

## 框架选择建议

```
选择标准：

1. 项目规模
   小脚本/单命令 → Argparse（无需依赖）
   中型工具/多命令 → Click（稳定可靠）
   现代 Python 项目 → Typer（类型安全）

2. 团队技术栈
   Python 3.10+，重类型提示 → Typer
   兼容 Python 2.7 / 3.6 → Click
   无外部依赖要求 → Argparse

3. 功能需求
   需要命令组合 → Click/Typer
   需要 Shell 自动补全 → Typer
   需要进度条/颜色 → Click/Typer
   需要插件系统 → Click
```

### 快速启动命令

```bash
# Argparse（无需安装）
python -c "import argparse; help(argparse)"

# Click
pip install click

# Typer（推荐）
pip install typer[all]  # 含 shell Completion
```

## 本节小结

**三大框架对比**：
- **Argparse**：标准库，无需安装，但语法啰嗦、类型安全差
- **Click**：Decorator 风格，命令组合强，生态丰富
- **Typer**：基于 Click，类型安全强，Auto-completion 好（**推荐**）

**选择建议**：
- 现代 Python 3.10+ 项目 → Typer
- 需要兼容旧版本 Python → Click
- 不想装依赖 → Argparse

**Typer 优势**：
- 类型提示 → 自动类型检查
- 自动生成 Shell Completion
- 与 FastAPI 生态无缝结合
- 代码简洁美观
