# 第六节 Claude CLI 集成

## Claude 的 CLI 工具

Anthropic 提供了多个官方 CLI 工具，让 Claude 的能力可以脱离浏览器直接在终端使用：

```
官方 CLI 工具：

1. Claude Code（推荐）
   → AI 代码助手，专注编程任务
   → claude --help

2. Claude Agent CLI
   → 通用的 AI Agent CLI
   → claude --agent --help

3. claude-stream
   → 流式输出 CLI
   → 支持管道组合
```

### 安装

```bash
# macOS
brew install claude

# 或下载二进制
# https://github.com/anthropics/claude-code/releases

# 验证安装
claude --version
claude --help
```

---

## Claude Code

### 基本用法

```bash
# 启动 Claude Code
claude

# 单次请求
claude "帮我写一个快速排序"

# 指定模型
claude --model claude-opus-4 "分析这段代码的时间复杂度"

# 输出到文件
claude "写一个 FastAPI 应用" > app.py
```

### Claude Code 的命令模式

Claude Code 内置了许多快捷命令（以 `/` 开头）：

```
/ask <question>     → 快速提问
/model <name>       → 切换模型
/clear              → 清除对话
/compact            → 压缩上下文
/exit               → 退出

# 代码相关
/review             → 代码审查
/test               → 生成测试
/fix                → 修复错误
/explain            → 解释代码
/refactor           → 重构代码
```

### 工作目录操作

```bash
# 指定工作目录
claude --dir /path/to/project "审查这个项目的代码结构"

# 自动读取目录下的文件
cd my-project
claude "生成这个项目的 README"
```

## Claude Agent CLI

### 通用的 Agent 交互

```bash
# Agent 模式（多轮对话）
claude --agent

# 指定任务
claude --task "帮我分析这个 CSV 文件的趋势"

# 限制迭代次数
claude --agent --max-turns 5

# 显示思考过程
claude --agent --show-thinking
```

### Agent 配置文件

Claude Agent 支持配置文件来自定义行为：

```json
// .claude.json
{
  "agent": {
    "maxTurns": 50,
    "tools": ["Read", "Write", "Bash", "Glob", "Grep", "WebSearch"],
    "model": "claude-opus-4-5"
  },
  "project": {
    "name": "my-project",
    "readOnly": false
  }
}
```

```json
// claude_desktop_config.json（Claude Desktop 配置）
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/demo"],
      "env": {}
    }
  }
}
```

## MCP 工具调用

### 通过 CLI 使用 MCP

```bash
# 查看可用的 MCP 工具
claude --mcp list

# 调用 MCP 工具
claude --mcp-server filesystem "读取 /path/to/file.txt"

# 自定义 MCP Server
claude --mcp-server my-server "python /path/to/mcp-server.py"
```

### MCP 配置示例

```bash
# Claude Desktop 中的 MCP 配置
# 位置：~/Library/Application Support/Claude/claude_desktop_config.json

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/demo"],
      "env": {}
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_xxx"
      }
    },
    "my-custom-server": {
      "command": "python",
      "args": ["/path/to/your/mcp/server.py"],
      "env": {}
    }
  }
}
```

### 在 Claude Code 中使用 MCP

```
在 Claude Code 中，直接用自然语言调用 MCP 工具：

用户：帮我读取 /Users/demo/project/README.md

Claude：
## README.md 内容

# My Project

这是一个示例项目...

---

用户：在 GitHub 上创建这个项目的 Issue

Claude（通过 GitHub MCP）：
✓ Issue 已创建
   标题："Initial Issue"
   URL: https://github.com/user/repo/issues/1
```

## 与 Claude API 的对比

| 维度 | Claude CLI | Claude API |
|------|-----------|-----------|
| **使用方式** | 终端命令 | HTTP API 调用 |
| **交互模式** | 对话式 / 单次请求 | 编程调用 |
| **上下文管理** | 自动压缩 | 手动管理 |
| **工具集成** | MCP 原生支持 | 需要手动实现 |
| **适用场景** | 快速实验 / 日常使用 | 生产集成 / 自动化 |

```
选择建议：

1. 快速实验 / 日常使用
   → Claude CLI（开箱即用）

2. 集成到自己的应用
   → Claude API + SDK

3. 需要自定义工具
   → MCP Server + Claude CLI 或 API

4. 自动化脚本
   → Claude API（更适合编程集成）
```

## CLI 输出处理

### 管道到其他工具

```bash
# Claude 输出 → grep 过滤
claude "列出所有 Python 文件" | grep "model"

# Claude 输出 → 保存
claude "分析这个项目" > analysis.txt

# Claude 输出 → jq 解析（JSON）
claude --format json "查询天气" | jq '.temperature'

# 管道输入
echo "帮我写一个函数" | claude
```

### 格式化输出

```bash
# 纯文本（默认）
claude "解释什么是闭包"

# JSON 输出
claude --format json "什么是 HTTP" | jq .

# Markdown 输出
claude --format markdown "Docker 入门" > docker-guide.md
```

## 实用技巧

### 1. 快速代码片段

```bash
# 生成代码并直接执行（Python）
claude "写一个计算斐波那契的函数" | python3

# 生成并保存
claude "写一个 FastAPI 例子" > app.py

# 管道输入
cat script.py | claaude "审查这段代码"
```

### 2. 批量处理

```bash
# 循环处理多个文件
for f in *.py; do
    claude "给 $f 添加文档注释"
done

# 并行处理
ls *.txt | xargs -P 4 -I {} claude "总结 {} 的内容"
```

### 3. Git 集成

```bash
# 提交信息
git diff | claude "生成提交信息"

# 代码审查
git log -10 | claude "分析这个项目的开发趋势"

# PR 描述
git log origin/main..HEAD | claude "生成 PR 描述"
```

### 4. 开发流程

```bash
# 需求 → 代码
claude "实现用户登录功能" > login.py

# 代码 → 测试
claude "为 login.py 写单元测试"

# 测试 → 修复
claude "运行测试并修复失败的用例"

# 文档
claude "为整个项目生成 API 文档" > API.md
```

## 与其他 CLI 的对比

| 工具 | 厂商 | 特点 |
|------|------|------|
| **Claude Code** | Anthropic | AI 原生，代码能力最强 |
| **Copilot CLI** | Microsoft | 集成 GitHub，代码补全强 |
| **ChatGPT CLI** | OpenAI | 通用对话，插件生态 |
| **Codex CLI** | OpenAI | 专注代码，API 调用 |

```
选择建议：

1. 主要写代码，追求代码质量
   → Claude Code / Copilot

2. 通用任务，日常使用
   → Claude Code / ChatGPT

3. 需要深度代码理解
   → Claude Code（更强推理）

4. 已有 GitHub 工作流
   → Copilot（GitHub 集成好）
```
