# 第五节：消息网关（gateway/）—— 同一套 Agent 引擎服务所有平台

## 1. 为什么需要消息网关

如果你只在终端里用 Hermes，`hermes` 命令就够了。但 Hermes 的真正威力在于**可以通过 Telegram、Discord、Slack、WhatsApp 等平台跟它对话**。这些平台的消息格式、API、认证方式完全不同，`gateway/` 就是用来统一这些差异的适配层。

```
┌─────────────────────────────────────────────┐
│  Telegram  │  Discord  │  Slack  │  ...  │
│   Bot API  │  Gateway   │   API   │       │
└────────────┬────────────┬─────────┬────────┘
             │            │         │
             ▼            ▼         ▼
┌─────────────────────────────────────────────┐
│              gateway/run.py                   │
│         GatewayRunner (主控逻辑)              │
└────────────┬────────────────────────────────┘
             │
             ▼
┌─────────────────────────────────────────────┐
│         AIAgent (同一套核心引擎)                │
└─────────────────────────────────────────────┘
```

这就是**适配器模式**（Adapter Pattern）的经典应用——上层统一，下层多变。

## 2. GatewayRunner：网关的心脏

```python
class GatewayRunner:
    def __init__(self, config_path: str = None):
        self.platforms: Dict[str, PlatformAdapter] = {}
        self.sessions: Dict[str, GatewaySession] = {}
        self.agent = AIAgent(...)
    
    async def start(self):
        """启动网关，连接所有平台适配器"""
        for platform_name, adapter in self.platforms.items():
            await adapter.connect()
        
        await self._run_message_loop()
    
    async def receive(self, platform: str, update: dict):
        """处理从某个平台收到的消息"""
        session = self._get_or_create_session(platform, update["chat_id"])
        response = await session.agent.run_conversation(update["message"])
        await self.platforms[platform].send(update["chat_id"], response)
```

关键设计：**一个 GatewayRunner 实例对应一个 AIAgent 实例**，所有平台的消息都汇入同一个 Agent。这意味着你在 Telegram 上开始的对话，在 Discord 上继续时，Agent 记得之前的内容。

## 3. 平台适配器：每个平台一套接口

每个平台有一个专门的适配器文件：

```
gateway/platforms/
  telegram.py      # Telegram Bot API
  discord.py       # Discord Bot API (gateway/intents)
  slack.py          # Slack Bot API
  whatsapp.py       # WhatsApp Business API
  signal.py         # Signal
  homeassistant.py  # Home Assistant
  qqbot.py          # QQ 机器人
```

适配器基类定义了统一接口：

```python
class BasePlatformAdapter:
    async def connect(self):
        """连接到平台的 WebSocket / Long Polling"""
        raise NotImplementedError
    
    async def send(self, chat_id: str, text: str, metadata: dict = None):
        """发送消息到指定会话"""
        raise NotImplementedError
    
    async def receive(self, update: dict):
        """解析平台特定的消息格式，转换为统一格式"""
        raise NotImplementedError
    
    def _transform_outgoing(self, text: str) -> str:
        """处理消息中的特殊元素（@提及、#话题、URL 等）"""
        return text
```

每个子类只需要实现这四个方法，其他逻辑（消息排队、重试、超时）由父类处理。

## 4. 会话管理：每个用户对话是独立的 AIAgent

```python
class GatewaySession:
    """管理单个用户会话的生命周期"""
    def __init__(self, platform: str, chat_id: str, agent: AIAgent):
        self.platform = platform        # 'telegram', 'discord', ...
        self.chat_id = chat_id         # 平台上的用户/群组 ID
        self.agent = agent             # 这个会话的 AIAgent 实例
        self.created_at = time.time()  # 用于判断会话是否过期
        self.last_activity = time.time()
    
    def is_expired(self) -> bool:
        """超过 SESSION_TIMEOUT 秒无活动，会话结束"""
        return time.time() - self.last_activity > SESSION_TIMEOUT
```

当用户通过 Telegram 发消息时，`GatewayRunner` 会检查是否已有该用户的会话：
- **有新会话**：创建新的 `GatewaySession`，里面有自己的 `AIAgent` 实例
- **有旧会话**：复用已有的 `AIAgent`，把新消息追加到现有会话

这里有一个重要的边界情况：**同一用户在多个平台上发消息，会被视为不同会话**（因为 `platform + chat_id` 是复合 key）。所以你在 Telegram 上跟 Agent 说\"帮我查一下航班\"，然后在 Discord 上问\"航班查到了吗\"，Agent 不知道你在问什么。

## 5. 消息流：从平台 Webhook 到 Agent 响应

```
Telegram 用户发送消息
         ↓
 Telegram 服务器 POST 到 Hermes 的 webhook URL
         ↓
 GatewayRunner.receive('telegram', update)
         ↓
 TelegramAdapter.parse(update) → 提取 chat_id 和 text
         ↓
 _get_or_create_session('telegram', chat_id)
         ↓
 session.agent.run_conversation(message)
         ↓
 AIAgent.run_conversation() → 调用 LLM → 工具 → ...
         ↓
 TelegramAdapter.send(chat_id, response_text)
         ↓
 Telegram Bot API → 发送消息给用户
```

每一步都可能出错，所以整个流程被包在 try/except 里，用日志记录异常，防止一个平台的问题影响其他平台。

## 6. 凭证锁：防止多实例抢占同一个 Bot Token

如果你在两台机器上同时运行同一个 Telegram Bot，两个实例会同时处理消息，用户会收到重复回复。Hermes 用**文件锁**防止这个问题：

```python
async def connect(self):
    from gateway.status import acquire_scoped_lock
    lock = await acquire_scoped_lock(f"telegram:{self.config['bot_token']}")
    if not lock:
        raise RuntimeError(f"Telegram bot already connected elsewhere")
    self._lock = lock
```

`acquire_scoped_lock()` 使用 Redis 或文件系统锁，确保同一个 bot token 只被一个 GatewayRunner 实例使用。

---

> [!NOTE]
> gateway/ 用适配器模式让一个 Agent 引擎同时服务所有消息平台；每个用户会话有独立的 AIAgent 实例；凭证锁防止多实例冲突；所有平台消息最终都走同一套 run_conversation() 核心循环。
