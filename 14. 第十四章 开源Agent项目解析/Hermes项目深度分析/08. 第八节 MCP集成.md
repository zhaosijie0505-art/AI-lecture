# 第七节：MCP 集成 —— 把外部工具接入 Hermes 的桥梁

## 1. 什么是 MCP

MCP（Model Context Protocol）是 Anthropic 提出的一个开放协议，**让 LLM 能够调用外部工具和数据源**。它类似于 USB 接口——有了 MCP，一个 Agent 就能连接各种外部工具（文件系统、GitHub、Slack 等），而不需要为每个工具单独开发适配代码。

```
┌─────────────────────────────────────────────┐
│         Hermes Agent (MCP Client)            │
└────────────────────┬──────────────────────┘
                     │ MCP 协议
┌────────────────────▼──────────────────────┐
│     MCP Server (如 @modelcontextprotocol/   │
│     server-filesystem, server-github)       │
│                                             │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐     │
│  │FileSystem│  │ GitHub  │  │ Slack   │     │
│  │ Server  │  │ Server  │  │ Server  │     │
│  └─────────┘  └─────────┘  └─────────┘     │
└─────────────────────────────────────────────┘
```

Hermes 的 `mcp_tool.py` 实现了 MCP Client，能够连接任何符合 MCP 协议的服务端。

---

## 2. 两种传输模式：Stdio 和 HTTP

MCP 支持两种传输方式，`mcp_tool.py` 两种都支持：

**Stdio 传输**（本地子进程）：
```python
from mcp.client.stdio import stdio_client
from mcp import StdioServerParameters

async with stdio_client(
    StdioServerParameters(
        command="npx",           # 可执行命令
        args=["-y", "@modelcontextprotocol/server-filesystem", "/tmp"]
    )
) as session:
    tools = await session.list_tools()
```

Stdio 用于本地工具——MCP 服务作为子进程启动，通过 stdin/stdout 与 Hermes 通信。

**HTTP Streamable 传输**（远程服务）：
```python
from mcp.client.streamable_http import streamablehttp_client

async with streamablehttp_client(
    "https://mcp-server.example.com/mcp",
    headers={"Authorization": "Bearer token"}
) as session:
    tools = await session.list_tools()
```

HTTP 用于远程工具——MCP 服务部署在远端，通过 HTTP WebSocket 或 SSE 与 Hermes 通信。

## 3. 工具注册：从 MCP 服务器到 Hermes 工具表

MCP 服务器的工具格式和 Hermes 的工具格式不一样，所以需要一个**转换层**：

```python
async def _discover_and_register(self, server_name: str, config: dict):
    """连接 MCP 服务器，获取工具列表，注册到 Hermes"""
    session = await self._create_session(server_name, config)
    mcp_tools = await session.list_tools()
    
    for mcp_tool in mcp_tools:
        hermes_schema = _convert_mcp_to_hermes_schema(mcp_tool)
        registry.register(
            name=f"mcp_{server_name}_{mcp_tool.name}",
            toolset="mcp",
            schema=hermes_schema,
            handler=_make_mcp_handler(session, mcp_tool),
            check_fn=lambda: session is not None,
            is_async=True,
        )
```

`mcp_*` 前缀区分了 MCP 工具和内置工具。调用时，`_make_mcp_handler()` 把 Hermes 的调用请求转换成 MCP 协议格式，发送给 MCP 服务器。

## 4. 安全环境过滤：不给 MCP 服务器乱传环境变量

MCP 服务器作为子进程运行，如果传了所有环境变量，可能泄露敏感信息（API key、数据库密码等）。Hermes 只传递**安全的环境变量**：

```python
_SAFE_ENV_KEYS = frozenset({
    "PATH", "HOME", "USER", "LANG", "LC_ALL",
    "TERM", "SHELL", "TMPDIR", "XDG_*",
})

def _build_safe_env(user_env: Optional[dict]) -> dict:
    """只传递安全变量给 MCP 子进程"""
    env = {}
    for key, value in os.environ.items():
        if key in _SAFE_ENV_KEYS or key.startswith("XDG_"):
            env[key] = value
    env.update(user_env or {})  # 用户显式指定的变量可以传递
    return env
```

用户可以在配置里显式指定要传给 MCP 的环境变量（用于 API key 等），但其他变量默认不传递。

## 5. 凭证脱敏：不让 MCP 错误信息泄露密钥

```python
_CREDENTIAL_PATTERN = re.compile(
    r"(?:ghp_[A-Za-z0-9_]{1,255}|"
    r"sk-[A-Za-z0-9_]{1,255}|"
    r"Bearer\s+\S+|...)"
)

def _sanitize_error(text: str) -> str:
    """从错误信息中移除凭证"""
    return _CREDENTIAL_PATTERN.sub("[REDACTED]", text)
```

当 MCP 调用出错时，错误信息中的 API key、GitHub token 等会被替换成 `[REDACTED]`。这防止了：错误日志被截图分享时，敏感凭证意外泄露。

## 6. 采样回调：MCP 服务器也能调用 LLM

MCP 服务器可能需要 LLM 来完成任务（比如让 Agent 生成一段代码）。Hermes 实现了 `SamplingHandler` 来处理这些请求：

```python
class SamplingHandler:
    """处理 MCP 服务器的 LLM 请求（sampling callbacks）"""
    
    def _check_rate_limit(self) -> bool:
        """滑动窗口限速"""
        now = time.time()
        self._rate_timestamps = [
            t for t in self._rate_timestamps if t > now - 60
        ]
        return len(self._rate_timestamps) < self.max_rpm
```

这个机制让 MCP 服务器和 Hermes Agent 之间形成**双向通信**——不只是 Hermes 调用 MCP 工具，MCP 服务器也能回调 Hermes 的 LLM。

## 7. 配置示例：接入文件系统 MCP 服务器

```yaml
mcp_servers:
  filesystem:
    command: "npx"
    args: ["-y", "@modelcontextprotocol/server-filesystem", "/tmp"]
    timeout: 120
  github:
    command: "npx"
    args: ["-y", "@modelcontextprotocol/server-github"]
    env:
      GITHUB_PERSONAL_ACCESS_TOKEN: "ghp_xxxx"
```

这是配置一个本地文件系统 MCP 服务器和一个 GitHub MCP 服务器。配置简洁明了——声明命令、参数、可选的环境变量就够了。

---

> [!NOTE]
> MCP 集成让 Hermes 能连接任何 MCP 协议的工具服务端。Stdio 用于本地、HTTP 用于远程；工具格式自动转换；环境变量被严格过滤；凭证在错误信息中被脱敏。
