# 第八节：agent/ 组件 —— Prompt 构建与上下文压缩

## 1. agent/ 目录里有什么

`agent/` 目录包含了 AIAgent 运行时的核心支撑模块：

```
agent/
├── prompt_builder.py      # 系统提示构建
├── context_compressor.py  # 上下文窗口压缩
├── model_metadata.py      # 模型元数据（上下文长度、token 估算）
├── auxiliary_client.py   # 辅助 LLM 客户端（视觉、总结）
├── display.py             # TUI 显示控制（spinner、工具预览）
├── error_classifier.py    # API 错误分类与恢复策略
└── retry_utils.py        # 重试策略
```

这些模块不参与核心 Agent 循环，但为循环提供必要支撑——Prompt 怎么写、上下文满了怎么办、模型支持什么功能。

---

## 2. prompt_builder.py：系统提示是怎么组装的

AIAgent 的系统提示不是硬编码的一长串文本，而是**由多个组件动态拼合**的：

```python
DEFAULT_AGENT_IDENTITY = (
    "You are Hermes Agent, an intelligent AI assistant created by Nous Research. "
    "You are helpful, knowledgeable, and direct..."
)

MEMORY_GUIDANCE = (
    "You have persistent memory across sessions. Save durable facts using the memory "
    "tool: user preferences, environment details, tool quirks, and stable conventions..."
)

TOOL_USE_ENFORCEMENT_GUIDANCE = (
    "You MUST use your tools to take action — do not plan to do something without "
    "actually doing it. Use the appropriate tool and return the result..."
)
```

组装流程：
1. 从配置文件读取系统提示模板
2. 注入当前日期、可用工具列表、启用的 skills
3. 添加内存指南、工具使用约束
4. 扫描内容安全（prompt injection 检测）

```python
def build_system_prompt(
    identity: str,
    memory_guidance: str,
    enabled_skills: List[str],
    enabled_tools: List[str],
    current_date: str,
    platform: str,
    session_context: dict,
) -> str:
    parts = [
        identity,
        memory_guidance,
        _build_skill_section(enabled_skills),
        _build_tool_section(enabled_tools),
        _build_platform_section(platform),
        _build_context_section(session_context),
    ]
    return "\n\n".join(filter(None, parts))
```

这种模块化设计让你可以**按需定制系统提示**——比如在特定平台（Slack）上添加平台特定的指令，在特定项目上添加 AGENTS.md 里的上下文。

## 3. 上下文威胁检测：防止 prompt injection

Hermes 支持加载用户提供的上下文文件（AGENTS.md、SOUL.md 等），但这些文件可能包含恶意指令——比如"ignore previous instructions"。

```python
_CONTEXT_THREAT_PATTERNS = [
    (r'ignore\s+(previous|all|above|prior)\s+instructions', "prompt_injection"),
    (r'do\s+not\s+tell\s+the\s+user', "deception_hide"),
    (r'system\s+prompt\s+hijack', "prompt_hijack"),
    ...
]

def _scan_context_content(content: str, filename: str) -> str:
    """扫描上下文文件中的 prompt injection 尝试"""
    threats = []
    for pattern, threat_type in _CONTEXT_THREAT_PATTERNS:
        if re.search(pattern, content, re.IGNORECASE):
            threats.append(f"[{threat_type}] in {filename}")
    
    if threats:
        warning = f"\n\n⚠️ SECURITY WARNING: Potentially malicious content detected:\n"
        warning += "\n".join(threats)
        warning += "\n\nPlease review these files for potential prompt injection attacks."
        return content + warning
    
    return content
```

如果检测到威胁，在提示末尾追加安全警告，但不拒绝加载——让用户决定是否继续。这个设计非常务实：**不阻断，但明确警告**。

## 4. context_compressor.py：上下文窗口满了怎么办

LLM 的上下文窗口是有限的。当对话太长时，Hermes 需要**压缩中间的消息**，保留开头和结尾的"头部+尾部"：

```
┌─────────────────────────────────────────────┐
│ [系统提示] [初始化对话] [早期对话] ... [最近对话] │
│    ↑                                      ↑ │
│  HEAD (保留)                           TAIL (保留)  │
└─────────────────────────────────────────────┘
         ↓
    压缩中间部分
         ↓
┌─────────────────────────────────────────────┐
│ [系统提示] [总结：早期对话...] [最近对话]       │
└─────────────────────────────────────────────┘
```

总结用**廉价快速的模型**（如 GPT-4o-mini），不占用主模型的上下文：

```python
SUMMARY_PREFIX = (
    "[CONTEXT COMPACTION — REFERENCE ONLY] "
    "Earlier turns were compacted into the summary below. "
    "This is a handoff from a previous context window — "
    "treat it as background reference, not direct conversation..."
)

async def _summarize_turns(turns: List[dict], summarizer_client) -> str:
    """用快速模型总结中间对话轮次"""
    summary_prompt = (
        f"Summarize the following conversation concisely, preserving key facts, "
        f"decisions, and any ongoing tasks:\n\n"
        + "\n\n".join(f"[Turn {i}]: {t}" for i, t in enumerate(turns))
    )
    response = await summarizer_client.chat.completions.create(
        model="gpt-4o-mini",  # 廉价模型
        messages=[{"role": "user", "content": summary_prompt}]
    )
    return response.choices[0].message.content
```

## 5. model_metadata.py：模型能力速查表

不同模型支持的功能不同——有些支持原生 function calling，有些只支持 tool calling；有些支持 reasoning，有些不支持。

```python
MODEL_METADATA = {
    "anthropic/claude-sonnet-4": {
        "context_window": 200_000,
        "supports_function_calling": True,
        "supports_vision": True,
        "supports_reasoning": True,
        "supports_prompt_caching": True,
        "max_output_tokens": 8192,
    },
    "openai/gpt-4o": {
        "context_window": 128_000,
        "supports_function_calling": True,
        "supports_vision": True,
        "supports_reasoning": False,
        "supports_prompt_caching": False,
        "max_output_tokens": 16384,
    },
    ...
}
```

这个元数据让 Hermes 能在运行时做出正确决策：
- 如果模型不支持 function calling，降级到 text mode
- 如果模型支持 prompt caching，用 `prompt_caching.py` 优化 token 费用
- 如果模型支持 vision，图片处理用 vision 工具而不是转成文本

> [!NOTE]
> agent/ 组件负责：组装系统提示（identity + memory + tools + platform）、检测 prompt injection 威胁、用快速模型压缩长对话的中间部分、提供模型能力元数据让运行时决策更智能。
