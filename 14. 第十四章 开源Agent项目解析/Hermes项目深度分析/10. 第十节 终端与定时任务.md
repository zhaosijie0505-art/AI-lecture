# 第九节：终端工具与定时任务 —— Agent 怎么执行命令和做日程管理

## 1. terminal_tool：跨环境执行命令

`tools/terminal_tool.py` 是 Hermes 执行 Shell 命令的核心工具。它不是简单调用 `subprocess.run()`，而是为不同执行环境提供了统一的抽象：

```python
def get_active_env():
    """根据 TERMINAL_ENV 选择执行环境"""
    env = os.getenv("TERMINAL_ENV", "local")
    if env == "docker":
        return DockerEnvironment()
    elif env == "modal":
        return ModalEnvironment()
    elif env == "ssh":
        return SSHEnvironment()
    elif env == "daytona":
        return DaytonaEnvironment()
    elif env == "singularity":
        return SingularityEnvironment()
    return LocalEnvironment()
```

**为什么需要这么多环境？**

- **Local**：本地执行，简单直接
- **Docker**：隔离运行，危险命令不会污染宿主机
- **Modal**：云端无服务器执行，有 GPU，适合 AI 任务
- **SSH**：远程服务器执行，代码在远端跑
- **Singularity**：HPC 高性能计算环境，容器化但不需要 root
- **Daytona**：另一个云端开发环境

每种环境都继承同一个基类，实现统一的接口：

```python
class BaseEnvironment:
    async def execute(self, command: str, timeout: int = None) -> dict:
        """执行命令，返回 {stdout, stderr, exit_code}"""
        raise NotImplementedError
    
    def transform_command(self, command: str) -> str:
        """命令转换（比如 sudo 处理）"""
        return command
```

## 2. Sudo 密码处理：让自动化不被打断

自动化脚本里执行 `sudo` 命令会卡住等待密码。Hermes 做了处理：

```python
def _transform_sudo_command(command: str):
    """将 'sudo cmd' 转换为从 stdin 读取密码的形式"""
    transformed, has_real_sudo = _rewrite_real_sudo_invocations(command)
    if has_real_sudo:
        sudo_password = os.environ.get("SUDO_PASSWORD", _cached_sudo_password)
        if sudo_password:
            return transformed, sudo_password + "\n"
    return command, None
```

转换后的命令会加上 `-S -p ''` 参数，告诉 sudo 从 stdin 读取密码。这样 `terminal_tool` 就能在执行前把密码塞进去，不需要人工干预。

这个功能有一个潜在风险：密码存在环境变量里可能泄露。所以 Hermes 提供了 `_cached_sudo_password` 选项，让密码只存在内存中，不落盘。

## 3. 危险命令审批：防止 Agent 删库跑路

```python
_DESTRUCTIVE_PATTERNS = re.compile(
    r"""(?:^|\s|&&|\|\||;|`)(?:
        rm\s|rmdir\s|
        mv\s(?!.*\.bak$)|
        sed\s+-i|
        truncate\s|
        dd\s|
        shred\s|
        git\s+(?:reset|clean|checkout)\s
    )""", re.VERBOSE)

def _check_all_guards(command: str, env_type: str) -> dict:
    """检查命令是否需要审批"""
    return _check_all_guards_impl(
        command, env_type,
        approval_callback=_approval_callback
    )
```

当检测到危险模式（如 `rm -rf`、`dd`、`git reset --hard`），工具会暂停执行，弹出交互式确认：

```
⚠️ DESTRUCTIVE COMMAND DETECTED
Command: rm -rf /tmp/test/*
Environment: local

This command will permanently delete files.
Do you want to proceed? [y/N]:
```

`approval_callback` 负责弹出这个确认框，按 y 就执行，按 n 就取消。这是 Hermes 最重要的安全机制之一。

## 4. 中断机制：Ctrl+C 能立刻停下

```python
def terminal_tool(command: str, timeout: int = None, ...):
    proc = subprocess.Popen(command, shell=True)
    
    while proc.poll() is None:
        if is_interrupted():  # ← 定期检查中断标志
            proc.kill()
            return json.dumps({"error": "Interrupted by user"})
        time.sleep(0.1)      # 每 100ms 检查一次
    
    return json.dumps({
        "stdout": stdout,
        "stderr": stderr,
        "exit_code": proc.returncode,
    })
```

`is_interrupted()` 检查一个全局的 `threading.Event`。当用户在 CLI 按 Ctrl+C，或者在 Gateway 发送 `/stop` 命令，这个 Event 就会被设置，所有正在运行的工具都会在下一次检查时退出。

## 5. cron/ 调度器：定时任务是怎么工作的

`cron/` 实现了一个类 cron 的调度器，但比传统 cron 更强大——任务描述用自然语言，执行结果可以投递到任意平台：

```python
def tick(adapters=None, loop=None):
    """每分钟被调度器调用一次"""
    due_jobs = get_due_jobs()  # 获取所有到期的任务
    
    for job in due_jobs:
        # 1. 执行任务
        output = _build_job_prompt(job)
        
        # 2. 投递结果
        if not output.startswith(SILENT_MARKER):
            _deliver_result(job, output, adapters, loop)
        
        # 3. 计算下次运行时间
        advance_next_run(job)
```

## 6. 文件锁防止重复执行

如果 `tick()` 同时被 gateway、daemon、systemd timer 触发，会导致同一个任务被执行多次。Hermes 用文件锁防止：

```python
_LOCK_FILE = _hermes_home / "cron" / ".tick.lock"

def _acquire_tick_lock() -> bool:
    """使用 fcntl 获取 tick 锁"""
    lock_fd = open(_LOCK_FILE, "w")
    try:
        fcntl.flock(lock_fd, fcntl.LOCK_EX | fcntl.LOCK_NB)
        lock_fd.write(str(os.getpid()))
        return True
    except BlockingIOError:
        return False
```

`LOCK_NB`（non-blocking）表示如果锁被占用就立即返回错误，不等待。只有拿到锁的进程才执行 tick，其他进程直接跳过。

## 7. 任务结果投递：让结果找到你

```python
def _resolve_delivery_target(job: dict) -> Optional[dict]:
    """解析任务的投递目标"""
    deliver = job.get("deliver", "local")
    
    if deliver == "local":
        return None  # 不投递，存本地
    if deliver == "origin":
        return {"platform": origin["platform"], "chat_id": origin["chat_id"]}
    
    # deliver: "telegram:123456" 或 "discord:#engineering"
    parts = deliver.split(":", 1)
    return {"platform": parts[0], "chat_id": parts[1]}
```

`deliver` 参数指定结果发到哪里：
- `"local"`：存到 `~/.hermes/cron/output/`，不发
- `"origin"`：回到触发这个 cron 的人所在的平台
- `"telegram:123456"`：发到指定 Telegram 会话
- `"discord:#engineering"`：发到 Discord 的指定频道

这就是 Hermes 的\"主动推送\"能力——Agent 不是等着你问，它会定时执行任务，然后把结果送到你面前。
